## Blocker — Runtime Capability Gap

The GSD autonomous workflow dispatches named subagents via the `Task()` tool with `subagent_type` arguments (e.g., `subagent_type="gsd-ui-researcher"`, `subagent_type="gsd-planner"`, `subagent_type="gsd-executor"`, `subagent_type="gsd-code-reviewer"`).

**This runtime does not expose `Task()`.** Verified via `ToolSearch query="select:Task"` — no match. The workflows that require `Task()` for subagent dispatch:

- `ui-phase.md` — `Task(subagent_type="gsd-ui-researcher")`, `Task(subagent_type="gsd-ui-checker")`
- `plan-phase.md` — planner + plan-checker subagents
- `code-review.md` / `code-review-fix.md` — reviewer subagents
- `ui-review.md` — ui-auditor subagent
- `audit-milestone.md` — auditor subagents
- `execute-phase.md` (via `execute-plan.md`) — executor subagents

`Skill()` is the only subagent-like dispatch available, but it returns the workflow prompt for the orchestrator to execute — and that prompt itself contains `Task()` calls. Recursion has no terminating tool.

This matches the operator's escalation criteria: *"The autonomous workflow encounters an unrecoverable error (not a single phase failure — a workflow-level error)."*
