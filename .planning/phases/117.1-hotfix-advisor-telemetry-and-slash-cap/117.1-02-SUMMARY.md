# Phase 117.1 Plan 02: Slash command guild-cap remediation — allowlist implementation Summary

Replace `native-cc-commands.classifyCommand`'s open-by-default fallback with an explicit ALLOWED_NATIVES set of six command names. Drops Discord registration body from 193 to ~77, comfortably under the 90/guild cap that fired CMD-07 in production 2026-05-12.

## What changed

### Source — `src/manager/native-cc-commands.ts`

- Added `ALLOWED_NATIVES: ReadonlySet<string>` containing `{model, permissions, effort, compact, cost, help}` with a multi-paragraph doc-block referencing Phase 117.1-02, the production CMD-07 incident, SLASH-AUDIT.md, and the 193 → ~77 math.
- Rewrote `classifyCommand(name)` body:
  - `if (!ALLOWED_NATIVES.has(name)) return "skip";`
  - `if (CONTROL_PLANE.has(name)) return "control-plane";`
  - `return "prompt-channel";`
- Retained `CONTROL_PLANE` and `SKIP` sets — `SKIP` is now documentation-only (those names are also absent from the allowlist, so the gate is the allowlist, not SKIP). Comment updated to call this out explicitly.
- Updated the docblock at `buildNativeCommandDefs` (ACL filter rationale) to reference the allowlist instead of the skip-set.

## Deviations from Plan

### Auto-fixed Issues

None — the implementation matches the audit's proposed code block verbatim.

### Scope-honest deviations from the original PLAN.md

The original PLAN.md (`117.1-02-PLAN.md`) framed the fix as a yaml-pruning exercise (T02 = prune per-agent `slashCommands` in production yaml). The T01 audit (`SLASH-AUDIT.md`) revealed the per-agent yaml is NOT the dominant contributor — the 5 per-agent commands on Admin Clawdy account for ~3% of the total. The root cause is the open-by-default classifier auto-registering every SDK-reported command (incl. project/user `.claude/commands/*.md` names) when an agent has `settingSources: [project, user]`. The plan was redirected by the audit toward the allowlist fix, which this commit implements. T03 (cap raise 90 → 95) is explicitly SKIPPED per the audit — the allowlist alone reaches ~77 commands.
