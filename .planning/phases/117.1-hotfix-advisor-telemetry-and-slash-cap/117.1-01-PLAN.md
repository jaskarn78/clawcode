# Plan 117.1-01 ‚Äî Bridge advisor event telemetry

**Phase:** 117.1
**Status:** ready
**Dependencies:** none (Phase 117 already shipped)
**Estimated tasks:** 2

## Goal

Add structured INFO log lines in `bridge.ts:onInvoked` and `bridge.ts:onResulted` so production logs contain ground truth for every native advisor invocation. Closes the silent-path gap that masked Issue 2 in the operator smoke.

## Why

Production deployed Phase 117 native advisor wiring. Admin-clawdy claimed advisor fired. Logs contain no evidence either way because the bridge subscribers at `bridge.ts:791,792` and `:814,815` emit no telemetry. We cannot diagnose the next operator report without this.

## Files touched

- **MODIFIED** `src/discord/bridge.ts` ‚Äî inside the `onInvoked`/`onResulted` callbacks registered around `:791-815`.
- **MODIFIED** `src/discord/__tests__/bridge-advisor-footer.test.ts` ‚Äî add assertions on captured log calls.

## Tasks

### T01 ‚Äî Add the two INFO log lines

In `src/discord/bridge.ts`, locate the `onInvoked` callback (around line 791) and the `onResulted` callback. Add one INFO log line per handler:

```typescript
// onInvoked
this.log.info(
  {
    agent: agentName,
    channel: channelId,
    userMessageId,
  },
  "advisor invoked (native server tool fired)",
);
```

```typescript
// onResulted
this.log.info(
  {
    agent: agentName,
    channel: channelId,
    variant: resultVariant,
    ...(resultVariant === "advisor_tool_result_error" ? { errorCode: event.errorCode } : {}),
  },
  "advisor resulted",
);
```

Field names match existing pino log line conventions in this file. INFO level keeps them visible in production without DEBUG noise.

**Verify:** `npm run typecheck` clean.

**Commit:** `feat(117.1-01): T01 ‚Äî bridge telemetry for advisor:invoked/resulted`

### T02 ‚Äî Test assertion

Extend `src/discord/__tests__/bridge-advisor-footer.test.ts`. Two new assertions:

- After firing a synthetic `advisor:invoked` event into the bridge's subscriber, capture the `log.info` mock and assert it was called with `level === "advisor invoked (native server tool fired)"` (or whatever message string is chosen) and a fields object containing `agent`, `channel`, `userMessageId`.
- After firing a synthetic `advisor:resulted` event with each of the three variants (`advisor_result`, `advisor_redacted_result`, `advisor_tool_result_error`), assert the `variant` field on the log line matches. For the error variant, assert `errorCode` is present.

**Verify:** `npm test -- src/discord/__tests__/bridge-advisor-footer.test.ts` ‚Äî all prior 12 cases plus 2 new ones (or 4 if expanded per variant) pass.
