## Why this phase exists

Phase 117 shipped to production and immediately surfaced two failure modes the smoke procedure didn't catch:

### Issue 1 — `/clawcode-verbose` invisible in Discord

Production daemon log at `2026-05-12T23:54:52Z`:

```
level=50  guildId=1467620213574074551  commandCount=193  limit=90
msg="too many commands for guild — refusing to register (CMD-07)"
```

ClawCode tried to register 193 application commands against one guild. Discord's hard limit is 100; the Phase 87 pre-flight assertion at `src/discord/slash-commands.ts:1186` correctly refused the entire batch to avoid a partial-state. **All 193 commands — including `/clawcode-verbose` from Phase 117-11 — were never registered with Discord.** Any control command added since the last successful registration is invisible to operators.

Breakdown of the 193:
- `DEFAULT_SLASH_COMMANDS`: 51 (slash-types.ts)
- `CONTROL_COMMANDS`: 15 (slash-types.ts; includes the new `/clawcode-verbose`)
- Per-agent custom `slashCommands` from `clawcode.yaml`: ~127 (sum across active agents)
- GSD_SLASH_COMMANDS: 0 contributed (correctly routed to `gsdSubcommands` and suppressed since Phase 999.32 `/gsd-do` consolidation)

The per-agent total is the dominant cost. Many agents declare overlapping custom slashCommands that have either fallen out of use or have native equivalents.

### Issue 2 — Native advisor production telemetry is dark

Admin-clawdy claimed in `#admin-clawdy` (channel `1494117043367186474`): *"Advisor fired (budget: 10 → 10 today, so it counted as a freebie, weird)."*

Three contradictions in that one report:

1. **Agent has no IPC to query AdvisorBudget.** The "10 → 10" reading is sourced from somewhere — likely the agent reading the awareness block in its own system prompt (which mentions the daily-default of 10), not actual introspection. Suspect: confabulation per the `feedback_soul_guard_against_agent_hallucinated_tool_use_claims` failure mode (Phase 999.22 precedent).

2. **The triggering question** *"So you have ask advisor capability yes?"* is a META-question about capability, not substantive work. Per the timing prompt the 117-08 awareness block injects: advisor fires *"BEFORE substantive work — before writing, before committing to an interpretation, before building on an assumption."* A capability check does not qualify. Native advisor should NOT have fired.

3. **No production telemetry confirms a real advisor invocation.** The Discord bridge subscribers at `bridge.ts:791,792` (`onInvoked`, `onResulted`) emit nothing observable — no log lines, no metrics. We cannot tell from the daemon log whether the SDK actually emitted `server_tool_use{name:"advisor"}` blocks. This is a **silent path** in the worst sense: we deployed a feature and can't tell if it works.

The pattern this matches verbatim is `feedback_silent_path_bifurcation`: *"before adding telemetry/IPC/handler code, verify production actually executes that path."* Phase 117 added code without verifying the production execution path; the smoke procedure in `117-10-SMOKE.md` documented the right steps but was never run for real.

## Architecture decisions

### Telemetry shape

Match existing pino log line patterns. INFO level (40), one line per event, with structured fields:

```typescript
this.log.info(
  { agent, channel: channelId, userMessageId, level: "advisor:invoked" },
  "advisor invoked",
);
```

```typescript
this.log.info(
  { agent, channel: channelId, variant: resultVariant, level: "advisor:resulted" },
  "advisor resulted",
);
```

The `variant` field discriminates `advisor_result` / `advisor_redacted_result` / `advisor_tool_result_error` per RESEARCH §13.4. Errors include `errorCode`.

### Slash command pruning strategy

Three pruning levers in order of preference:

1. **Remove duplicate/unused per-agent slashCommands** from production `clawcode.yaml`. The biggest single opportunity. Many agents have inherited slashCommands from earlier phases that are now redundant with `CONTROL_COMMANDS` or never invoked.
2. **Cap raise from 90 → 95** if (1) doesn't get under 90 cleanly. Keeps a 5-slot safety margin under Discord's 100.
3. **Defer hard re-architecture** (per-guild sharding, lazy registration) — not in scope; flag for follow-up if (1)+(2) insufficient.

The executor for 117.1-02 must audit the actual production yaml (`/etc/clawcode/clawcode.yaml` on clawdy), not the local example file. The audit produces a per-agent diff showing which slashCommands are removed and the reasoning.
