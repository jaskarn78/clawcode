# Plan 117-06: Config schema (`advisor` block) + loader resolvers

## Goal

Land the Zod schema for `defaults.advisor.{backend, model, maxUsesPerRequest, caching}` plus a per-agent override at `agents[].advisor`. Implement loader resolver functions following the `resolveRuntime` pattern at `src/config/loader.ts:393`. Reject `"portable-fork"` as a backend value (Phase 118 only).

<task id="117-06-T04">
  <objective>Add resolver functions to `src/config/loader.ts`.</objective>
  <action>
    1. Near `resolveRuntime` (~:393), add:
       ```typescript
       import type { BackendId } from "../advisor/types.js";

       export function resolveAdvisorBackend(
         agent: { advisor?: { backend?: BackendId } } | undefined,
         defaults: { advisor?: { backend?: BackendId } } | undefined,
       ): "native" | "fork" {
         const v = agent?.advisor?.backend ?? defaults?.advisor?.backend ?? "native";
         return v === "fork" ? "fork" : "native";
       }
       export function resolveAdvisorModel(
         agent: { advisor?: { model?: string } } | undefined,
         defaults: { advisor?: { model?: string } } | undefined,
       ): string {
         return agent?.advisor?.model ?? defaults?.advisor?.model ?? "opus";
       }
       export function resolveAdvisorMaxUsesPerRequest(
         agent: { advisor?: { maxUsesPerRequest?: number } } | undefined,
         defaults: { advisor?: { maxUsesPerRequest?: number } } | undefined,
       ): number {
         return agent?.advisor?.maxUsesPerRequest ?? defaults?.advisor?.maxUsesPerRequest ?? 3;
       }
       export function resolveAdvisorCaching(
         agent: { advisor?: { caching?: { enabled?: boolean; ttl?: "5m" | "1h" } } } | undefined,
         defaults: { advisor?: { caching?: { enabled?: boolean; ttl?: "5m" | "1h" } } } | undefined,
       ): { enabled: boolean; ttl: "5m" | "1h" } {
         return {
           enabled: agent?.advisor?.caching?.enabled ?? defaults?.advisor?.caching?.enabled ?? true,
           ttl: agent?.advisor?.caching?.ttl ?? defaults?.advisor?.caching?.ttl ?? "5m",
         };
       }
       ```
    2. Doc-comment cites RESEARCH.md §3 — same fall-through pattern as `resolveRuntime`.
  </action>
  <verify>`npm run build` clean.</verify>
</task>
