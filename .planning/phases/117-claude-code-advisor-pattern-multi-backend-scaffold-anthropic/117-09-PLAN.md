# Plan 117-09: Discord visibility — 💭 reaction + footer (level-aware seam for 117-11)

## Goal

Consume the `advisor:invoked` / `advisor:resulted` events emitted by `session-adapter.ts` (Plan 117-04) in the Discord bridge: add a 💭 reaction on the triggering user message; append a footer to the agent's response. The mutation is applied at a SINGLE point (bridge.ts:739) so all three downstream delivery exits inherit the augmented `response`. Visibility is in-band — NO new threads are spawned.

This plan lays the level-aware seam (a `getLevel(channelId)` lookup that today defaults to `"normal"`). Plan 117-11 attaches the `VerboseState` instance and wires the `"verbose"` branch to actually show advisor advice inline.

       ```typescript
       import type { AdvisorInvokedEvent, AdvisorResultedEvent } from "../advisor/types.js";

       // Plan 117-09 — advisor visibility (RESEARCH §2 Gate 3, §4.5).
       let didConsultAdvisor = false;
       let lastAdvisorResult:
         | { kind: AdvisorResultedEvent["kind"]; text?: string; errorCode?: string }
         | null = null;

       const onInvoked = (ev: AdvisorInvokedEvent) => {
         if (ev.agent !== sessionName) return;
         // (turnId correlation: ev.turnId is the session-adapter-assigned turn id; correlate
         //  to this Discord-message turn by matching the turn-scoped EventEmitter's audience.
         //  In the simplest topology — one in-flight dispatch per channel — we accept all events
         //  while dispatchStream is in flight. The listener is registered/unregistered around it,
         //  scoping naturally to this turn.)
         didConsultAdvisor = true;
         void addReaction(message, "💭");
       };
       const onResulted = (ev: AdvisorResultedEvent) => {
         if (ev.agent !== sessionName) return;
         lastAdvisorResult = { kind: ev.kind, text: ev.text, errorCode: ev.errorCode };
       };
       this.sessionManager.advisorEvents.on("advisor:invoked", onInvoked);
       this.sessionManager.advisorEvents.on("advisor:resulted", onResulted);
       try {
         response = await this.turnDispatcher.dispatchStream(/* ...existing args... */);
       } finally {
         this.sessionManager.advisorEvents.off("advisor:invoked", onInvoked);
         this.sessionManager.advisorEvents.off("advisor:resulted", onResulted);
       }
       ```
    3. The listener is scoped to this message-handler call (register-around-dispatch); naturally garbage-collected at turn end (RESEARCH §13.12 A13).
