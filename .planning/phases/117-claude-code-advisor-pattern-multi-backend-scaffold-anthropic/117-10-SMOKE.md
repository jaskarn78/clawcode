### 3. Capability awareness check (validates Plan 117-08)

In Discord channel `1491623782807244880`, send:

### 4. Native backend visibility (default path — advisor_result variant)

Ask a hard question that should trigger the advisor:

> Design a worker pool with graceful shutdown — consult your advisor
> first.

Expected:

- 💭 reaction lands on the user message that triggered the turn
  (visible side effect; pre-delivery gate).
- Assistant response carries the footer
  `*— consulted advisor (Opus) before responding*`.
- NO new Discord thread is spawned (visibility is in-band; advisor
  consultations and `subagent-thread` are independent systems).

Tail the daemon logs:

```bash
journalctl -u clawcode -f | grep -E "advisor|forkSession"
```

Expected:

- An `advisor` `server_tool_use` block appears in the SDK stream
  (native path).
- NO `forkSession` log line — the native backend does NOT fork.
- `manager/advisor-budget.db` records exactly one new call for
  `test-agent` (verify via `sqlite3 ~/.clawcode/manager/advisor-budget.db
  'select * from calls where agent_name=test-agent order by ts desc limit 1'`).

### 6. Verbose toggle (validates Plan 117-11)

a. In channel `1491623782807244880`:

```
/clawcode-verbose on
```

Expected ephemeral reply: `verbose: verbose for this channel`.

b. Re-ask a hard question that should trigger the advisor (e.g.,
   "review my locking strategy for the worker pool above"). Expected:
   the response carries the fenced advice block in place of the plain
   footer:

   ```

   ```

   current level (`verbose`) + ISO timestamp.

d. Run `/clawcode-verbose off`. Expected ephemeral reply: `verbose:
   normal for this channel`.

e. Re-trigger the advisor. Expected: return to plain footer-only display.

### 7. Fork backend rollback test

Edit `clawcode.yaml`:

```yaml
agents:
  - name: test-agent
    # ... existing config ...
    advisor:
      backend: fork    # operator rollback lever
```

Then:

```bash
clawcode reload
```

Expected (no daemon restart needed; per-agent override cascades):

- Reload completes cleanly. `test-agent` warm-path remains ready.

Re-trigger the advisor with another hard question. Expected:

- A `forkSession` log line appears in `journalctl -u clawcode` (legacy
  fork path is now selected for `test-agent`).
- 💭 reaction + footer still appear (visibility is backend-agnostic;
  Plan 117-09 listeners subscribe to `advisorEvents` regardless of
  backend).
- The footer wording is unchanged (`*— consulted advisor (Opus) before
  responding*`).

After verifying, revert the yaml change and `clawcode reload` again to
return `test-agent` to the default `native` backend.

  the existing message). This is the `subagent-thread` skill working
