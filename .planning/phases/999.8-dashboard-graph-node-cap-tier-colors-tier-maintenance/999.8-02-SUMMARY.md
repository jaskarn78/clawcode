## Accomplishments

- `nodeClr` extracted to `src/dashboard/static/graph-color.js` as a pure ESM export with the canonical D-COLOR-01 palette: hot=`#e06c75`, warm=`#7f6df2`, cold=`#5a8db8` (NEW), orphan=`#444`. The orphan check (`linkCount===0`) dominates over `tier`, matching the prior behaviour.
- `graph.html`'s inline `<script>` converted to `<script type="module">` with `import { nodeClr } from "./graph-color.js"`. The duplicate inline `nodeClr` definition is gone (`grep -c "function nodeClr" graph.html === 0`).
- Top-right legend overlay added inside `.graph-area`, between `#stats` and `#loading`. Uses native `<details open>` / `<summary>` (matches Forces panel at graph.html:362). 4 `.legend-row` blocks, each with a swatch (`.sw`), tier label, and live count placeholder (`.ct`).
- Legend CSS uses `position:absolute; top:12px; right:12px; width:150px; pointer-events:auto; z-index:5` — sits above the SVG without intercepting clicks/drags on the bulk of the canvas (corner footprint is ~150x100px).
- Live counts populated inside `render(data)` immediately after the existing `linkCount` assignment loop. Classification ladder: `linkCount===0 → orphan`, then `tier === "hot" → hot`, `tier === "cold" → cold`, else `warm`. Updates every agent switch.
- Backend byte-identical: `git diff master~2..master -- src/dashboard/server.ts src/manager/daemon.ts` is empty.

## Route A vs Route B

**Chose Route A — `<script type="module">` + ESM import.** The plan permitted either path with Route A as preferred. Selecting it gave us:

- Zero shim code in `graph-color.js` (no `globalThis.nodeClr = nodeClr`).
- One-directional call graph: `graph.html` imports from `graph-color.js`; nothing imports back.
- The existing inline `<script>` was self-contained — no external consumer of its definitions, so converting to `module` was safe.
- D3 v7 is loaded from CDN as a global (`<script src="https://d3js.org/d3.v7.min.js">`) BEFORE the module script, so `d3.select`, `d3.zoom`, etc. remain accessible from inside the module — no rewiring required.

Route B (globalThis shim) would have been needed only if we'd had a separate non-module consumer of `nodeClr` outside `graph.html`. We don't, so it would have been pure overhead.

## Decisions Made

- **Route A (module import) over Route B (globalThis shim).** No external consumer of `nodeClr` exists outside `graph.html`, so the shim would be pure overhead. Cleaner one-directional dependency: `graph.html` → `graph-color.js`.
- **Static-HTML regex tests for the legend** — JSDOM/cheerio not needed; regex coverage of the markup contract is sufficient and keeps the test under 200ms.
- **Counts inside render(data) (not a separate observer).** Aligns with the answer to RESEARCH Open Question 3 — `render` already runs on every agent switch and any future SSE re-render, so counts stay in lockstep with the displayed nodes for free.
- **`<details open>` (open by default).** Matches the existing Forces-panel idiom at line 362 and gives operators an immediate read of the tier distribution without a click. Users can collapse if it gets in the way.
