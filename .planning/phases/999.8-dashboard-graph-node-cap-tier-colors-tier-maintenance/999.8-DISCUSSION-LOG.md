## Issue 3: Tier maintenance investigation finding

| Finding | Verdict |
|---------|---------|
| Tier-maintenance code exists at `src/heartbeat/checks/tier-maintenance.ts` | ✓ |
| Production logs show ZERO tier-maintenance ticks in 7 days | 🚨 |
| `dist/heartbeat/` directory does NOT exist on prod | 🚨 |
| `tsup.config.ts` bundles only `cli/index.ts` with `splitting: false` | Root cause |
| `src/heartbeat/discovery.ts` does dynamic disk-based imports of `dist/heartbeat/checks/*.js` | Discovery returns 0 modules |
| Auto-linker still works because Phase 1.6 made it eager-on-save (NOT heartbeat-driven) | ✓ |
| Affected: tier-maintenance, consolidation, attachment-cleanup, thread-idle, task-retention, trace-retention, fs-probe, mcp-reconnect, context-fill, inbox(heartbeat-path) | 10 broken checks |

**Root cause:** tsup bundles into a single file but discovery expects per-file disk layout. Two incompatible designs.

**Fix approach:** Static import registry. Replace dynamic discovery with compile-time `check-registry.ts` that imports each module statically and exports them as an array. `tsup` bundles the registry imports automatically.
