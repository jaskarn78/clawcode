## Decisions Made

- **Handler module extraction (vs inlined daemon case).** The plan body had the full daemon-case body inline; the auto-ingest-handler.ts precedent (Plan 02 T03) extracted the body to a separate module for direct testability without standing up the daemon harness. Followed the precedent: created `src/manager/set-doc-priority-handler.ts` with three exports + DI types, and the daemon case became a 30-line param-validate + delegate. Acceptance criterion `grep -c "phase999.43-priority" src/manager/daemon.ts >= 2` was honored by adding two comment references in the daemon (the structural intent — that runtime logs use that tag — is met because the handler emits the logs and the daemon invokes the handler).

- **listDocumentSources() new public method.** The plan's reclassify-docs code sample used `docStore.listSources()`, but that method iterates `document_chunks` (Phase 101 wave). My test 5 upsert'd documents rows directly (no chunks) and `listSources()` returned `[]` → `updated:0`. Added a new public method `listDocumentSources()` that iterates the `documents` table (uses the existing private `listAllDocumentsSources` statement from Plan 03 T02). Semantically correct for reclassify — it covers every ingested doc including pre-Plan-01-backfill placeholders. Documented in code + tracked as Rule 1 (bug fix) deviation.

- **Refusal paths still write an audit entry.** The plan acceptance test 2 says "Audit log records the refusal attempt." Implemented: `refused-escalation` and `refused-isolation` are first-class `outcome` values in the audit row alongside `applied` / `applied-bulk`. Operator's grep on `outcome=refused-escalation` surfaces every agent self-escalation attempt without sifting error logs.

- **Emoji branch is purely additive.** The plan said explicitly: "do NOT `return` here — let the existing reaction-forwarding pipeline ALSO fire (the agent may want to see '🔴 added' too). Both surfaces are additive." Honored verbatim: the new branch runs at the top of `handleReaction` after partial-fetch resolution, then control falls through to the original forward-to-agent block. The IPC call is fire-and-forget (errors logged, not rethrown).

- **CLI bypasses agent sandbox by hard-coding who="operator".** Both subcommand action handlers in `rag.ts` set `who: "operator"` in the IPC params. The reclassify-docs daemon handler additionally PINS `who: "operator"` regardless of what the caller sent (defense-in-depth — even a stray IPC client cannot smuggle a who="agent" bulk reclassification through).

- **Layer-1 + Layer-2 sandbox parallel structure.** The MCP tool's `z.enum(["medium", "low"])` is the primary gate the agent's model interacts with — `"high"` never reaches the daemon from agent calls. But the daemon handler ALSO validates `who === "agent" && level === "high"` → refusal. This is the textbook D-08 defense-in-depth pattern: two independent checks, either of which is sufficient.
