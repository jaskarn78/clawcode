## Decisions Made

- **Empty-chunks → no documents row.** The Phase 101 engine can return empty text (e.g., a PDF whose OCR yields nothing). When `chunks.length === 0`, the handler returns `{ skipped: true, reason: 'no chunks produced (empty text)' }` and does NOT call `docStore.upsertDocumentRow`. Rationale: a provenance row pointing at zero chunks would be retrieved-by-LEFT-JOIN-then-find-nothing in Plan 03. Plan didn't specify; safer choice (advisor recommendation).
- **Try/catch scope.** The catch encompasses `readFile`, `ingestDocumentEngine`, `chunkText/chunkPdf`, `embedder.embedV2` loop, AND `docStore.ingest`. Anything that can throw on the happy path is caught and returned as `{ ok: false, error }` — never propagated to the bridge caller. Cross-ingest failure is handled separately (nested try/catch, logged as warn, non-fatal).
- **Handler extraction at T03 not T01.** T01's plan acceptance gates required grep matches IN `daemon.ts` (`classifyAttachment`, `phase999.43-autoingest >= 2`, `high: 1.5 >= 2`, `low: 0.7 >= 1`, `low: 0.5 >= 1`). Inlining the body in T01 satisfied those gates. T03 then split the body into `auto-ingest-handler.ts` so the integration test could target a pure helper — Plan §3 explicitly anticipates this ("IF Task 1 already inlined the body... this Task 3 SPLITS it out"). T03 commit removes the inline body and the now-unused top-level `classifyAttachment` import from daemon.ts.
- **Dynamic import in daemon case.** The daemon case calls `await import("./auto-ingest-handler.js")` to fetch the handler. Two reasons: (a) it matches the pattern shown in Plan §3, and (b) lazy-load keeps daemon boot lean — handler module only loads when the first auto-ingest IPC arrives.
- **D-01 multipliers duplicated in handler (not imported from `store.ts`).** `CONTENT_PRIORITY_WEIGHTS` exists in `src/documents/store.ts` (Plan 01 T02), but the handler defines its own `CONTENT_PRIORITY_WEIGHTS` + `AGENT_PRIORITY_WEIGHTS` constants with literal numbers in source. Rationale: operator grep on `high: 1.5` / `low: 0.7` / `low: 0.5` should hit the handler file directly (verbatim D-01 LOCKED table), not an import indirection. The values are asserted equal by the integration test (Test 3 row write).
- **Hot-reload via `configs.find`.** The daemon's `configs` array is the LIVE reference — `setAllAgentConfigs(resolvedAgents)` at line ~5583 mutates this array in place when `clawcode reload` runs. Reading `configs.find((c) => c.name === a)` at IPC-receive time honors Plan 01 SUMMARY's hot-reload promise.
