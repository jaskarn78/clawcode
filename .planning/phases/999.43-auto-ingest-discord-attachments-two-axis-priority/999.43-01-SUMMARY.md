## Decisions Made

- **D-04 placeholder agent_name** — Chose literal `"_unknown"` (leading underscore) for pre-999.43 backfill placeholders. Underscore-prefix follows the existing Phase 90 unparseable-memory-tag convention; future operator-driven retroactive priority assignment can find these via `WHERE agent_name = '_unknown'`.
- **Per-document `agent_priority_weight_at_ingest` is snapshot-only** — D-04 stores the agent weight at ingest time but Plan 03 reads the LIVE per-agent weight at query time (RELOADABLE semantics). The stored value is informational / audit-trail. Resolver code that wants to detect "agent priority changed since ingest" can diff against `live ingestionPriority`.
- **setDocumentPriority `who` parameter is currently unused** — Kept in the signature for future D-08 sandbox enforcement at this layer. Plan 04 will enforce the rule "agents cannot escalate own doc beyond MEDIUM" at the IPC handler layer, not in this low-level write surface, but the signature is already plumbed for inversion-of-control.
- **ResolvedAgentConfig fields typed as OPTIONAL not REQUIRED** — Phase 127 streamStallTimeoutMs + Phase 115 excludeDynamicSections precedent. Production loader always populates them, but the type-level `?` saves ~30 test fixture rewrites (cost/benefit favors the precedent).
- **Backfill `source_kind = 'manual_pre_999_43'`** — Distinguishes legacy chunks from Phase 101's manual `ingest_document` MCP tool surface (which will get `source_kind = 'manual_ingest_document'` in Plan 02). Future operator can `DELETE FROM documents WHERE source_kind = 'manual_pre_999_43'` to wipe the placeholders once they've manually re-tagged provenance.
