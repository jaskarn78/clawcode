<objective>
P0 fix for sub-bug C — eliminate the cross-channel file leak that put fin-acquisition's Schwab AIP markdown into content-creator's primary channel. Compliance-relevant for Finmentum (could be PII routed to wrong audience in different scenario).

Root cause (confirmed via code inspection — see PHASE.md sub-bug C + advisor reconciliation):
- The MCP `clawcode_share_file` tool description says "pass your own name" — the LLM in a subagent session may pass either its own session name (e.g., `fin-acquisition-sub-OV9rkf`) OR a confused name (e.g., `finmentum-content-creator` from shared-workspace soul/identity drift).
- Daemon IPC at `daemon.ts:6661 (case "share-file")` resolves: `channelId = channelIdParam ?? agentConfig.channels[0]`.
- For subagent invocations, this falls back to whatever channel the LOOKED-UP agent has — completely missing the actual subagent thread the work is happening in.

Fix surface (per D-09, D-10): the daemon IPC handler is the right place. Resolve via thread registry FIRST: if `agentName` matches a binding's `sessionName` (subagent session), use `binding.threadId`. Falls back to existing logic only when no binding exists (regular agent invocations).

Defense-in-depth: clarify the subagent's threadContext to instruct the LLM to pass `agent: <its own session name>` — so the registry lookup succeeds.

Output:
- New `getBindingForSession(registry, sessionName)` helper in thread-registry.ts.
- Updated daemon IPC handler resolving channelId via thread binding first.
- Strengthened subagent threadContext prompt instruction.
- Integration test fixture covering the shared-workspace invariant (D-11).
- DEFERRED-WORKSPACE-LOOKUPS.md catalogue of OTHER workspace-keyed paths NOT fixed here (per D-02).
