## What Was Built

### 1. Sub-bug A — typing indicator (CODE FIX, ships immediately)

`src/discord/subagent-typing-loop.ts` (new, 68 lines): pure helper exporting `startTypingLoop(thread, log) → { stop }`. Mirrors the `bridge.ts:357-377 fireTypingIndicator` + `bridge.ts:606-611 8-second heartbeat` reference pattern. Wires sendTyping eagerly at t=0 and re-fires every 8 seconds (D-05 cadence pin via exported `TYPING_REFRESH_MS = 8000`). Rejections (archived threads, rate limits) caught + log.debug'd. `src/discord/subagent-thread-spawner.ts` `postInitialMessage`: invokes `startTypingLoop` after `placeholder.send`, declared outside the try{} so the new finally{} block can stop the indicator BEFORE the autoRelay/autoArchive cleanup chain (operator should not see typing during cleanup).

### Minor implementation choices (within Claude's discretion per CONTEXT)

1. **D-12 example used `binding.lastActivity` for `idleSeconds` field**, but at the daemon-level callback site `binding` is not in scope (lives inside the sweep tick). Logged only `threadId + source + quiescenceMinutes` at the callback; Plan 02 will pull `idleSeconds` from the binding shape if needed. This is observably equivalent — the source tag alone identifies the firing path, which is the load-bearing field for D-12.

2. **D-04 example placed sub-bug A anchor comment 3× in the spawner**; acceptance criteria pinned exactly 1. Resolved by consolidating to a single anchor at the `startTypingLoop` invocation site (line 712); the import + handle declaration use shorter cross-references that don't include the full anchor phrase.

3. **Test assertion for finally-block cleanup in 999.36-A2** uses fake-timer advance + sendTyping call-count delta to prove the interval was cleared, since the test surface can't peek the `setInterval` handle directly. Equivalent to the operator-visible failure mode (continued typing after cleanup).
