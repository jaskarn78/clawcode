<objective>
Build the session manager, daemon process, and IPC communication layer that together form the running agent lifecycle manager.

Purpose: This is the core runtime. The session manager handles start/stop/restart/crash-recovery for individual agents. The daemon wraps it in a long-running process with signal handling and Unix socket IPC. The IPC client enables CLI commands (Plan 03) to communicate with the running daemon.

Output: SessionManager class, daemon entry point, IPC server and client, with tests using MockSessionAdapter for deterministic lifecycle testing.
</objective>
