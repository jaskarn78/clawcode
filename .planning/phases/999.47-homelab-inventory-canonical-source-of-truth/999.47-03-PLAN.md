<objective>
Author the polling + drift-quarantine + verify scripts that drive the homelab repo's Live State, with synthetic fixtures + vitest tests that let the script logic be bisect-compiled and CI-tested in workspace-coding. Then ship the scripts to `/home/clawcode/homelab/scripts/` on clawdy under operator control.

Purpose:
- SC-2 (full coverage): polling sources land here. The hourly scheduler from Plan 02 starts emitting non-zero counts the moment refresh.sh exists on clawdy.
- SC-3 (drift quarantine): DRIFT.md is the staging surface; INVENTORY.md is never auto-modified outside the managed Live State blocks. Stale-down marks items `unreachable since <ts>` rather than deleting.
- SC-8 (verify.sh): a reachability sanity-check that exits 0 against a healthy inventory and non-zero on first unreachable item.

Output:
- `scripts/homelab/{refresh.sh,verify.sh,lib/*.sh}` authored in workspace-coding with vitest-driven tests against synthetic fixtures.
- `scripts/homelab/deploy-to-clawdy.sh` operator-gated deploy.
- After deploy, the heartbeat tick from Plan 02 starts producing real `phase999.47-homelab-refresh` log lines on clawdy.

Architectural justification (per CONTEXT.md "Claude's Discretion"):
- **Language: bash** for refresh.sh and verify.sh — pure polling with no daemon IPC chatter (the daemon-IPC re-index is owned by Plan 02's `clawcode homelab reindex` CLI which the bash script invokes as a subprocess). Bash keeps the scripts small, auditable, and runnable on clawdy without a TypeScript toolchain.
- **Survey order**: Tailscale (cheapest, no SSH) → Unraid (SSH virsh + docker in one session) → Cloudflare tunnels (HTTP API) → 1Password (`op` CLI). Failures isolated per source — one source failing logs a warning but the others still update.
- **Atomicity**: every file write goes through `lib/common.sh` atomic temp+rename (Phase 96 pattern).
</objective>
