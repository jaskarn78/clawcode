## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| refresh.sh (untrusted disk) → daemon (parsed JSON) | `.refresh-last.json` is parsed via Zod schema; malformed input is logged as `reason: "refresh-output-malformed"` and never crashes the tick. |
| CLI process → daemon IPC | `sendIpcRequest` is the existing Unix-socket transport; no new auth surface. |
| Heartbeat tick → execa spawn of refresh.sh | bash script runs as the `clawcode` user (the daemon's user); same blast radius as the daemon itself. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-99947-06 | Tampering | `.refresh-last.json` write-then-read race | mitigate | Plan 03 will write the file via atomic temp+rename (Phase 96 pattern); this plan's reader uses safeParse and tolerates partial-write states via Test 3 (malformed-JSON path). |
| T-99947-07 | Denial of service | Overlapping refresh ticks if a prior tick stalled | mitigate | `isRunning` in-memory mutex (Test 7) caps concurrency at 1; skipped tick still logs `phase999.47-homelab-refresh` so operators never lose telemetry. |
| T-99947-08 | Repudiation | Silent refresh failures | mitigate | D-04c "noisy commit, never silent": every tick emits exactly one `phase999.47-homelab-refresh` log line, with `ok: false` + `reason: <code>` on failure. 3 consecutive failures additionally fires `phase999.47-homelab-fleet-alert`. |
| T-99947-09 | Information disclosure | Reindex pulling secrets into `memory_chunks` | mitigate | The repo's ACCESS.md contains only `op://` references (enforced by Plan 01's grep gate); no literal secrets reach the embedding/chunk store. |
| T-99947-SC | Tampering | npm/pip/cargo installs | accept | This plan adds no new package dependencies — uses existing `zod` (v4) + `execa` + `pino` + `commander` already in the lockfile. Package legitimacy gate not triggered. |
