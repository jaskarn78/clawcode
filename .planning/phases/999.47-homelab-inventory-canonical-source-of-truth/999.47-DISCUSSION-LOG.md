## Refresh cadence & drift policy

| Option | Description | Selected |
|--------|-------------|----------|
| Hourly cron + DRIFT.md quarantine + operator promotes | Discovered items NOT in INVENTORY appended to DRIFT.md; operator reviews & promotes. Canonical-source-of-truth invariant preserved. | ✓ |
| Hourly cron + auto-promote unknown items with TODO markers | Auto-add to INVENTORY with [NEEDS REVIEW]. Risk: pollution from test containers. | |
| Daily cron + Discord alert on drift, no auto-write | Webhook alert; operator triages manually. Slower drift detection. | |
| On-demand only (no cron) | Operator runs refresh.sh manually. Live State will be stale most of the time. | |

**User's choice:** Hourly cron + DRIFT.md quarantine + operator promotes.
**Notes:** Quarantine preserves operator-intent gate on INVENTORY.md. Stale-down policy = mark as `unreachable since <ts>`, don't delete. Failure mode = noisy commit, fleet-alert webhook after 3 consecutive refresh failures.
