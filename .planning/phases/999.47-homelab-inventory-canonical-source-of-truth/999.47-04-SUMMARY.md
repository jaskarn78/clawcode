# Phase 999.47 Plan 04: MEMORY.md pointer fleet-wide — Summary

Wired the homelab discoverability pointer (D-03 / SC-5) into both MEMORY.md
surfaces — the forward path (new-agent `clawcode agent-create`) and the
backfill (operator-gated bash seeder for the existing fleet). **Zero new
package dependencies. Zero IPC additions. Zero daemon restarts.** SC-5
partially closed (5/10 agents seeded; 5 finmentum agents deferred to a
follow-up session by operator decision 2026-05-15).

## What landed

### Task 1 — forward path (commit `5e67b05`)

Added two exported constants to `src/config/defaults.ts`:

- `HOMELAB_POINTER_LINE` — the verbatim pointer string (em-dash and all).
  Single-sourced — the bash seeder reads byte-for-byte the same string,
  so the two surfaces cannot drift.
- `DEFAULT_MEMORY_TEMPLATE` — `# Memory\n\n${HOMELAB_POINTER_LINE}\n`.
  Matches the byte layout the seeder writes on the create-missing path.

Extended `createWorkspace` in `src/agent/workspace.ts` to initialize
MEMORY.md alongside SOUL.md and IDENTITY.md (Rule 2 — the function did
not previously write MEMORY.md at all). Three behaviors:

| MEMORY.md state                          | Action                                         |
| ---------------------------------------- | ---------------------------------------------- |
| missing                                  | write `DEFAULT_MEMORY_TEMPLATE` verbatim       |
| present, pointer found (`String.includes`) | no-op (preserve operator / dreaming edits) |
| present, pointer missing                 | idempotent append (`\n` separator + pointer)   |

5 new vitest cases pin the contract:

- pointer present in default template (verbatim string assertion)
- template exact-match regression guard
- markdown list-item format (leading `- ` prefix)
- idempotency on operator-edited content (preserves arbitrary body)
- append-to-legacy + no-duplicate-on-rerun

### Task 2 — backfill seeder (commit `6d8bf41`)

`scripts/homelab/seed-memory-pointer.sh`:

- Discovers agent directories via `find ~/.clawcode/agents -maxdepth 1
  -mindepth 1 -type d` (override via `--agents-dir DIR` for tests).
- Per-agent decision tree mirrors the in-process workspace branch.
- Atomic temp+rename writes only (`<file>.tmp.$$` → `mv -f`). Never
  in-place stream edits — no truncate-then-rewrite race window.
- Idempotency via `grep -Fxq -e "$POINTER_LINE" -- "$file"`. The `-e`
  + `--` matters: without them, grep parses the leading `- ` as an
  option flag and the idempotency check is broken (Rule 1 fix below).
- Structured per-agent log lines tagged `phase999.47-homelab-seed-pointer`
  with JSON payload. Final summary line same prefix.
- `--dry-run` produces zero filesystem changes; prints the action plan.
- Exits 1 on any per-agent error with a list of failed agents on stderr.
- Static-grep gate clean: zero `systemctl|service.*restart|killall|pkill`
  references. Ramy-safe by construction.

12 vitest cases cover all 7 plan-mandated scenarios plus 5 architectural
guards (static-grep gate, atomic-write pattern, byte-equality with
`HOMELAB_POINTER_LINE`, missing-agents-dir defensive branch, no
`.tmp.$$` leftovers).

## Deviations from plan

### Rule 2 — added MEMORY.md initialization to workspace.ts

The plan's T01 assumed `src/agent/workspace.ts` already wrote MEMORY.md
and instructed "append the pointer line to the default MEMORY.md
content". In reality, `createWorkspace` only wrote SOUL.md and
IDENTITY.md — there was no MEMORY.md content to append to. SC-5 was
unreachable via the forward path without adding initialization. Caught
by the advisor before any code was written.

### Rule 2 — bonus idempotent append branch

The plan only mandated default-write-on-missing. I additionally added
an idempotent append-to-existing branch (mirrors the bash seeder's
logic, in-process). Closes the SC-5 gap where an agent has a
pre-existing MEMORY.md and the bash seeder hasn't yet run against
that workspace.

### Architectural divergence from acceptance-criterion literal text

The plan's T01 acceptance criterion said
`grep -F "Homelab inventory](/home/clawcode/homelab/INVENTORY.md)" src/agent/workspace.ts`
should return exactly 1 match. It now returns **0** — the verbatim
string lives in `src/config/defaults.ts` (alongside `DEFAULT_SOUL` per
CONVENTIONS.md) and `workspace.ts` imports `HOMELAB_POINTER_LINE`.
The spirit of the criterion (single occurrence of the verbatim line
in the codebase) is preserved: `grep -rF "Homelab inventory](/home/clawcode/homelab/INVENTORY.md)" src/` returns exactly 1 match in
`src/config/defaults.ts:62`. Operator ratified this placement during
T3 handoff (no pushback raised in-turn before the live seed).

### Rule 1 — grep leading-dash idempotency bug

Manual smoke test during T02 caught it: `grep -Fxq "$POINTER_LINE"`
with a pattern starting with `- ` causes grep to parse it as options
(`invalid option -- ' '`). The idempotency check silently failed and
every re-run duplicated the pointer line. Fixed by passing the
pattern via `grep -Fxq -e "$POINTER_LINE" -- "$memory_file"`. This is
the single most operationally important issue found in this plan —
without the fix, T3's idempotency proof step would have produced
visible duplicates on the live fleet.

### Rule 3 — operator T3 path bifurcation (in-flight discovery)

The seeder's `find ~/.clawcode/agents -maxdepth 1 -mindepth 1 -type d`
did not follow symlinks in the live layout, blocking the staging-dir
approach the operator wanted for the explicit non-finmentum allowlist.
Operator opted to inline the seed logic directly in-session, preserving
the same `grep -Fxq -e` idempotency check and the same
`phase999.47-homelab-seed-pointer` structured log tag. Followup
candidate: add a `-L` flag (or `--follow-symlinks`) and an explicit
`--include`/`--exclude` allowlist surface to the seeder so the operator
doesn't have to inline next time.
