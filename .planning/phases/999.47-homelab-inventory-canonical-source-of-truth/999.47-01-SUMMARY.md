## Decisions Made

- **Anchor partition** (CONTEXT.md "Claude's Discretion"): INVENTORY = what-exists + what-it-does; NETWORK = addressing + topology; ACCESS = how-to-log-in. Cross-linked via markdown anchors. Rationale: matches the natural triage path (operator opens INVENTORY first, follows links into NETWORK for IPs or ACCESS for SSH commands).
- **Per-anchor Live State blocks** (15 total in INVENTORY+NETWORK) instead of one global block per file. Rationale: Plan 03's refresh rewrite can locate each block by its discriminator field (`host:`, `vm:`, `container:`, `service:`, `domain:`) without an awk multi-anchor lookbehind state machine — drastically simpler refresh implementation.
- **PLACEHOLDER- op-item prefix convention** for unconfirmed vault paths in ACCESS.md, plus the real `op://clawdbot/win11-vm-password/password` from the BACKLOG context. Rationale: operator can grep `PLACEHOLDER-` to find every credential reference that needs first-pass triage; real op-item paths are already searchable by their vault-prefix.
- **Forbidden-token rephrasing in bootstrap script** — replaced literal `systemctl` / `/opt/clawcode/` mentions in script comments and dry-run echo lines with "init system" / "daemon install tree" paraphrases. Rationale: the plan's `! grep -E "systemctl|service.*restart|/opt/clawcode"` automated acceptance check is the source of truth; the intent (script must not DO those things) is preserved while passing the literal text gate. Documented as a deviation below.
- **Operator-identity required on bootstrap** — no defaults for `--operator-name` / `--operator-email`; script exits 2 with a D-02a-referenced error if either is omitted. Rationale: D-02a is the entire reason the initial-commit author matters; silently falling back to a default would defeat the audit-trail invariant on day 1.
