**Extend src/dashboard/types.ts** — add types:
- `ScheduleData`: `{ name: string; agentName: string; cron: string; enabled: boolean; lastRun: number | null; lastStatus: string; lastError: string | null; nextRun: number | null }`
- `HealthData`: `{ agents: Record<string, { checks: Record<string, { status: string; message: string; lastChecked: string }>; overall: string; zone?: string; fillPercentage?: number }> }`
- `DeliveryQueueData`: `{ stats: { pending: number; inFlight: number; failed: number; delivered: number; totalEnqueued: number }; failed: Array<{ id: string; agentName: string; content: string; lastError: string | null; createdAt: string; attempts: number }> }`
- `MemoryStatsData`: `{ agents: Record<string, { entryCount: number; episodeCount: number; tierDistribution: Record<string, number> }> }` (aggregated from per-agent queries)

**Extend src/dashboard/sse.ts** — add additional SSE event broadcasts:
- In `pollAndBroadcast()`, add parallel IPC calls for:
  - `sendIpcRequest(socketPath, "schedules", {})` -> broadcast as `event: schedules`
  - `sendIpcRequest(socketPath, "heartbeat-status", {})` -> broadcast as `event: health`
  - `sendIpcRequest(socketPath, "delivery-queue-status", {})` -> broadcast as `event: delivery-queue`
- Add a slower poll (every 15 seconds, separate interval) for memory stats:
  - For each agent in the latest status result, call `sendIpcRequest(socketPath, "memory-list", { agent: name, limit: 1 })` to get entry count, and `sendIpcRequest(socketPath, "episode-list", { agent: name, count: true })` for episode count
  - Aggregate into `MemoryStatsData` and broadcast as `event: memory-stats`
  - Catch per-agent errors (agent may not be running) and skip that agent
- All new IPC calls wrapped in try/catch — if one fails, others still broadcast

**Extend src/dashboard/server.ts** — add one-shot API endpoints:
- `GET /api/schedules` -> returns schedules IPC result as JSON
- `GET /api/health` -> returns heartbeat-status IPC result as JSON
- `GET /api/delivery-queue` -> returns delivery-queue-status IPC result as JSON

`renderMemoryPanel(data)` for `#memory-content`:
- For each agent: a compact card showing:
  - Agent name
  - Entry count (bold number)
  - Episode count
  - Tier distribution as a tiny stacked bar (hot/warm/cold colored segments)
