    - initialize cache-and-replay (Pitfall 1): child sees exactly ONE initialize on stdin even with N concurrent agent connects
    - crash → in-flight error fanout: in-flight calls resolve with `{error: {code: BrokerErrorCode.PoolChildCrashed, ...}}` when child emits 'exit'
    - notification pass-through (drop on no-id): JSON-RPC notification from child is logged, optionally fanned out to all routes (per test's recorded expectation in 108-00)

  <action>
    Implement `PooledChild` class per <interfaces>. File 200–350 LOC realistic.

    Key implementation pieces (RESEARCH.md §3 patterns 2–3):

    1. **Lazy spawn:** First call to `dispatch()` or `serveInitialize()` triggers `ensureSpawned()`. Inside:
       - Call `spawnFn("npx", ["-y", "@takescake/1password-mcp@latest"], { env: { ...process.env, OP_SERVICE_ACCOUNT_TOKEN: this.opts.tokenLiteral } })`.
       - Wire `readline.createInterface({ input: child.stdout })` for line-framed reads.
       - Hook `child.on('exit', this.onExit)` and `child.on('error', this.onError)`.
       - Initialize state: `inflight = new Map<number, AgentRoute>()`, `nextPoolId = 1`, `cachedInitializeResult: JsonRpcResponse | null = null`.

    2. **Initialize cache-and-replay (Pattern 3):**
       - `serveInitialize(route, msg)`: if `cachedInitializeResult` is null, this is the first ever initialize — forward to child (with id rewrite), wait for response, cache it, return synthesized response with original `msg.id`. If null AND another initialize is mid-flight, await the in-flight promise (single-flight). If non-null, synthesize response from cache + `msg.id` immediately (no child round-trip).
       - Use a `initializeOnce: Promise<JsonRpcResponse> | null` field for single-flight semantics.

    3. **Id rewriter (Pattern 2):**
       - `dispatch(route, msg)`: assert msg.method !== "initialize" (caller routes initialize through `serveInitialize`).
       - `poolId = nextPoolId++; inflight.set(poolId, route);`
       - Write `JSON.stringify({...msg, id: poolId}) + "\n"` to `child.stdin`.
       - Return Promise that resolves when matching response arrives.
       - On stdout line: parse JSON; if `id` matches an inflight entry, resolve that promise with `{...response, id: route.agentRequestId}`; if no match (notification or late response), see Pattern 4 below.

    4. **Notification + late-response handling:**
       - Lines without `id`: per 108-00 test recorded expectation (consult test file). If "drop" — log at debug level with structured fields; do not fan out. If broadcast — call an injected `onNotification` handler (broker.ts subscribes; not implemented in this plan).
       - Lines with `id` not in `inflight`: drop with debug log (late response after agent disconnect).

    5. **Crash handler (`onExit`):**
       - Set `this.alive = false; this.child = null;`.
       - For every entry in `inflight`: resolve with `{jsonrpc:"2.0", id: route.agentRequestId, error: {code: BrokerErrorCode.PoolChildCrashed, message: "Pooled MCP child exited unexpectedly"}}`.
       - Clear `inflight` map.
       - Log structured: `{...BrokerLogFields, event: "child_exit", exitCode, signal}`.
       - Do NOT auto-respawn here — that decision lives in `Broker` (108-02) so it can also clear `cachedInitializeResult` (initialize must replay against the new child).

    6. **`drainAndShutdown()` (Pitfall 3):**
       - Set `this.draining = true` so new dispatches reject immediately with `BrokerErrorCode.PoolDrainTimeout`.
       - Wait `drainTimeoutMs` (default 2000) OR until `inflight.size === 0`, whichever first.
       - For any still-inflight: resolve with `BrokerErrorCode.PoolDrainTimeout` error.
       - `child?.kill("SIGTERM")`.
       - Wait for 'exit' event (max 1s extra), then `child?.kill("SIGKILL")` if still alive.
       - Log structured `{event: "drain_complete", drainedCount}`.

    7. **Logging discipline (Phase 104 SEC-07):**
       - Every log call uses `this.opts.log.{level}(BrokerLogFields shape, "msg text")`.
       - NEVER log `this.opts.tokenLiteral`.
       - Pool field always `"1password-mcp:" + this.opts.tokenHash`.

    8. **Spawn fn injection:**
       - Default `opts.spawnFn` to a small wrapper that imports `child_process.spawn` lazily so tests can pass a fake without monkey-patching node builtins.

    Use `node:readline`, `node:child_process` types, no new deps.
