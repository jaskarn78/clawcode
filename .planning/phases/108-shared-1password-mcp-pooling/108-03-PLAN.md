  <action>
    Implement `mcp-broker-shim.ts` mirroring the file shape of `src/cli/commands/browser-mcp.ts` and `search-mcp.ts`.

    **`runShim(opts)` flow:**

    1. **Compute tokenHash:**
       ```typescript
       const tokenHash = crypto.createHash("sha256")
         .update(opts.tokenLiteral)
         .digest("hex")
         .slice(0, 8);
       ```

    2. **Connect socket:**
       ```typescript
       const socket = (opts.connectFn ?? net.createConnection)({ path: opts.socketPath });
       ```
       - On `'error'` before connect: write structured error JSON-RPC response to stdout (so SDK sees something), log to stderr, return exit 75.

    3. **Send handshake (first line on socket):**
       ```typescript
       const handshake = {
         agent: opts.agentName,
         tokenHash,
         tokenLiteral: opts.tokenLiteral,  // broker uses for spawn env; never logs
         turnIdHint: process.env.CLAWCODE_TURN_HINT ?? null,
       };
       socket.write(JSON.stringify(handshake) + "\n");
       ```

    4. **Wire byte-pipe (after handshake):**
       - `opts.stdin.pipe(socket, { end: false })` — agent stdin → broker.
       - `socket.pipe(opts.stdout, { end: false })` — broker → agent stdout.
       - **Important:** This is a transparent passthrough. Do NOT JSON.parse → restringify; preserve byte identity for the broker's id-rewriting layer to do its work.

    5. **Exit conditions:**
       - `socket.on('close', () => resolve(75))` — daemon restart / broker shutdown. Exit 75 (EX_TEMPFAIL) so SDK reconnects.
       - `opts.stdin.on('end', () => { socket.end(); resolve(0); })` — agent's normal stdio close.
       - `socket.on('error', err => { log error with tokenHash, NOT tokenLiteral; resolve(75); })`.
       - SIGTERM handler → end socket gracefully → resolve(0).

    6. **Logging:**
       - Use `opts.log ?? pino({level: process.env.CLAWCODE_LOG_LEVEL ?? "info"}, pino.destination(opts.stderr ?? process.stderr))`.
       - Every log line includes `{component: "mcp-broker-shim", agent: opts.agentName, tokenHash}`.
       - NEVER include `tokenLiteral`.
       - Defensive: add a pino redaction config: `redact: { paths: ['*.tokenLiteral', 'tokenLiteral', 'OP_SERVICE_ACCOUNT_TOKEN'], remove: true }`.

    7. **`registerMcpBrokerShimCommand(program)`** — register subcommand:
       ```typescript
       program
         .command("mcp-broker-shim")
         .description("Per-agent stdio bridge to daemon mcp-broker (Phase 108)")
         .option("--pool <name>", "Pool name", "1password")
         .action(async (options) => {
           const code = await runShim({
             pool: options.pool as "1password",
             agentName: process.env.CLAWCODE_AGENT ?? "unknown",
             tokenLiteral: process.env.OP_SERVICE_ACCOUNT_TOKEN ?? "",
             socketPath: process.env.CLAWCODE_BROKER_SOCKET ?? "/var/run/clawcode/mcp-broker.sock",
           });
           process.exit(code);
         });
       ```
       - Validate `CLAWCODE_AGENT` is non-empty; if missing, log error to stderr and `process.exit(64)` (EX_USAGE).
       - Validate `OP_SERVICE_ACCOUNT_TOKEN` is non-empty (otherwise the broker has nothing to spawn the pool with). On missing, exit 64 with structured error.

    File 100–200 LOC. No new deps; uses `node:net`, `node:crypto`, `pino` (already in stack), `commander` (already in stack).
