export type BrokerOptions = {
  log: import("pino").Logger;             // already child-bound to {subsystem:"mcp-broker"}
  socketPath: string;                     // e.g. "<runtimeDir>/mcp-broker.sock"
  spawnFn?: PooledChildSpawnFn;           // injection for tests
  perAgentMaxConcurrent?: number;         // default 4 (CONTEXT decision 4)
  drainTimeoutMs?: number;                // default 2000
  // Hook for daemon → process tracker integration (108-04)
  onPoolSpawn?: (tokenHash: string, pid: number) => void;
  onPoolExit?: (tokenHash: string, pid: number) => void;
};

export type PoolStatus = {
  tokenHash: string;
  alive: boolean;
  agentRefCount: number;
  inflightCount: number;
  queueDepth: number;
  respawnCount: number;
  childPid: number | null;
};

export class OnePasswordMcpBroker {
  constructor(opts: BrokerOptions);
  start(): Promise<void>;          // start ShimServer
  preDrainNotify(): void;          // reject new connections; existing finish naturally
  shutdown(timeoutMs?: number): Promise<void>;  // drain inflight, SIGTERM all pool children, close socket
  getPoolStatus(): PoolStatus[];   // for heartbeat check
  // Internal — exposed for tests + ShimServer
  acceptConnection(handshake: ShimHandshake): ConnectionHandle;
}

export type ShimHandshake = {
  agent: string;       // CLAWCODE_AGENT
  tokenHash: string;   // sha256(OP_SERVICE_ACCOUNT_TOKEN).slice(0,8)
  tokenLiteral: string;// passed in handshake env-relay only; broker uses for spawn, never logs
  turnIdHint?: string; // optional propagation
};

export type ConnectionHandle = {
  agentId: string;
  tokenHash: string;
  dispatch(msg: JsonRpcRequest, turnId?: string, tool?: string): Promise<JsonRpcResponse>;
  serveInitialize(msg: JsonRpcRequest): Promise<JsonRpcResponse>;
  close(): Promise<void>;          // decrements refcount; triggers drain if last
};

    Implement `OnePasswordMcpBroker` per <interfaces>.

    **Internal state:**
    - `pools: Map<string /*tokenHash*/, { child: PooledChild; refCount: number; respawnCount: number }>`
    - `agentSemaphores: Map<string /*agentId*/, { active: number; queue: Array<() => void> }>`
    - `draining: boolean = false`

    **`acceptConnection(handshake)`** flow (called by ShimServer per shim connection):
    1. If `draining` → throw `BrokerErrorCode.PoolDrainTimeout`-shaped error.
    2. Lookup pool by `handshake.tokenHash`; if missing, construct new `PooledChild({tokenLiteral: handshake.tokenLiteral, tokenHash: handshake.tokenHash, log: this.poolLog(handshake.tokenHash), spawnFn: this.opts.spawnFn, drainTimeoutMs: this.opts.drainTimeoutMs})` — DO NOT log handshake.tokenLiteral. Increment `refCount`.
    3. Subscribe to PooledChild's exit signal — on exit while `refCount > 0` and not draining: invalidate pool entry's child, construct fresh PooledChild (auto-respawn), bump `respawnCount`. Wire `onPoolSpawn`/`onPoolExit` callbacks for daemon integration (108-04 will pass these).
    4. Return ConnectionHandle whose `dispatch` and `serveInitialize` route through the per-agent semaphore (Task: see below) and ultimately call `pool.child.dispatch(...)` / `pool.child.serveInitialize(...)`.
    5. ConnectionHandle.close decrements refCount; if it hits 0 → call `pool.child.drainAndShutdown()` and remove pool from map.

    **Per-agent semaphore (POOL-05):**
    - On dispatch: lookup `agentSemaphores.get(agentId) ?? {active:0, queue:[]}`. If `active < perAgentMaxConcurrent` → increment active, dispatch immediately, on completion decrement + drain queue. Else → push a release callback onto queue; await it; then dispatch.
    - FIFO via plain array shift.
    - perAgentMaxConcurrent default 4 (CONTEXT decision 4).
    - Initialize calls do NOT count against the semaphore (they're synchronous against cache after first agent).
    - When agentRefCount drops to 0, delete the agentSemaphore entry.

    **Audit logging (POOL-06, decision 5):**
    - Log every `dispatch` invocation BEFORE child write, with `{component: "mcp-broker", pool: "1password-mcp:" + tokenHash, agent: agentId, turnId, tool: msg.method}`.
    - Log every dispatch completion at debug level with same fields + `durationMs` and result kind ("ok" / "error").
    - For initialize calls: log "initialize_replay_from_cache" or "initialize_first_round_trip" at info level.

    **Token redaction (Phase 104 SEC-07):**
    - `tokenLiteral` lives in the pool entry only insofar as PooledChild needs it for spawn; broker itself NEVER passes it to its own logger. Helper `this.poolLog(tokenHash)` returns `this.opts.log.child({pool: "1password-mcp:" + tokenHash})`.
    - Add a defensive check in `acceptConnection`: if log mock is provided, the test's token redaction expectation must pass. Lean on PooledChild already redacting; broker just needs to not introduce leaks.

Track `agentTokenStickyMap: Map<agentId, tokenHash>` set on first connection; cleared on agent's last connection close. (This may not have a Wave 0 test; if not, add a defensive check + simple unit test inline.)

    - Wait `min(timeoutMs, 5000)` for all pools' inflight to drain.
    - For each pool: `pool.child.drainAndShutdown()`.
    - Close `ShimServer`.

    - Snapshot of `pools.entries()` projected to `PoolStatus[]` for heartbeat (108-04).

  <action>
    Implement `ShimServer` per <interfaces>.

    **Socket setup:**
    - `node:net.createServer()` listening on `opts.socketPath`. Permission: chmod 0600 (owner-only) — match daemon IPC pattern.
    - On boot, if socketPath exists and is a stale unix socket, unlink before bind. (Mirror existing daemon IPC behavior.)
    - Listen up to backlog 32 — small fleet, low concurrency.

    **Per-connection state machine:**
    1. `'connection'` event → wrap `socket` with line-buffered reader (PassThrough + readline OR manual `data` accumulation split on `\n`).
    2. Read first line; parse JSON; validate against `ShimHandshake` shape. If malformed → write structured error JSON line back, then `socket.destroy()`. Validation: agent is non-empty string ≤ 64 chars matching `/^[a-zA-Z0-9_\-]+$/`; tokenHash is 8 lowercase hex chars; tokenLiteral is non-empty string.
    3. If preDrainReject set → write error line ("broker draining"), destroy.
    4. Call `broker.acceptConnection(handshake)` — get ConnectionHandle.
    5. Subsequent lines from socket = JSON-RPC requests / notifications:
       - Parse JSON; extract `id`, `method`.
       - If `method === "initialize"` → `await handle.serveInitialize(msg)`; write response line back.
       - If has `id` → `await handle.dispatch(msg, handshake.turnIdHint, msg.method)`; write response line.
       - If notification (no `id`) → log debug; do not forward (out of scope; broker doesn't multiplex agent → child notifications).
    6. On socket `'close'` or `'error'` → `await handle.close()` → broker decrements refcount.

    **Token redaction (SEC-07):**
    - On handshake validation error, log `{agent: handshake.agent ?? null, tokenHash: handshake.tokenHash ?? null, error: "..."}`. NEVER log handshake.tokenLiteral.
    - Test verifies an injected literal `ops_TESTTOKEN_FAKE_XYZ` never appears in log output.

    **`closeAllConnections`:**
    - Iterate active socket set; `socket.end()` each; await all close events with 1s ceiling per socket; force-destroy remainder.

    **`preDrainReject`:**
    - Set flag → new `'connection'` events get the rejection path.
    - Existing connections continue.

    **Concurrency-safety:**
    - Multiple `'connection'` events may fire concurrently — each socket has its own state, so no shared mutex needed beyond what broker maintains.
    - Per coding-style.md: no mutation of input objects. Construct new ConnectionHandle wrappers per socket.

    File 150–250 LOC.
