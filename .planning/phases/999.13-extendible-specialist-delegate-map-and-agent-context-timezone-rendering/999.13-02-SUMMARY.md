## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Updated Test 8 saveSummary back-compat from .length === 3 to === 4**
- **Found during:** Task 2 (after wiring optional `agentTz?: string` 4th param on saveSummary)
- **Issue:** Existing Test 8 asserted `expect(saveSummary.length).toBe(3)` to pin "legacy 3-arg signature stays callable". With the new optional 4th param, JavaScript's `Function.length` returns 4 — TypeScript's `?` suffix is purely compile-time. The test failed not because back-compat broke (legacy 3-arg calls still work fine) but because the static-arity check needed updating. The Wave 0 RED tests separately verify the new 4-arg behavior; this Test 8 update is purely about correcting the obsolete static assertion.
- **Fix:** Updated to `expect(saveSummary.length).toBe(4)` with explanatory comment about TS `?` not affecting runtime arity and noting that legacy 3-arg callers still work via host-TZ fallback.
- **Files modified:** `src/memory/__tests__/context-summary.test.ts`
- **Verification:** Test 8 now passes; the 4-arg explicit-TZ test (`saveSummary-generated-tz`) also passes.
- **Commit:** `0566dee` (Task 2)

**2. [Rule 3 - Blocking] Removed 4 now-unused `@ts-expect-error` directives**
- **Found during:** Tasks 1 + 2 (after wiring the new symbols on `defaultsSchema`, `AssembleBriefInput`, `DreamPromptInput`)
- **Issue:** Wave 0 RED tests used `// @ts-expect-error Phase 999.13 RED — Plan 02 adds X` directives to suppress type errors that would otherwise fire when accessing not-yet-shipped fields. Once Plan 02 ships these fields, the directives become unused and tsc emits `TS2578: Unused '@ts-expect-error' directive`. These mask any future regressions in the affected test files.
- **Fix:** Removed all 4 directives across 3 files (`schema.test.ts` ×2, `conversation-brief.test.ts` ×1, `dream-prompt-builder.test.ts` ×1). Added inline comments explaining the intent (Q1=YES / Q2=YES / Phase 999.13 TZ-04 references).
- **Files modified:** `src/config/__tests__/schema.test.ts`, `src/memory/__tests__/conversation-brief.test.ts`, `src/manager/__tests__/dream-prompt-builder.test.ts`
- **Verification:** `npx tsc --noEmit | wc -l` returns 261 (matches baseline; the 4 newly-unused directive errors are gone).
- **Commits:** `2495f91` (schema directives, Task 1), `0566dee` (test directives, Task 2)

**3. [Rule 1 - Bug] Used `getById` not `findById` in TZ-05 regression test**
- **Found during:** Task 3 (after writing the round-trip regression test)
- **Issue:** Initial draft used `store.findById(inserted.id)` which threw `TypeError: store.findById is not a function`. The correct MemoryStore method is `getById` (returns `MemoryEntry | null`).
- **Fix:** Changed `findById` → `getById` and `expect(read).toBeDefined()` → `expect(read).not.toBeNull()` to match the null-return contract.
- **Files modified:** `src/memory/__tests__/store.test.ts`
