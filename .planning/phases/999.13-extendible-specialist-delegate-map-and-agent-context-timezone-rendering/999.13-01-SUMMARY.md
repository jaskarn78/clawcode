## Decisions Made

- **Empty/undefined short-circuits to `""`** — NO header, NO whitespace, byte-identical to no-delegates baseline. Critical for prompt-cache hash stability (Pitfall 2 in 999.13-RESEARCH.md). Verified via `delegates-block-empty-baseline` snapshot test.
- **Alphabetical sort before render** — `Object.keys(delegates).sort()` is required for prompt-cache hash stability across daemon boots. Object key iteration is not spec-guaranteed for non-integer keys (engines do insertion-order in practice; explicit sort matches Phase 94 `resolveSystemPromptDirectives` pattern at loader.ts:653). Verified via `renderDelegatesBlock-deterministic` test (two different insertion orders → byte-identical output).
- **Append-after-fs-capability placement** — delegatesBlock is the LAST element of the stable prefix's tools-and-capability cluster (after toolDefinitions, after filesystemCapabilityBlock). Matches CONTEXT.md "block goes at the bottom of the agent's system prompt" and avoids inserting between Phase 94's `<tool_status>` and Phase 95's `<dream_log_recent>` sentinel anchors. Verified via `delegates-block-injection` test (`delegIdx > toolsIdx`).
- **superRefine cross-field validation** — Lifted verbatim from the soul/soulFile mutex pattern at schema.ts:1738. Error message includes both the offending agent name AND the unknown target name per D-DELEG-03 ("operators don't grep yaml in the dark"). Verified via `delegates-superRefine-unknown-target` test (assertion: message contains both `fin-acquisition` AND `ghost-agent`).
- **ResolvedAgentConfig propagation** — Same root-cause shape as the Phase 96 fileAccess / Phase 100 settingSources fixes: schema field + resolver propagation + session-config consumption are 3 sites that must stay in lockstep. Adding `delegates` to ResolvedAgentConfig + propagating via `resolveAgentConfig` ensures the daemon's session-config can read `config.delegates` without reaching back into the raw zod parse output.
- **Cleaned up now-unused `@ts-expect-error` directives** in loader.test.ts (Rule 3 - blocking: 8 unused directives produced 8 new tsc errors that would mask future regressions). Schema.test.ts directives left intact (those were ineffective on main pre-stash because zod's `.safeParse()` accepts `unknown` — the directives never fired).
