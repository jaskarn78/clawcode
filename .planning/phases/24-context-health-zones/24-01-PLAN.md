Output: Zone classifier module, updated heartbeat check, config schema with 4 thresholds, auto-snapshot on yellow+ entry.

    - ContextZoneTracker.update(0.55) on first call returns transition {from: "green", to: "yellow"} (assumes starting green)
    - ContextZoneTracker.update(0.55) on second call returns null (no transition, same zone)
    - ContextZoneTracker.update(0.72) after yellow returns transition {from: "yellow", to: "orange"}
    - ContextZoneTracker.update(0.30) after orange returns transition {from: "orange", to: "green"}
    - ContextZoneTracker.update(0.55) triggers snapshot callback (yellow is >= yellow)
    - ContextZoneTracker.update(0.30) does NOT trigger snapshot callback (green zone)
    - ZONE_SEVERITY: green=0, yellow=1, orange=2, red=3 for comparison
  </behavior>
  <action>
    Create src/heartbeat/context-zones.ts with:

    1. Type `ContextZone = "green" | "yellow" | "orange" | "red"` (literal union, not enum).

    2. Type `ZoneThresholds = { readonly yellow: number; readonly orange: number; readonly red: number }`.
       Default thresholds: `{ yellow: 0.50, orange: 0.70, red: 0.85 }` matching CTXH-01 spec (green 0-50%, yellow 50-70%, orange 70-85%, red 85%+).

    3. `ZONE_SEVERITY` record mapping zone to numeric severity (green=0, yellow=1, orange=2, red=3) for comparison.

    4. Pure function `classifyZone(fillPercentage: number, thresholds: ZoneThresholds): ContextZone`.
       Logic: if >= red threshold return "red", if >= orange return "orange", if >= yellow return "yellow", else "green". Check from highest to lowest.

    5. Type `ZoneTransition = { readonly from: ContextZone; readonly to: ContextZone; readonly fillPercentage: number; readonly timestamp: string }`.

    6. Type `SnapshotCallback = (agentName: string, zone: ContextZone, fillPercentage: number) => Promise<void>`.

    7. Class `ContextZoneTracker` with:
       - Constructor takes `{ agentName: string; thresholds: ZoneThresholds; onSnapshot?: SnapshotCallback }`.
       - Private `currentZone: ContextZone` initialized to "green".
       - Method `update(fillPercentage: number): Promise<ZoneTransition | null>`.
         Classifies zone, compares to currentZone. If different: creates ZoneTransition, updates currentZone. If new zone severity >= yellow (severity >= 1) AND transitioning UP (new severity > old severity), calls onSnapshot callback. Returns transition or null.
       - Getter `zone: ContextZone` returns current zone.
       - Method `reset(): void` sets currentZone back to "green".

    Export: ContextZone, ZoneThresholds, ZoneTransition, ZONE_SEVERITY, DEFAULT_ZONE_THRESHOLDS, classifyZone, ContextZoneTracker, SnapshotCallback.

    Write tests FIRST in src/heartbeat/__tests__/context-zones.test.ts covering all behaviors listed above. Run tests (expect RED). Then implement. Run tests (expect GREEN).
