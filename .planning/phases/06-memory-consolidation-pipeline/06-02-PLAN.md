**Then create `src/memory/consolidation.ts` (GREEN phase).** Implement these exported functions:

1. **`detectUnconsolidatedWeeks(memoryDir: string, threshold: number): WeekGroup[]`**
   - Scan `memoryDir` for files matching `YYYY-MM-DD.md` pattern (daily logs)
   - Group by ISO week using `date-fns` `getISOWeek()` and `getISOWeekYear()`
   - For each group with >= threshold files, check if `memoryDir/digests/weekly-{year}-W{week}.md` exists
   - Return groups that need consolidation (no existing digest)
   - CRITICAL: Use `getISOWeekYear()` not `getFullYear()` for the year component (ISO week year boundary pitfall)

2. **`detectUnconsolidatedMonths(memoryDir: string, threshold: number): MonthGroup[]`**
   - Scan `memoryDir/digests/` for files matching `weekly-YYYY-WNN.md`
   - Group weekly digests by the month their ISO week START falls in (per discretion recommendation in research)
   - For each group with >= threshold digests, check if `memoryDir/digests/monthly-{year}-{month}.md` exists
   - Return groups that need consolidation

3. **`buildWeeklySummarizationPrompt(dailyLogs: Array<{date: string, content: string}>): string`**
   - Concatenate daily log content with date headers
   - Include structured extraction instructions per D-06: key facts, decisions made, topics discussed, important context
   - Request markdown output with exact section headers
   - Handle large payloads: if combined content exceeds ~30000 chars (~7500 tokens), truncate each day proportionally with a note

4. **`buildMonthlySummarizationPrompt(weeklyDigests: Array<{week: string, content: string}>): string`**
   - Similar to weekly but synthesizes weekly digests into monthly themes

5. **`writeWeeklyDigest(deps: ConsolidationDeps, weekGroup: WeekGroup, llmContent: string): Promise<WeeklyDigest>`**
   - Write markdown file to `memoryDir/digests/weekly-{year}-W{week}.md` with metadata header (per D-07)
   - Insert memory entry with: source="consolidation", importance=0.7, tags=["weekly-digest", "{year}-W{week}"] (per D-08)
   - Embed the digest content and store with the memory entry
   - Return the WeeklyDigest object

6. **`writeMonthlyDigest(deps: ConsolidationDeps, monthGroup: MonthGroup, llmContent: string): Promise<MonthlyDigest>`**
   - Write markdown file to `memoryDir/digests/monthly-{year}-{month}.md` (per D-07)
   - Insert memory entry with: source="consolidation", importance=0.8, tags=["monthly-digest", "{year}-{month}"] (per D-08)
   - Return the MonthlyDigest object

7. **`archiveDailyLogs(memoryDir: string, memoryStore: MemoryStore, files: readonly string[]): Promise<number>`**
   - For each file: move to `memoryDir/archive/YYYY/` (per D-09), extract date from filename, call `memoryStore.deleteSessionLog(date)` (per D-10)
   - Create archive directories as needed
   - Use `fs.rename()` for same-filesystem move (atomic), fall back to copy+unlink (per research)
   - Return count of archived files

8. **`archiveWeeklyDigests(memoryDir: string, files: readonly string[]): Promise<number>`**
   - Move weekly digest files to archive after monthly consolidation (per D-12)

9. **`runConsolidation(deps: ConsolidationDeps, config: ConsolidationConfig): Promise<ConsolidationResult>`**
   - Orchestrates the full pipeline: detect weeks -> summarize -> write digests -> archive dailies -> detect months -> summarize -> write monthly -> archive weeklies
   - Order matters: weekly BEFORE monthly (D-12 -- weekly digests archived after monthly created)
   - Archive LAST after confirming digest write + SQLite insert succeeded (pitfall 6 from research)
   - Collect errors without stopping -- partial consolidation is better than none
   - Return ConsolidationResult with counts and any errors
