## Decisions Made

- **`console.info` over pino `log.info` for the deprecated-alias metric.** The plan example used `log.info({alias: ..., canonical: ...}, "deprecated IPC alias used")` (pino-style structured logging), but `routeMethod` is a top-level function declared OUTSIDE the `startDaemon` scope where the pino-childed `log` is bound (see daemon.ts:1494). Threading `log` through `routeMethod`'s already-long parameter list would be a refactor far beyond Plan 02's scope. `console.info` matches what `routeMethod` already uses for analogous tracing (e.g. line 4259's webhook-delivery-failure warn). The grep target — the literal substring `[deprecated IPC alias used]` — is preserved verbatim, so the operator workflow is unchanged.
- **Did NOT add `ask_agent`/`post_to_agent` to `TOOL_DEFINITIONS`.** The metadata object exports tool→IPC mappings consumed by other tests (the `has exactly 22 tools defined` pin at server.test.ts:36-39 + the per-tool ipcMethod pins). Adding entries would break the count and require updating ~5 other tests for no runtime benefit — the actual `server.tool()` registrations are what control MCP runtime behavior. If Plan 03 needs the metadata path (e.g. for the offline-target text template), it can revisit then.
- **Schema closures hoisted to module-scope `as const` objects.** Rather than inlining the schema at each `server.tool()` call, the shared `askAgentSchema` and `postToAgentSchema` objects ensure the canonical and aliased registrations stay in lockstep automatically. When Plan 03 adds `mirror_to_target_channel` to `askAgentSchema`, both registrations get it — no drift possible.
- **MCP wire requests use canonical IPC names.** When an agent calls `ask_agent` (canonical tool), the MCP wrapper sends `sendIpcRequest(SOCKET_PATH, "ask-agent", ...)`. When an agent calls `send_message` (alias tool), the MCP wrapper ALSO sends `"ask-agent"` because both tools share the same handler closure. This means the deprecated-alias metric only fires for IPC consumers that bypass the MCP layer (CLI tools, external IPC clients still using `send-message` directly) — which is correct. The operator metric specifically tracks WIRE-LEVEL deprecated-name usage.
