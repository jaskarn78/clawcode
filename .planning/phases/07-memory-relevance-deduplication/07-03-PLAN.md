    2. Modify src/memory/search.ts (per D-04, D-05, D-06, D-07, D-08):
       - Import `scoreAndRank, ScoringConfig, RankedSearchResult` from `./relevance.js`
       - Import `DecayConfig` from `./schema.js`
       - Update constructor to accept optional `scoringConfig?: ScoringConfig` parameter (default: { semanticWeight: 0.7, decayWeight: 0.3, halfLifeDays: 30 })
       - Store config as private readonly field
       - Modify search method:
         a. Over-fetch: query KNN with `topK * 2` instead of `topK` (Pitfall 4 from research)
         b. Convert raw rows to SearchResult objects (existing logic)
         c. Call `scoreAndRank(results, this.scoringConfig, new Date())` to get RankedSearchResult[]
         d. Slice to topK after re-ranking
         e. Update access_count and accessed_at ONLY for the final topK results (per D-05, Pitfall 6)
         f. Return `readonly RankedSearchResult[]` (change return type)
       - IMPORTANT per Pitfall 6: Calculate decay scores BEFORE updating accessed_at. The access update happens after scoring.

    3. Modify src/memory/store.ts (per D-09, D-10, D-11, D-12):
       - Import `checkForDuplicate, mergeMemory, MergeInput` from `./dedup.js`
       - Import `DedupConfig` from `./schema.js`
       - Update constructor to accept optional `dedupConfig?: { enabled: boolean; similarityThreshold: number }` parameter (default: { enabled: true, similarityThreshold: 0.85 })
       - Store config as private readonly field
       - Modify insert method:
         a. If `this.dedupConfig.enabled` AND NOT `input.skipDedup`:
            - Call `checkForDuplicate(embedding, this.db, { similarityThreshold: this.dedupConfig.similarityThreshold })`
            - If result.action === "merge": call `mergeMemory(this.db, result.existingId, { content: input.content, importance: input.importance ?? 0.5, tags: input.tags ?? [], embedding })`
            - Then return the merged entry by calling `this.getById(result.existingId)` (which already increments access and returns MemoryEntry). Assert non-null since we just merged into it.
         b. If skipDedup or not enabled or result is "insert": proceed with existing insert logic unchanged
  </action>
