    - mergeMemory updates accessed_at and updated_at timestamps
    - mergeMemory replaces the embedding vector in vec_memories

  <action>
    1. Create src/memory/dedup.ts (per D-09, D-10, D-11):
       - Import Database type from better-sqlite3
       - Import MemoryEntry from types.js

       - Export `DedupConfig` type: `{ readonly similarityThreshold: number }`

       - Export `DedupResult` type:
         `{ readonly action: "insert" } | { readonly action: "merge"; readonly existingId: string; readonly similarity: number }`

       - Export `checkForDuplicate(embedding: Float32Array, db: DatabaseType, config: DedupConfig): DedupResult`
         - Prepare and run: `SELECT memory_id, distance FROM vec_memories WHERE embedding MATCH ? AND k = 1`
         - If no row returned: return `{ action: "insert" }`
         - Compute similarity = 1 - row.distance
         - If similarity >= config.similarityThreshold: return `{ action: "merge", existingId: row.memory_id, similarity }`
         - Else: return `{ action: "insert" }`

       - Export `MergeInput` type: `{ readonly content: string; readonly importance: number; readonly tags: readonly string[]; readonly embedding: Float32Array }`

       - Export `mergeMemory(db: DatabaseType, existingId: string, input: MergeInput): void`
         - Get existing entry: `SELECT importance, tags, access_count FROM memories WHERE id = ?`
         - If not found, throw MemoryError
         - Compute: mergedTags = [...new Set([...existingTags, ...input.tags])], maxImportance = Math.max(existing.importance, input.importance)
         - In a transaction:
           - UPDATE memories SET content=?, importance=?, tags=?, access_count=access_count+1, updated_at=?, accessed_at=? WHERE id=?
           - DELETE FROM vec_memories WHERE memory_id=?
           - INSERT INTO vec_memories (memory_id, embedding) VALUES (?, ?)
         - Log merge with: `console.debug("Memory merged:", { existingId, similarity: "N/A", newImportance: maxImportance, tagCount: mergedTags.length })`
