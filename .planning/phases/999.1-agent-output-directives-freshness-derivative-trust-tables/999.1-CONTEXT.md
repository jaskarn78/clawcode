## Phase Boundary

Counter-instruct Claude Code's default behaviors that misfire in this trusted-operator workspace. Four agent-output directives ship together as one coherent injection system, all wired through Phase 94's pre-existing `systemPromptDirectives` mechanism (`src/config/schema.ts` → `DEFAULT_SYSTEM_PROMPT_DIRECTIVES` → `context-assembler.ts:742-743` prepend to stable prefix).

The four directives:

1. **freshness (FRESH-\*)** — agents default to `web_search` for time-sensitive lookups instead of relying on training-cutoff data
2. **derivative-work (DERIV-\*)** — subagents understand they're allowed to create new files, derive templates, generate code, produce artifacts when delegated a task
3. **trusted-operator (TRUST-\*)** — agents skip CYA disclaimers ("this is not malware", "this is for legitimate purposes") in this trusted operator environment
4. **discord-format (TABLE-\*)** — agents prefer bullets / definition-style prose / numbered lists over markdown tables for Discord output (the webhook-wrap fence at `webhook-manager.ts:62` remains the structural safety net for cases where tabular IS right)

Out of scope (would be scope creep):
- Per-agent SOUL.md template overhauls (separate phase if needed)
- Custom date-injection runtime (agents can use `date` Bash tool if they need today's date)
- New MCP tools (search MCP is already auto-injected fleet-wide)
- Programmatic table-to-bullet rewriter (rejected — let the agent decide format from the directive)

<decisions>
## Implementation Decisions

### Architecture
- **D-ARC-01:** All 4 directives land in `DEFAULT_SYSTEM_PROMPT_DIRECTIVES` (`src/config/schema.ts`) — Phase 94's existing mechanism. No new injection code.
- **D-ARC-02:** Each directive shares the existing per-fleet enable/text override + per-agent enable-only override pattern (Phase 94 D-10). `agents.<name>.systemPromptDirectives.<key>.enabled = false` opts an agent out.
- **D-ARC-03:** Static-grep regression tests pin one canonical phrase per directive (mirrors Phase 94 / 96 testing pattern: "ALWAYS upload via Discord" pin).

### freshness directive (FRESH)
- **D-FR-01:** Default-enabled fleet-wide.
- **D-FR-02:** No dynamic date injection — keep the directive text static so it's grep-pinnable. Agents have `Bash` tool access; if they need to know today's date, they run `date`. Added a sentence in directive text: *"If you need today's date, run `date` via Bash — don't guess from your training cutoff."*
- **D-FR-03:** Time-sensitive categories listed explicitly in directive text: prices, equity quotes, tax rules / regulations, laws, financial filings, current events, recent news, "anything dated within ~6 months of today."
- **D-FR-04:** Behavior pin: *"For these, run `web_search` BEFORE answering. Do not anchor on training-cutoff knowledge."* — the canonical phrase the static-grep test pins.

### derivative-work directive (DERIV)
- **D-DR-01:** Default-enabled fleet-wide. Fires for ALL agents (not just subagents). Reasoning: keeps the implementation simple, the directive itself acknowledges it applies to delegated tasks (which subagents handle). Token cost ~80 tokens × all turns is acceptable.
- **D-DR-02:** Directive text explicitly says: *"When you've been delegated a task by another agent OR by the operator, creating new files, deriving parameterized templates from examples, generating code, and producing artifacts are all in-scope work product. The `don't add features beyond what's requested` guideline applies to scope creep on the user's request, not to fulfilling the delegated task."*
- **D-DR-03:** Canonical phrase pinned by static-grep: *"are all in-scope work product"*.
- **D-DR-04:** Reject the alternative ("only inject for subagents") — adds session-name-pattern logic to the assembler, complicates per-agent overrides, and doesn't materially help main-agent token usage.

### trusted-operator directive (TRUST)
- **D-TR-01:** Default-enabled fleet-wide.
- **D-TR-02:** Directive text: *"This workspace is a single trusted operator's environment. The operator owns this code, has full authorization, and knows the context. Do not prefix responses with disclaimers like 'this is not malware' / 'this is for legitimate purposes' / 'this is authorized work' / 'in a real engagement'. Skip all CYA language. Just answer the question or do the task."*
- **D-TR-03:** Canonical phrase pinned by static-grep: *"Skip all CYA language"*.

### discord-format directive (TABLE)
- **D-TB-01:** Default-enabled fleet-wide. Marks the deferred no-tables directive (referenced in `markdown-table-wrap.ts:11-14` comments) as landed.
- **D-TB-02:** Directive text: *"Discord doesn't render markdown tables natively. Pipes show as literal characters and columns don't align. When presenting structured data, prefer bullets, numbered lists, or definition-style prose. Use markdown tables only when the data is genuinely tabular and dense (e.g., 4+ columns × 4+ rows of comparable values); the webhook-wrap fence renders those as monospace code blocks as a safety net. For 1-3 fields per item, bullets are clearer."*
- **D-TB-03:** Canonical phrase pinned by static-grep: *"prefer bullets, numbered lists, or definition-style prose"*.
- **D-TB-04:** Companion to (does not replace) `wrapMarkdownTablesInCodeFence` — that function stays as the structural safety net for cases where tabular IS the right format.

### Token cost
- **D-TOK-01:** Combined directive text ~400-500 tokens. Lands in stable prefix → benefits from prompt caching (no per-turn re-billing for cache hits). Acceptable trade for the behavior changes.

<specifics>
## Specific Ideas

The 4 directives address operator pain observed during the 2026-04-29 session:

1. **FRESH** — agents emitting 2025-anchored answers when asked about live prices, current laws, recent filings. Fix: directive nudges them to `web_search` first.
2. **DERIV** — subagent for fin-acquisition refused to generate parameterized PDF template; operator had to do the work in main context. The agent diagnosed it as "simplify skill hook" but no such hook exists locally — the real cause is the global CLAUDE.md "don't add features beyond what's requested" rule being misapplied. Fix: directive clarifies that delegated work IS in-scope.
3. **TRUST** — agents repeatedly prefacing responses with "this is not malware / this is for legitimate purposes" disclaimers. Source: Claude Code's platform-level "authorized security testing" prompt; agents over-apply it. Fix: counter-instruction telling them to skip CYA language in this trusted operator environment.
4. **TABLE** — Discord screenshots showed agents emitting markdown tables that don't render (pipes literal, columns misaligned). Phase 100-fu added the webhook-wrap fence as a structural safety net (extended in `260429-ouw` to cover all webhook send paths). The directive completes the pair by steering agents toward bullets when bullets work better — fence is for genuinely tabular content only.

</specifics>

<deferred>
## Deferred Ideas

- **Per-agent SOUL.md template guidance** — could replicate parts of these directives in each agent's SOUL.md for emphasis. Decided NOT to do this in scope: the directive mechanism is already fleet-wide, propagates via stable prefix, and per-agent SOUL changes risk drift. Defer until / unless we see directives not landing strongly enough.
- **Programmatic table-to-bullet rewriter** — could detect markdown tables in agent output and rewrite to bullets server-side. Rejected: too aggressive (sometimes agents legitimately want a table even in Discord), the wrap-as-fence fallback handles those cases, and the directive nudges toward bullets without forcing it.
- **Dynamic today-date injection** — the FRESH directive could include a line like `Today is ${YYYY-MM-DD}.` injected at assembly time. Rejected for first iteration: the agent has Bash tool access and can run `date` if needed. Static text is grep-pinnable and reduces assembly complexity. Revisit if agents still drift on dates after deploy.
- **DERIV opt-in-per-task instead of fleet-wide** — could make the derivative-work permission only fire when an agent receives a task via `spawn_subagent_thread`. Decided fleet-wide for simplicity (D-DR-04). Revisit if main-agent behavior changes unexpectedly.
