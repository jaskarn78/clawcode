## FRESH: Today's date — dynamic injection or static directive text?

| Option | Description | Selected |
|--------|-------------|----------|
| Static text + agent runs `date` if needed | Directive text says "if you need today's date, run `date` via Bash" — keeps text grep-pinnable | ✓ |
| Dynamic injection at assembly time | Assembler interpolates `Today is ${YYYY-MM-DD}.` into the directive text per-turn | |
| Per-agent override of date format | Agents could opt out or override how the date appears | |

**Auto-selected:** Static text.
**Notes:** Dynamic injection adds assembly complexity, breaks the static-grep regression-test pattern, and requires only marginal benefit (agent can already run `date`).

## DERIV: Fleet-wide or subagent-only?

| Option | Description | Selected |
|--------|-------------|----------|
| Fleet-wide (all agents see the directive) | Simple — directive itself acknowledges it applies to "delegated tasks" | ✓ |

## TABLE: Hard ban on tables, or steer-toward-bullets?

| Option | Description | Selected |
|--------|-------------|----------|
| Steer toward bullets, allow tables when genuinely tabular | Pairs with existing wrap-fence safety net for tables | ✓ |
| Hard ban — never emit markdown tables | Eliminates ambiguity but loses the legitimate-table case | |
| Programmatic rewrite (server-side) | Detect tables and convert to bullets before posting | |

**Auto-selected:** Steer toward bullets + wrap-fence safety net.
**Notes:** Hard ban is too aggressive (4+ column comparisons genuinely benefit from tabular layout). Programmatic rewrite is too aggressive in the other direction (would mangle legitimate tables).
