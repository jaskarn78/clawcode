# Plan 116-01: Tier 1 read-only surfaces

## Goal

Replace the 116-00 smoke component with the five read-only Tier 1 features. All data sources exist via `/api/agents/:name/cache` + `/api/status` + `/api/fleet-stats` — **no backend additions**. Pure React/component work.

## Features

- **F01** SLO breach banner — dismissible header banner; "⚠ N SLO breaches active" with click-to-drill into offending agent's drawer (drawer ships in 116-04)
- **F03** Agent tile grid — one card per agent: status dot, name + model badge, context meter, first-token p50 SLO color (uses F02 per-model threshold from 116-00 T02), 24h activity sparkline, last-turn relative time, migration phase pill (if active), MCP server health badge
- **F04** Tier-1 budget meter — horizontal Progress bar, 16,000-char cap; color band green <70%, amber 70-85%, red >85%; hover tooltip with raw chars
- **F05** Tool cache hit rate gauge — Recharts PieChart donut showing hit rate; target ≥40% indicator; per-tool breakdown on click (popover)
- **F08** Prompt bloat + lazy recall counters — two compact pill-style counters in tile; >0 amber-flagged for prompt-bloat-warnings; lazy-recall-calls as proxy for memory tool usage
