# Plan 115-07: MCP tool-response cache (folds Phase 999.40)

## Goal

Daemon-side content-keyed cache for repeated MCP tool calls. Eliminates the observed 120s `mysql_query` p50 hit on cache misses + the ~14s cache-miss penalty on first_token. Per-tool TTL policy gives different caching behavior to `web_search` (5min, public), `mysql_query` (60s for explicit-read), `search_documents` (30min, per-agent), `image_generate` (never), `spawn_subagent_thread` (never — each call unique).

**Critical isolation invariant** (per roadmap line 857): `search_documents` is per-agent (cache key includes `agent_name`); `web_search` cross-agent OK (cache key OMITS `agent_name`, public data shared). Both keying strategies must be grep-verifiable.

This plan fully absorbs Phase 999.40 (now SUPERSEDED-BY-115).

## Sub-scopes covered

- Sub-scope 15 (roadmap line 857): MCP tool-response cache.

## must_haves

- A new SQLite database `~/.clawcode/manager/tool-cache.db` exists with table `tool_cache(key, tool, agent_or_null, response_json, created_at, expires_at)`.
- LRU eviction + 100MB default cap (configurable via `defaults.toolCache.maxSizeMb`).
- Per-tool TTL policy enforced: `web_search` 300s, `brave_search` 300s, `exa_search` 300s, `search_documents` 1800s, `mysql_query` 60s for read-only patterns, `google_workspace_*_get` 300s, `image_generate` 0 (no cache), `spawn_subagent_thread` 0.
- Bypass via `bypass_cache: true` in tool args.
- Cache stamping: response wrapped with `cached: { age_ms, source }` so agents can decide whether to trust or refresh.
- `search_documents` cache key includes `agent_name` (per-agent isolation).
- `web_search` / `brave_search` / `exa_search` cache keys OMIT `agent_name` (cross-agent shared, public data).
- Cache hit-rate metric `tool_cache_hit_rate` is written to `traces.db.tool_cache_hit_rate` (column added in 115-00-T02).
- Cache size metric `tool_cache_size_mb` is written to `traces.db.tool_cache_size_mb`.
- `mysql_query` write-pattern detector — caches ONLY when query starts with `SELECT` / `WITH` (read patterns); never caches `INSERT` / `UPDATE` / `DELETE` / `DROP` / `ALTER`.
- A new CLI command `clawcode tool-cache status|clear|inspect` exists.

    import Database from "better-sqlite3";
    import { promises as fs } from "fs";
    import { homedir } from "os";
    import { dirname, join } from "path";
    import { createHash } from "crypto";

    const DB_PATH = join(homedir(), ".clawcode", "manager", "tool-cache.db");

    export interface CacheRow {
      key: string;
      tool: string;
      agent_or_null: string | null;
      response_json: string;
      created_at: number;        // ms epoch
      expires_at: number;        // ms epoch
      bytes: number;
      last_accessed_at: number;  // for LRU
    }

    export class ToolCacheStore {
      private db: Database.Database;

      constructor(opts?: { path?: string }) {
        const p = opts?.path ?? DB_PATH;
        // synchronous mkdir is fine — happens once at startup.
        const fs2 = require("fs");
        fs2.mkdirSync(dirname(p), { recursive: true });
        this.db = new Database(p);
        this.db.pragma("journal_mode = WAL");
        this.db.exec(`
          CREATE TABLE IF NOT EXISTS tool_cache (
            key TEXT PRIMARY KEY,
            tool TEXT NOT NULL,
            agent_or_null TEXT,
            response_json TEXT NOT NULL,
            created_at INTEGER NOT NULL,
            expires_at INTEGER NOT NULL,
            bytes INTEGER NOT NULL,
            last_accessed_at INTEGER NOT NULL
          );
          CREATE INDEX IF NOT EXISTS idx_tool_cache_tool ON tool_cache(tool);
          CREATE INDEX IF NOT EXISTS idx_tool_cache_expires ON tool_cache(expires_at);
          CREATE INDEX IF NOT EXISTS idx_tool_cache_lru ON tool_cache(last_accessed_at);
        `);
      }

      get(key: string, now: number = Date.now()): CacheRow | null {
        const row = this.db.prepare("SELECT * FROM tool_cache WHERE key = ?").get(key) as CacheRow | undefined;
        if (!row) return null;
        if (row.expires_at <= now) {
          // Lazy expiration.
          this.db.prepare("DELETE FROM tool_cache WHERE key = ?").run(key);
          return null;
        }
        // Update LRU.
        this.db.prepare("UPDATE tool_cache SET last_accessed_at = ? WHERE key = ?").run(now, key);
        return { ...row, last_accessed_at: now };
      }

      put(row: Omit<CacheRow, "last_accessed_at" | "bytes">, maxSizeMb: number = 100): void {
        const bytes = Buffer.byteLength(row.response_json, "utf8");
        const now = Date.now();
        this.db.transaction(() => {
          this.db.prepare(`
            INSERT OR REPLACE INTO tool_cache (key, tool, agent_or_null, response_json, created_at, expires_at, bytes, last_accessed_at)
            VALUES (?, ?, ?, ?, ?, ?, ?, ?)
          `).run(row.key, row.tool, row.agent_or_null, row.response_json, row.created_at, row.expires_at, bytes, now);
          this.evictIfNeeded(maxSizeMb, now);
        })();
      }

      private evictIfNeeded(maxSizeMb: number, now: number): void {
        const totalRow = this.db.prepare("SELECT COALESCE(SUM(bytes), 0) AS total FROM tool_cache").get() as { total: number };
        const limit = maxSizeMb * 1024 * 1024;
        if (totalRow.total <= limit) return;

        // Drop expired first.
        this.db.prepare("DELETE FROM tool_cache WHERE expires_at <= ?").run(now);

        // LRU evict until under limit.
        let total = (this.db.prepare("SELECT COALESCE(SUM(bytes), 0) AS total FROM tool_cache").get() as { total: number }).total;
        while (total > limit) {
          const oldest = this.db.prepare("SELECT key, bytes FROM tool_cache ORDER BY last_accessed_at ASC LIMIT 1").get() as { key: string; bytes: number } | undefined;
          if (!oldest) break;
          this.db.prepare("DELETE FROM tool_cache WHERE key = ?").run(oldest.key);
          total -= oldest.bytes;
        }
      }

      sizeMb(): number {
        const totalRow = this.db.prepare("SELECT COALESCE(SUM(bytes), 0) AS total FROM tool_cache").get() as { total: number };
        return totalRow.total / (1024 * 1024);
      }

      clear(): void {
        this.db.exec("DELETE FROM tool_cache");
      }

      inspect(tool?: string, agent?: string, limit = 100): CacheRow[] {
        let sql = "SELECT * FROM tool_cache";
        const params: any[] = [];
        const where: string[] = [];
        if (tool) { where.push("tool = ?"); params.push(tool); }
        if (agent) { where.push("agent_or_null = ?"); params.push(agent); }
        if (where.length) sql += " WHERE " + where.join(" AND ");
        sql += " ORDER BY last_accessed_at DESC LIMIT ?";
        params.push(limit);
        return this.db.prepare(sql).all(...params) as CacheRow[];
      }

      close(): void {
        this.db.close();
      }
    }
    ```

    /**
     * Phase 115 sub-scope 15 — tool-response cache policy table.
     */

    export type CacheKeyStrategy = "per-agent" | "cross-agent" | "no-cache";

    export interface ToolCachePolicy {
      ttlSeconds: number;          // 0 = never cache
      keyStrategy: CacheKeyStrategy;
      // Optional content-validation predicate — for mysql_query, only cache read patterns.
      cacheable?: (args: Record<string, unknown>) => boolean;
    }

    /**
     * Per-tool policy. Defaults defined here; operators can override per-tool via clawcode.yaml
     * `defaults.toolCache.policy.<tool_name>: { ttlSeconds, keyStrategy }`.
     */
    export const DEFAULT_TOOL_CACHE_POLICY: Record<string, ToolCachePolicy> = {
      // Public web data — cross-agent shared, 5min.
      web_search:                  { ttlSeconds: 300,  keyStrategy: "cross-agent" },
      brave_search:                { ttlSeconds: 300,  keyStrategy: "cross-agent" },
      exa_search:                  { ttlSeconds: 300,  keyStrategy: "cross-agent" },

      // Per-agent document corpus — 30min, agent-scoped.
      search_documents:            { ttlSeconds: 1800, keyStrategy: "per-agent" },

      // mysql_query — 60s read-only patterns; never cache writes.
      mysql_query:                 {
        ttlSeconds: 60,
        keyStrategy: "per-agent",
        cacheable: (args) => isReadOnlySql(typeof args.query === "string" ? args.query : ""),
      },

      // Google workspace — get-shaped only, 5min.
      google_workspace_drive_get:    { ttlSeconds: 300, keyStrategy: "per-agent" },
      google_workspace_calendar_get: { ttlSeconds: 300, keyStrategy: "per-agent" },
      google_workspace_gmail_get:    { ttlSeconds: 300, keyStrategy: "per-agent" },

      // Image generation + subagent threads — NEVER cache.
      image_generate:              { ttlSeconds: 0, keyStrategy: "no-cache" },
      spawn_subagent_thread:       { ttlSeconds: 0, keyStrategy: "no-cache" },
    };

    /**
     * mysql_query write-pattern detector.
     * Caches ONLY when query starts with SELECT or WITH (read patterns).
     * Never caches INSERT / UPDATE / DELETE / DROP / ALTER / TRUNCATE / REPLACE / CREATE / GRANT.
     */
    export function isReadOnlySql(query: string): boolean {
      const trimmed = query.trim().toLowerCase();
      // Guard against e.g. "WITH foo AS (SELECT ...) UPDATE bar ..." — be strict.
      if (/^(insert|update|delete|drop|alter|truncate|replace|create|grant|revoke|merge|call|exec)\b/.test(trimmed)) return false;
      // Look for nested write keywords (CTE-then-write) — refuse.
      if (/\bupdate\b|\binsert\b|\bdelete\b|\bdrop\b|\balter\b|\btruncate\b/.test(trimmed)) return false;
      return /^(select|with|show|describe|explain)\b/.test(trimmed);
    }

    /**
     * Build cache key. Per-agent: includes agent_name. Cross-agent: omits agent_name (public data shared).
     */
    export function buildCacheKey(tool: string, args: Record<string, unknown>, agentName: string, strategy: CacheKeyStrategy): string {
      const argsJson = stableStringify(args);
      const components = strategy === "per-agent"
        ? [tool, agentName, argsJson]
        : [tool, argsJson];   // cross-agent OMITS agentName — public data shared across agents
      const hash = createHash("sha256").update(components.join(" ")).digest("hex");
      return `${tool}:${strategy === "per-agent" ? agentName.slice(0, 12) + ":" : ""}${hash.slice(0, 32)}`;
    }

    function stableStringify(obj: unknown): string {
      // Sorted-keys JSON for deterministic key generation.
      if (obj === null || typeof obj !== "object") return JSON.stringify(obj);
      if (Array.isArray(obj)) return `[${obj.map(stableStringify).join(",")}]`;
      const keys = Object.keys(obj as Record<string, unknown>).sort();
      return `{${keys.map(k => JSON.stringify(k) + ":" + stableStringify((obj as Record<string, unknown>)[k])).join(",")}}`;
    }

    /**
     * Cache stamping — wraps a cached response with metadata so the agent can detect/handle staleness.
     */
    export interface CacheStamped<T> {
      cached: { age_ms: number; source: "tool-cache" };
      data: T;
    }

    export function stampCachedResponse<T>(data: T, createdAtMs: number): CacheStamped<T> {
      return {
        cached: { age_ms: Date.now() - createdAtMs, source: "tool-cache" },
        data,
      };
    }

    import { createHash } from "crypto";
    ```

    import { ToolCacheStore } from "./tool-cache-store.js";
    import { DEFAULT_TOOL_CACHE_POLICY, buildCacheKey, stampCachedResponse, ToolCachePolicy } from "./tool-cache-policy.js";

    interface DispatchOptions {
      tool: string;
      args: Record<string, unknown>;
      agentName: string;
      cacheStore: ToolCacheStore;
      maxSizeMb: number;
      log: Logger;
      traceCollector?: TraceCollector;
      effectivePolicy: ToolCachePolicy;
    }

    export async function dispatchTool(opts: DispatchOptions): Promise<unknown> {
      const { tool, args, agentName, cacheStore, log, traceCollector, effectivePolicy } = opts;

      // Bypass.
      if (args.bypass_cache === true) {
        log.debug({ tool, action: "cache-bypass" }, "[diag] tool-cache-bypass");
        return await dispatchUpstream(tool, args, agentName);
      }

      // No-cache policy.
      if (effectivePolicy.keyStrategy === "no-cache" || effectivePolicy.ttlSeconds <= 0) {
        return await dispatchUpstream(tool, args, agentName);
      }

      // Content-validation gate (e.g., mysql_query writes).
      if (effectivePolicy.cacheable && !effectivePolicy.cacheable(args)) {
        log.debug({ tool, action: "cache-skip-not-cacheable" }, "[diag] tool-cache-skip");
        return await dispatchUpstream(tool, args, agentName);
      }

      const argsForKey = { ...args };
      delete argsForKey.bypass_cache;
      const key = buildCacheKey(tool, argsForKey, agentName, effectivePolicy.keyStrategy);

      const hit = cacheStore.get(key);
      if (hit) {
        traceCollector?.recordToolCacheHit(agentName, tool);
        const data = JSON.parse(hit.response_json);
        return stampCachedResponse(data, hit.created_at);
      }
      traceCollector?.recordToolCacheMiss(agentName, tool);

      const result = await dispatchUpstream(tool, args, agentName);
      const now = Date.now();
      cacheStore.put({
        key,
        tool,
        agent_or_null: effectivePolicy.keyStrategy === "per-agent" ? agentName : null,
        response_json: JSON.stringify(result),
        created_at: now,
        expires_at: now + effectivePolicy.ttlSeconds * 1000,
      }, opts.maxSizeMb);

      return result;
    }

## Threat Model

| Threat | Vector | Severity | Mitigation |
|---|---|---|---|
| Per-agent isolation breach — agent A reads agent B's `search_documents` cached results | Wrong cache key strategy | CRITICAL | `search_documents` policy locks `keyStrategy: "per-agent"`; `buildCacheKey` includes `agentName` in components for per-agent strategy; explicit test fixture with two agents asserts distinct cache keys |
| `web_search` accidentally cached per-agent (wasted memory + isolation theater) | Default policy misconfigured | LOW | `web_search` / `brave_search` / `exa_search` locked to `cross-agent` in DEFAULT_TOOL_CACHE_POLICY; operator overrides allowed but tested |
| `mysql_query` returns stale write data (e.g., reads after INSERT show pre-INSERT state) | TTL-bound cache + write happened externally | MEDIUM | mysql_query TTL is 60s (short); `cached.age_ms` stamp visible in response so agent can re-call with `bypass_cache: true` if needed |
| `mysql_query` write masquerading as read — `WITH x AS (UPDATE bar) SELECT 1` accepted as cacheable | CTE-then-write detection bypassed | HIGH | `isReadOnlySql` checks for nested write keywords (`update|insert|delete|...`) anywhere in the trimmed query, not just at the start |
| Cache leaks API keys / tokens via cached responses | Cached `mysql_query` result contains credential rows | HIGH | Cache key uses sha256 hash; cached response is stored as JSON in tool-cache.db (local-only file). NOT redacted at storage time — operators must avoid querying credentials with cacheable=true. Documented warning in CLI `tool-cache inspect` output |
| Cache size bloats unbounded → disk fill | LRU eviction failing | MEDIUM | 100MB hard cap default; LRU eviction runs in same transaction as insert; periodic size metric to traces.db |
| Stale tool cache outlives a security-sensitive permission change | Agent permissions revoked but cached tool result remains accessible to that agent | MEDIUM | Per-agent caches keyed by agentName; operator can `clawcode tool-cache clear <tool>` to flush; future enhancement: clear-on-config-change |
