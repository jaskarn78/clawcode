# Plan 115-08: Tool-latency methodology audit + parallel-tool-call instrumentation + tool_use_rate measurement

## Goal

Three deliverables in service of the gating decision for sub-scope 6-B (1h-TTL direct-SDK fast-path):

(a) **Trace-collector audit** — split today's monolithic `tool_call` metric (currently full LLM round-trip) into `tool_execution_ms` (SDK dispatch → tool result) and `tool_roundtrip_ms` (full duration including LLM resume). Surfaces the prompt-bloat-tax distinct from genuine tool latency.
(b) **Parallel-tool-call instrumentation** — count per-turn parallel tool_use blocks vs sequential.
(c) **`tool_use_rate_per_turn` measurement (sub-scope 6-A)** — count of turns with ≥1 tool_use vs total turns, per-agent. This is the GATE for sub-scope 6-B in plan 115-09.

Plan 115-09 reads this plan's measurement output to decide whether 6-B ships. D-12 30% threshold is the starting threshold; this plan's data may refine it.

Also writes a wave-2-checkpoint perf-comparison report.

    ```ts
      tool: string;
      executionStartMs: number;     // when tool dispatch began
      executionEndMs: number;       // when tool result returned to model
      roundtripEndMs: number;       // when assistant resumed and emitted next text/tool_use
    }

    export class TraceCollector {
      // ... existing
      recordToolCall(turnId: string, t: ToolCallTimings): void {
        const executionMs = t.executionEndMs - t.executionStartMs;
        const roundtripMs = t.roundtripEndMs - t.executionStartMs;
        // Update span / row.
        this.db.prepare("UPDATE turns SET tool_execution_ms = COALESCE(tool_execution_ms, 0) + ?, tool_roundtrip_ms = COALESCE(tool_roundtrip_ms, 0) + ? WHERE id = ?")
          .run(executionMs, roundtripMs, turnId);
      }

      recordParallelToolCallCount(turnId: string, count: number): void {
        this.db.prepare("UPDATE turns SET parallel_tool_call_count = ? WHERE id = ?").run(count, turnId);
      }
    }
    ```

    Step 1 — Add a periodic computation pass (every 5min via heartbeat OR on-demand via CLI). Computes for each agent:
    ```ts
    export interface ToolUseRateRow {
      agent: string;
      windowHours: number;       // e.g., 24
      turnsTotal: number;
      turnsWithTools: number;
      rate: number;              // turnsWithTools / turnsTotal
    }

    export async function computeToolUseRatePerTurn(
      db: Database,
      agent: string,
      windowHours: number = 24,
    ): Promise<ToolUseRateRow> {
      const sinceMs = Date.now() - windowHours * 60 * 60 * 1000;
      const totalRow = db.prepare("SELECT COUNT(*) AS n FROM turns WHERE agent_name = ? AND created_at >= ?").get(agent, sinceMs) as { n: number };
      const toolRow = db.prepare("SELECT COUNT(*) AS n FROM turns WHERE agent_name = ? AND created_at >= ? AND parallel_tool_call_count IS NOT NULL AND parallel_tool_call_count > 0").get(agent, sinceMs) as { n: number };
      const rate = totalRow.n > 0 ? toolRow.n / totalRow.n : 0;
      return {
        agent,
        windowHours,
        turnsTotal: totalRow.n,
        turnsWithTools: toolRow.n,
        rate,
      };
    }
    ```
