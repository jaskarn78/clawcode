## Wave 1 contract — thrower stubs

Wave 1 replaces the BODY of each thrower stub. The TYPES stay byte-stable.

**`src/discord/thread-cleanup.ts`:**
- `classifyDiscordCleanupError(err: unknown): "prune" | "retain" | "unknown"` — pure function, total over the truth table:
  - 50001 / 10003 / 404 → `"prune"`
  - 5xx / 429 / `{ECONNRESET, ETIMEDOUT, ENOTFOUND, EAI_AGAIN}` → `"retain"`
  - Anything else → `"unknown"`
- `cleanupThreadWithClassifier({spawner, registryPath, threadId, agentName, log, lock?}): Promise<{archived, bindingPruned, classification}>` — wraps `spawner.archiveThread`, classifies thrown errors, prunes registry on prune, emits canonical warn log with `{component:'thread-cleanup', action:'prune-after-discord-error', discordCode, threadId, agentName}`.

**`src/discord/stale-binding-sweep.ts`:**
- `scanStaleBindings({registry, now, idleMs}): readonly ThreadBinding[]` — pure filter, oldest-first sort, idleMs<=0 disables.
- `parseIdleDuration(input: string): number` — `"24h"` / `"6h"` / `"30m"` / `"0"` recognized, garbage throws.
- `sweepStaleBindings({spawner, registryPath, now, idleMs, log}): Promise<{staleCount, prunedCount, agents}>` — orchestrator, ONE summary warn log per cycle, idleMs<=0 short-circuits, individual cleanup failures continue the sweep.

**`src/cli/prompts.ts`:**
- `confirmPrompt(message: string): Promise<boolean>` — true on `y`/`yes` (case-insensitive), false otherwise; Wave 1 implements with `node:readline/promises`.

**Daemon side (Wave 1 also adds):**
- New IPC method `threads-prune-stale` — params `{staleAfter: string}`, response `{staleCount, prunedCount, agents}`.
- New IPC method `threads-prune-agent` — params `{agent: string}`, response `{prunedCount}`.
- Existing `archive-discord-thread` IPC handler swap-in: now calls `cleanupThreadWithClassifier` instead of bare `archiveThread`. Returns `{ok:true, archived, bindingPruned, classification}` on every path (including 50001/10003 — no longer throws).
- `defaults.threadIdleArchiveAfter: z.string().optional()` config field; `"24h"` default; `"0"` disables sweep.
- `startOrphanReaper({...args, onTickAfter: () => sweepStaleBindings({...})})` — wire MCP-09 into the existing reaper loop.

## Deviations from Plan

### Auto-fixed Issues

**[Rule 3 - Blocking] Test 8 (orphan-reaper) `runAllTimersAsync` infinite-loop**
- **Found during:** Task 3 — adding the onTickAfter sequence-pinning test.
- **Issue:** First draft used `vi.runAllTimersAsync()` to drain microtasks after advancing time by `intervalMs`. Because the setInterval handle was still active, vitest hit its 10k-timer abort guard.
- **Fix:** Reordered the test — clear the interval first, then advance fake timers by 100ms to drain the in-flight async wrapper microtasks. Sequence is still observable: the callOrder array captures both the reap-side `listAllPids` calls and the onTickAfter call.
- **Files modified:** `src/mcp/__tests__/orphan-reaper.test.ts`
- **Verification:** `npx vitest run src/mcp/__tests__/orphan-reaper.test.ts` → 8/8 GREEN.
- **Commit:** 521066a

### Architectural choices not in original plan

**Thrower stubs instead of `declare module` shims (CONTEXT-PLAN deviation, accepted by additional_context)**
- The PLAN's verification section lists "decision (a) — declaration shims" via top-level `declare module` blocks in each test file.
- The additional_context provided with this execution explicitly overrode that to "real declaration shim modules with thrower stubs (Wave 1 Task 4 replaces with real impl)".
- I followed the additional_context. The thrower-stub approach has identical type-checking properties (tsc clean) AND a slightly cleaner contract surface (the stub modules ARE the contract, no inline `declare module` boilerplate cluttering the test files).
- Wave 1 replaces the function bodies; types stay byte-stable.
- **Tracking:** Documented as a deliberate-and-superior-to-PLAN choice; Wave 1 plan should NOT delete the stub files, just rewrite the bodies.

**Total deviations:** 1 auto-fixed (Rule 3 — test infrastructure). **Impact:** None — sequence-pinning invariant still proved by Test 8, just with a clearer test-time API.
