## Plan-vs-Reality Notes

- **Plan said "extend src/config/__tests__/schema.test.ts"** — chose to create a separate file (`schema-heartbeat-inbox-timeout.test.ts`) following the existing per-feature schema test convention (snapshot-defaults, autoStart, fileAccess, outputDir). Cleaner isolation; matches Phase 999.6 SNAP-04's pattern verbatim.
- **Plan said "include exactly 4 ts-expect-error directives"** — TypeScript collapses excess-property checks per object literal (only first error fires), so each test gets ONE directive on `botDirectSender` and the implicit excess-property check covers `agentChannels`. Avoids "unused @ts-expect-error" errors.
