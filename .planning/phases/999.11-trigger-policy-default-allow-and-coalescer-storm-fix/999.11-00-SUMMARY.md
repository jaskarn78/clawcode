## Decisions Made

- **MC-6 renamed to MC-7.** The plan numbered the new test MC-6, but message-coalescer.test.ts already had a test labeled "MC-6: default perAgentCap is 50". Renaming to MC-7 avoids overwriting an existing assertion. Tracked here so Plan 02 references the right label.
- **POLICY tests are regression locks, not driver-RED.** The engine ternary is already correct on main; the actual fail-closed bug lives at the daemon boot site (daemon.ts:2034). The plan author called this out explicitly in Task 0b's `<behavior>`. Plan 01's GREEN moment will be verified via build smoke (boot log + journal grep), not via these tests.
- **Hard ceiling in storm-test mocks.** Without a ceiling, CO-10 / CO-11 OOMed the vitest worker on current main (the loop is unbounded). HARD_CEILING=25 with force-success after the cap means the assertion fails cleanly with a meaningful number (e.g. "expected 26 to be less than or equal to 5") instead of crashing the worker. Post-Plan 02 with cap=3, the ceiling is unreachable.
- **createBridge default `() => false` for hasActiveTurn.** Mirrors RESEARCH.md Pitfall 4. Existing CO-1..CO-6 use the helper without modification; only CO-9 opts in to `() => true`.
