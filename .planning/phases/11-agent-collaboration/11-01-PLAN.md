  <action>
Create `src/collaboration/types.ts` with:

```typescript
export type MessagePriority = "normal" | "high" | "urgent";

export type InboxMessage = {
  readonly id: string;
  readonly from: string;
  readonly to: string;
  readonly content: string;
  readonly timestamp: number;
  readonly priority: MessagePriority;
};
```

Create `src/collaboration/inbox.ts` with three exported functions:

1. `writeMessage(inboxDir: string, message: InboxMessage): Promise<void>` — Per D-05, writes JSON file named `{timestamp}-{from-agent}.json` to `{inboxDir}/`. Create directory if missing (`mkdir -p` equivalent). Use atomic write pattern: write to `.tmp` then rename. Use `nanoid` for uniqueness suffix if timestamp collides.

2. `readMessages(inboxDir: string): Promise<readonly InboxMessage[]>` — Reads all `.json` files in `{inboxDir}/`, parses each, returns sorted by timestamp ascending. Returns empty array if directory doesn't exist. Validate each file parses correctly, skip malformed files with a warning log.

3. `markProcessed(inboxDir: string, messageId: string): Promise<void>` — Per D-08, moves message file from `{inboxDir}/` to `{inboxDir}/processed/`. Create processed dir if missing. Match file by scanning for file containing matching id (since filename is timestamp-based, not id-based). If file not found, no-op (idempotent).

Use `nanoid` (already in package.json) for message ID generation in a helper `createMessage(from, to, content, priority?)` that returns an `InboxMessage` with generated id and current timestamp.

Export `createMessage` as well.
  </action>
