<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| disk → ingestor | Untrusted file content (PDF/docx/xlsx/image) crosses here |
| ingestor → Tesseract CLI | Spawned subprocess receives a file path |
| ingestor → Claude vision API | Page images crossed over network |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-101-01 | Tampering | input file path → Tesseract CLI spawn | mitigate | spawn via `execa(['tesseract', absoluteFilePath, '-'])` with arg array (no shell interpolation); validate path with `path.resolve` + existsSync before spawn |
| T-101-02 | Information Disclosure | OCR'd document content → pino logs | mitigate | telemetry logs only docSlug + type + page count + ocrConfidence; never log extracted text content |
| T-101-03 | DoS | adversarial 10000-page PDF | mitigate | hard cap MAX_PAGES=500 in page-batch.ts; reject with structured error above cap |
| T-101-04 | Tampering | image dimensions → Claude vision request | mitigate | sharp resize to DIMENSION_MAX_PX=2000 before send; assert max bytes ≤ 30MB total per batch |
| T-101-SC | Tampering | npm installs (file-type, node-tesseract-ocr, mammoth, exceljs, pdf-parse, tesseract.js) | mitigate | RESEARCH.md slopcheck audit table covers all six packages — verified before this plan; no [ASSUMED]/[SUS] entries |
</threat_model>
