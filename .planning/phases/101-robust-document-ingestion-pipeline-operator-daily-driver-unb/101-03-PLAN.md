Existing applyTimeWindowFilter (src/memory/memory-chunks.ts:166-175):
```typescript
export function applyTimeWindowFilter<
  T extends { path: string; file_mtime_ms: number },
>(chunks: readonly T[], days: number, now: number = Date.now()): readonly T[] {
  const cutoff = now - days * 86_400_000;
  return chunks.filter((c) => {
    if (c.path.includes("/memory/vault/")) return true;
    if (c.path.includes("/memory/procedures/")) return true;
    return c.file_mtime_ms >= cutoff;
  });
}
```

CF-1 fix: add one branch — `if (c.path.startsWith("document:")) return true;` between the procedures branch and the cutoff check. Single LOC change + a dedicated regression test.

Phase 115 dual-write coordination (CF-2 cross-write):
- Each agent has a "migration phase" config: `v1-only` | `dual-write` | `v2-only`.
- v1-only: writes only to vec_memory_chunks (float[384]) via embedder.embed().
- dual-write: writes to both vec_memory_chunks AND vec_memory_chunks_v2 (int8[384]) via embedder.embed() + embedder.embedV2().
- v2-only: writes only to vec_memory_chunks_v2 (int8[384]).
- U6 cross-ingest MUST honor the current phase: if v1-only, auto-flip to dual-write (one-time write to a coordination flag, e.g., agent-level config persisted to ~/.clawcode/agents/<agent>/migration-phase.json — researcher's recommendation per CF-2).
