## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → ingest_document | Tool input crosses to daemon |
| extracted text → Anthropic tool-use call | document content sent to extraction model |
| failed extraction → admin-clawdy alert | structured failure metadata flows over IPC |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-101-05 | Tampering | MCP tool input → daemon | mitigate | zod validate IngestDocumentInput at MCP server entry; reject invalid file_path / unknown agent |
| T-101-06 | Information Disclosure | extracted PII (SSN, account #) → pino logs | mitigate | alerts.ts logs docSlug + missingFields + ocrConfidence only; never log extracted field values |
| T-101-07 | Repudiation | Mistral backend smuggled in via env | mitigate | config knob defaults false; backend selection requires explicit `defaults.documentIngest.allowMistralOcr: true` AND `--backend mistral` flag; stub throws otherwise |
| T-101-08 | Elevation of Privilege | file_path traversal | mitigate | resolve file_path against agent workspace root; reject paths outside workspace via `path.relative(workspaceRoot, resolved).startsWith('..')` check |
</threat_model>
