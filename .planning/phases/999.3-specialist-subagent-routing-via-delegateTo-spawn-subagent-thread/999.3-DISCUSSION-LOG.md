## Config inheritance: caller vs. delegate split

| Field | Source | Rationale |
|-------|--------|-----------|
| model, soul, identity, skills, mcpServers, subagentModel, effortLevel | Delegate | The delegate IS the work-doer — use their config |
| channels (thread location) | Caller | Thread spawns where caller asked |
| threads.maxThreadSessions (quota) | Caller | Caller's quota gets consumed |
| webhook.webhookUrl | Caller | Webhook is per-channel — caller's channel needs caller's URL |
| webhook.displayName + avatarUrl | Delegate | Visual identity in thread shows the delegate (work-doer) |

**Auto-selected:** Above split.
**Notes:** Rejected alternative — "use delegate's webhook URL too" — would attempt to post to delegate's channel from caller's channel context, which Discord webhooks don't support cross-channel.
