## Decisions Made

- **Type-name confirmation:** `Config` (not `ClawcodeConfig`); `agents` is array (not record). Adjusted walker on first write — no rework cycle. See "Type-Shape Confirmation" table above.
- **Walker uses fs/promises.readdir, not glob:** glob isn't a project dep (despite CLAUDE.md "tech stack" listing it as supporting; it's not actually installed). Used the readdir fallback explicitly listed in the plan as the no-new-deps option.
- **Sync wrapper throws on miss instead of fallback async re-resolve:** Plan-prescribed; the loader's existing onMcpResolutionError handler degrades the affected MCP gracefully. Wave 3's ConfigWatcher.onChange hook will auto-resolve newly-added op:// refs before this throw path can trigger in practice.
- **Edit B implemented as one contiguous block (lines 1510-1564) instead of three separate edits:** Keeps boot step 4a/4b visually grouped + easier for Wave 3 to navigate. Same observable behavior; just better cohesion.
- **execSync import dropped:** Was only used by the rewritten botToken block. No other code in daemon.ts shells out via execSync (verified via grep before removing).
