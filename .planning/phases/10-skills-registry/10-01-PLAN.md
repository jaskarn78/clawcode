  <name>Task 1: Create skills types and scanner</name>
  <files>src/skills/types.ts, src/skills/scanner.ts, src/skills/scanner.test.ts</files>
  <read_first>src/config/schema.ts, src/shared/types.ts</read_first>
  <behavior>
    - scanSkillsDirectory("path/to/skills") returns a SkillsCatalog Map keyed by skill name
    - Each SkillEntry has: name (string, directory name), description (string, first paragraph of SKILL.md), version (string or null, from SKILL.md YAML frontmatter), path (string, absolute directory path)
    - Directories without SKILL.md are skipped (logged as warning, not error)
    - Empty skills directory returns empty Map
    - Non-existent skills directory returns empty Map (logged as warning)
    - SKILL.md with YAML frontmatter "version: 1.2.0" extracts version; without frontmatter version is null
    - First paragraph = first non-empty, non-frontmatter line(s) before a blank line
  </behavior>
  <action>
    Create `src/skills/types.ts` (per D-04):
    - `SkillEntry` type: `{ readonly name: string; readonly description: string; readonly version: string | null; readonly path: string }`
    - `SkillsCatalog` type: `ReadonlyMap<string, SkillEntry>`

    Create `src/skills/scanner.ts` (per D-02, D-03):
    - Export `scanSkillsDirectory(skillsPath: string, log?: Logger): Promise<SkillsCatalog>`
    - Use `readdir` to list subdirectories in `skillsPath`
    - For each subdirectory, check for `SKILL.md` file existence
    - Parse SKILL.md: extract YAML frontmatter (between `---` delimiters) for `version` field; extract first paragraph after frontmatter as `description`
    - Use `readFile` (not sync) for SKILL.md content
    - Skip directories without SKILL.md (log warning)
    - If `skillsPath` does not exist, return empty Map (log warning)
    - Do NOT add any external YAML parsing dependency -- frontmatter is simple enough for regex: `/^version:\s*(.+)$/m`

    Create `src/skills/scanner.test.ts`:
    - Use vitest with tmp directories (mkdtemp)
    - Test: empty directory returns empty catalog
    - Test: directory with valid skill (SKILL.md with frontmatter) returns correct SkillEntry
    - Test: directory with SKILL.md without frontmatter returns version as null
    - Test: subdirectory without SKILL.md is skipped
    - Test: non-existent directory returns empty catalog
  </action>
  <verify>
    <automated>npx vitest run src/skills/scanner.test.ts</automated>
  </verify>
  <acceptance_criteria>
    - grep -q "export type SkillEntry" src/skills/types.ts

    - grep -q "export.*scanSkillsDirectory" src/skills/scanner.ts
