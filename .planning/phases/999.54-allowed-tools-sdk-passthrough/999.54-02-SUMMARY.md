# Phase 999.54 Plan 02: Loader cascade — spread-conditional propagation + clawcode auto-inject alwaysLoad: true Summary

## What Was Built

Two surgical edits inside `src/config/loader.ts`'s `resolveAgentConfig()`, riding on top of Plan 01's already-merged schema/type cascade:

1. **Fleet-wide leverage edit (Task 1)** — the auto-injected `clawcode` MCP server entry now carries `alwaysLoad: true`. This is the only mechanism that reaches all 15+ existing fleet agents without requiring any yaml change (RESEARCH.md Pitfall 1: a yaml-level `defaults.mcpServers.clawcode.alwaysLoad: true` would silently no-op because no production yaml declares `clawcode` in its top-level `mcpServers:` block).
2. **Resolved-construction propagation + D-05a log (Task 2)** — added the spread-conditional `...(s.alwaysLoad === true ? { alwaysLoad: true as const } : {})` sibling to `description` / `accessPattern` inside `mcpServers.push({...})`, plus a per-agent `console.info("phase999.54-resolver", JSON.stringify({...}))` log emitted right after the loop completes.

The combination lands the **mechanism behind D-03a** while preserving operator-override semantics (D-03 / RESEARCH.md OQ-3): an operator who declares `clawcode` explicitly in their per-agent `mcpServers:` block populates `resolvedMcpMap` BEFORE the `has("clawcode")` gate at line 295, so the auto-inject is skipped and the operator's entry flows through Task 2's spread instead.
