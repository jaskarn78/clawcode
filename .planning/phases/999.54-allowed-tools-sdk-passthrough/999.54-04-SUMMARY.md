## Deviations from Plan

### Task 4 — Differ classification: behavioral pin → set-membership pin

**Rule applied:** Rule 3 (auto-fix blocking issue — the pseudocode was unreachable).

**Found during:** Task 4 execution.

**Issue:** Plan 04's pseudocode for Task 4 was illustrative; line 366 explicitly invited adaptation. After writing the behavioral test (mutate `defaults.mcpServers.clawcode.alwaysLoad` true→false, expect a leaf change at that fieldPath), the test failed because the differ does NOT examine the top-level shared `mcpServers` Record:

```ts
// src/config/differ.ts:23-30
export function diffConfigs(oldConfig: Config, newConfig: Config): ConfigDiff {
  const changes: ConfigChange[] = [];
  // Diff defaults (non-agent top-level fields)
  diffObject(oldConfig.defaults, newConfig.defaults, "defaults", changes);
  // Diff agents by name
  diffAgents(oldConfig.agents, newConfig.agents, changes);
  ...
}
```

The schema also has no `defaults.mcpServers` — the `mcpServers` Record is a sibling of `defaults` at top level (`src/config/schema.ts:2376`). Additionally, the differ's `diffObject` does NOT recurse into arrays (`isPlainObject` excludes them), so `agents.*.mcpServers.*.alwaysLoad` is also unreachable as a leaf path.

**Fix:** Both of Plan 03's NON_RELOADABLE_FIELDS entries are PURE doc-of-intent — same status as the v2.5 SHARED-01 memoryPath entries and the DI8 settingSources/gsd entries that already use this idiom (`differ.test.ts:929-939`). Replaced the behavioral diff approach with `NON_RELOADABLE_FIELDS.has(...)` + `!RELOADABLE_FIELDS.has(...)` checks for both yaml path shapes. Two `it()` cases — one per shape — as required by the plan.

**Verification:** Plan 04 line 366 explicitly admits this adaptation: "Either the set-membership path (matched via wildcard expansion) or the fallthrough path (no match → false) is acceptable behavior." Set-membership pin matches the surrounding test idiom.

**Files modified:** `src/config/__tests__/differ.test.ts`.
**Commit:** `ce79220`.
