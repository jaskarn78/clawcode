<objective>
Refactor context assembly from single-blob to two-block (stablePrefix / mutableSuffix) and wire the stable block into the SDK's systemPrompt preset so Anthropic auto-caches it. Implement the hot-tier stable_token mechanism (CONTEXT D-05) so a hot-tier change doesn't cache-thrash: when the hot-tier set changes, hot-tier falls OUT of the stable prefix for that turn only, landing in the mutable suffix instead; the next turn with unchanged hot-tier brings it back into the cached prefix cleanly.

Write prefix_hash (sha256 of stablePrefix) and cache_eviction_expected (true when this turn's prefixHash differs from the same agent's prior turn) into the TurnRecord via Turn.recordCacheUsage.

Purpose: Close CACHE-01 (stable prefix carries cache scaffolding via SDK preset+append), CACHE-02 (hot-tier + skills header are inside cached prefix WHEN stable), and CACHE-04 (prefix change → prefixHash differs → cacheEvictionExpected=true → observable in telemetry).

Output: Two-block context assembly, SDK preset+append wiring, stable_token hot-tier logic, per-turn prefixHash + cacheEvictionExpected recorded on Turn.
</objective>
