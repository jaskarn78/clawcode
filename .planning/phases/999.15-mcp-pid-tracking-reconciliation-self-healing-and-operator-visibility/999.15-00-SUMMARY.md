## Decisions Made

- **Reconciler test harness** uses a fake McpProcessTracker (vi.fn-backed object) rather than a partial subclass. Reasoning: the tracker's extended API ships in Plan 01; instantiating the real class with mocked extensions would either fail compile or paper over Plan 01's actual API surface. The fake forces Plan 02's reconciler implementation to call only the locked surface.
- **`@ts-expect-error` strategy:** placed at the EXACT line of each missing call (not at the import line). Effect: tsc flips green only when Plan 01/02/03 land their respective surfaces. Pre-existing `.sort()` errors at process-tracker.test.ts:145-155 left untouched (out of scope, pre-existing).
- **session-manager polled-discovery tests** use `await import("../../mcp/proc-scan.js")` + `vi.spyOn` per test (not vi.mock at module scope) so the existing 999.14 cases above are not affected by the new spies. SessionManager teardown uses `mgr.stopAll()` (not `shutdown()` — verified against existing afterEach pattern).
- **mcp-tracker CLI fixture** matches the table format from CONTEXT.md `<specifics>` TRACK-05 spec verbatim. CLI-1 asserts header columns + body cell content (substring match) rather than full snapshot equality, so trivial whitespace tweaks during Plan 03 GREEN don't tank the test.
