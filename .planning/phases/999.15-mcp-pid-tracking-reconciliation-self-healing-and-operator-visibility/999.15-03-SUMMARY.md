## Decisions Made

1. **Pure builder in a sibling module (`mcp-tracker-snapshot.ts`)** instead of inlining into daemon.ts — the Wave 0 IPC contract tests dynamic-import `buildMcpTrackerSnapshot` directly. Inlining would force daemon.ts instantiation just to test response shape. Sibling module also keeps daemon.ts focused on wiring.
2. **Structural `SnapshotTracker` interface** — only `getRegisteredAgents` required + `getCmdlinesForAgent` optional. Both production `McpProcessTracker` and IPC-1/IPC-2 fakeTracker fixtures satisfy it without a `getCmdlinesForAgent` stub. Decoupled from the McpProcessTracker class identity.
3. **`getCmdlinesForAgent` reads the existing per-PID cmdlines Map** populated at register/replaceMcpPids time. Iterates `entry.mcpPids` in registration order and skips PIDs not yet in the cache (race-safe — enrichment is async best-effort). Returns a fresh array; immutable.
4. **Closure-intercept BEFORE `routeMethod`** mirrors Phase 104 `secrets-status` pattern — keeps the 24-arg `routeMethod` signature byte-stable. Captures `let mcpTracker` ref by closure so the handler always sees the post-construction singleton (or null → returns `{ agents: [] }` gracefully).
5. **CLI exit codes 0/1/2/3** — 0 healthy, 1 partial liveness OR explicit claudeAlive=false, 2 daemon down (handles BOTH `ManagerNotRunningError` AND raw `ECONNREFUSED`/`ENOENT` codes — test mocks bypass the IPC translator), 3 unknown agent under -a filter. Three is reserved so monitoring scripts can distinguish "agent gone" from "partial liveness".
6. **`claudeAlive` classification — only `false` flips to partial; `undefined` defaults to alive.** The builder always sets the field, so undefined only occurs in test fixtures (CLI-5 fixture omits it) or older payloads. Conservative default avoids false-positive exit 1.
7. **Verbose mode renders FULL cmdlines (no truncation)** — operator debug surface; pairs each PID with `cmdlines[i]` in registration order; missing cmdlines render as `(cmdline pending)` since enrichment may not have completed when the operator queries.
8. **CLI subcommand name LOCKED as `mcp-tracker`** — Phase 85 already owns `mcp-status`. CLI-4 actively asserts mcp-tracker.ts does NOT register mcp-status, preventing accidental rename. Both commands wired side-by-side in `src/cli/index.ts`.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] CLI-5 expected exit 0 but classifyHealth flagged the fixture as partial**

- **Found during:** Task 2 verification (`expected 1 to be +0`).
- **Issue:** CLI-5 fixture omits the `claudeAlive` field on the agent object. Initial `classifyHealth` used `if (!entry.claudeAlive) return "partial"` — `!undefined === true` flipped a healthy agent to partial. Test expected exit 0; we returned exit 1.
- **Fix:** Tightened the check to `if (entry.claudeAlive === false) return "partial"`. Only an explicit `false` flips to partial; `undefined` defaults to alive. The `buildMcpTrackerSnapshot` builder always sets `claudeAlive`, so undefined only occurs in test fixtures or older payloads. Conservative default; documented inline.
- **Files modified:** `src/cli/commands/mcp-tracker.ts` (classifyHealth function).
- **Verification:** CLI-5 GREEN; CLI-1..4 unaffected.
- **Commit:** `7896af4` (folded into Task 2 commit).
