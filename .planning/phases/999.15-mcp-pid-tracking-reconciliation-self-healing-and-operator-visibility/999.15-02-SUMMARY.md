## Reconciler Reason Classifier — Final Decisions

| Pre-state | /proc finding | Post-state | reason emitted | mutation |
|-----------|---------------|------------|----------------|----------|
| oldClaudePid alive AND oldMcpCount > 0 AND new mcpPids set-equals old | no change | no log | no mutation |
| oldClaudePid alive AND oldMcpCount === 0 AND new mcpPids non-empty | filled | `missing-mcps` | replaceMcpPids(name, newPids) |
| oldClaudePid alive AND oldMcpCount > 0 AND new mcpPids set differs | drift | `missing-mcps` | replaceMcpPids(name, newPids) |
| oldClaudePid dead AND new claude found AND oldMcpCount === 0 | new claude only | `stale-claude` | updateAgent + replaceMcpPids |
| oldClaudePid dead AND new claude found AND oldMcpCount > 0 AND mcpPids same | new claude same MCPs | `stale-claude` | updateAgent + replaceMcpPids |
| oldClaudePid dead AND new claude found AND oldMcpCount > 0 AND mcpPids differ | full restart | `agent-restart` | updateAgent + replaceMcpPids |
| oldClaudePid dead AND new claude NOT found | agent gone | `agent-gone` | unregister(name) |

Pinned by reconciler.test.ts cases 1-9. The `pidSetsDiffer` predicate uses sorted-array equality so stable-ordered duplicates ARE no-ops (Test 8: `[202, 201]` vs `[201, 202]` is no-op).

## Decisions Made

1. **Always-complete polled budget over break-on-first-found.** Completing all 6 attempts with `claudePid = lastNonNullResult` captures the SURVIVING claudePid even when the SDK respawns claude during warmup. Break-on-found would miss the surviving second-PID if the dying first-PID is found at the start of the window.
2. **Reason classifier upgrades stale-claude to agent-restart when mcpPids ALSO differs.** Distinguishes "claude got OOM-killed and re-spawned but kept its MCPs" from "operator restarted the whole agent which respawned everything". Operators can grep `reason: "agent-restart"` to audit clean restarts.
3. **getRegisteredAgents().keys() snapshot via Array.from BEFORE iteration.** Pitfall 4 in CONTEXT — JS Map iterators reflect live state; concurrent unregister (agent-gone path) AND register (TRACK-02 polled discovery for a fresh agent) during iteration would corrupt the walk.
4. **Per-agent try/catch in reconcileAllAgents.** Sibling agents continue reconciling even when one fails. Error log emits at warn with { component, action: "reconcile", agent, err: String(err) } so the err-likely heuristic in Test 5 finds it.
5. **No reconcile in killAll().** Graceful-exit determinism — /proc walks during shutdown teardown can hit timing issues. Trust the most-recent reconcile-tick state. killAgentGroup is the per-agent path that DOES reconcile.
6. **killAgentGroup reconcile failure falls back to listForAgent's recorded PIDs.** Safety net per CONTEXT — kill is best-effort cleanup; we'd rather SIGTERM stale recorded PIDs than let live MCPs leak when /proc is misbehaving.
7. **Late-bound reconcileAgent closure in daemon.** Captures `let mcpTracker` (mutable) so the closure resolves to the LIVE singleton at call time, not the null pre-construction value. Avoids bootstrap circular dep where reconciler.ts would need to import the tracker module to type its dep.
8. **onTickAfter ALWAYS-ON shape.** Even when sweepStaleBindings is disabled (idleMs=0 or no spawner), the reconciler still runs every 60s. Both calls in their own try/catch — failures never propagate to startOrphanReaper.
9. **mcpBootTimeUnix + mcpClockTicksPerSec hoisted out of inner try block.** SessionManager and onTickAfter both need them; previously they were local to the construction try and re-read at the reaper site (line ~4118 still re-reads via `bootTimeUnixForReaper` for backwards compat with 999.14 substrate).
10. **Test-only afterEach fix in session-manager.test.ts.** Wave 0 SM-{1..4} describe lacked `vi.restoreAllMocks()` so spy.mock.calls.length leaked between sequential cases (SM-3 saw SM-1+SM-2 prior calls). Added the call to fix Plan 02 GREEN — Rule 1 deviation.
