      table: string;
      idColumn: string;

    Implement `MysqlSource` class:
    - `sourceId`: `mysql:{table}` (unique per table)
    - Constructor: stores options, initializes `lastSeenId` to `0`
    - `start()`: creates `setInterval(pollIntervalMs)` calling `pollOnce()`. Call `.unref()` on the timer handle (per anti-pattern from research: "Without .unref(), daemon shutdown hangs waiting for timers").
    - `stop()`: clears interval
    - `pollOnce()`: (private async)
      1. `pool.getConnection()` — get a dedicated connection from pool
      2. Primary query: `SELECT \`${idColumn}\` AS id, * FROM \`${table}\` WHERE \`${idColumn}\` > ? ${filter ? 'AND ' + filter : ''} ORDER BY \`${idColumn}\` ASC LIMIT ?` with params `[lastSeenId, batchSize]`
      3. If no rows, release connection, return
      4. `maxId = rows[rows.length - 1].id`
      5. Committed-read confirmation: `SELECT \`${idColumn}\` AS id FROM \`${table}\` WHERE \`${idColumn}\` = ?` with `[maxId]`
      6. If confirmation returns 0 rows: log warn "mysql-source: max row disappeared (probable ROLLBACK)", do NOT advance watermark, release connection, return
      7. If confirmed: build TriggerEvent[] (sourceId, idempotencyKey=`{table}:{row.id}`, targetAgent, payload=row, timestamp=Date.now()), update `lastSeenId = maxId`, call `ingestFn(event)` for each
      8. `conn.release()` in finally block — ALWAYS release even on error
    - `poll(since)`: watermark replay — same query as pollOnce but with `parseInt(since) || 0` as the starting id. Returns events without calling ingestFn (engine does that).

    **1. Create `src/dashboard/webhook-handler.ts`:**

    Pure HTTP handler function extracted from dashboard server. Uses `node:crypto` (createHmac, timingSafeEqual).

    ```typescript
    import { createHmac, timingSafeEqual } from "node:crypto";
    import type { IncomingMessage, ServerResponse } from "node:http";
    import type { Logger } from "pino";

    export type WebhookConfig = Readonly<{
      triggerId: string;
      secret: string;
      targetAgent: string;
      maxBodyBytes: number;
    }>;

    export type WebhookIngestFn = (triggerId: string, payload: unknown) => Promise<void>;

    export function createWebhookHandler(
      configs: ReadonlyMap<string, WebhookConfig>,
      ingestFn: WebhookIngestFn,
      log: Logger,
    ): (triggerId: string, req: IncomingMessage, res: ServerResponse) => Promise<void>;
    ```

    The returned handler function:
    1. Look up triggerId in configs map. If not found, respond 404 `{ error: "Unknown trigger" }`.
    2. Read raw body as Buffer with size tracking. If accumulated length exceeds `config.maxBodyBytes` (default 65536), destroy socket, respond 413 `{ error: "Body exceeds size limit" }`.
    3. Check `x-signature-256` header. If missing, respond 401 `{ error: "Missing signature" }`.
    4. Compute HMAC-SHA256 of raw body bytes using `config.secret`. Compare with `timingSafeEqual` (convert both to Buffer, check length match first). If mismatch, respond 403 `{ error: "Invalid signature" }`.
    5. Parse body as JSON. If parse fails, respond 400 `{ error: "Invalid JSON" }`.
    6. Call `ingestFn(triggerId, parsedPayload)`.
    7. Respond 200 `{ ok: true }`.

    Helper `sendJson(res, status, data)` — import from server.ts or duplicate as a small utility. Prefer importing if server.ts exports it; otherwise duplicate (3 lines).
