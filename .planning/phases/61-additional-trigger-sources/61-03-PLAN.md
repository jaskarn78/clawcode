    // Pool is daemon-level — shared across all MysqlSource instances.
    // Created only if mysql trigger sources are configured. Pool size 2 per CONTEXT.md.
    let mysqlPool: Pool | null = null;
    const mysqlConfigs = config.triggers?.sources?.mysql ?? [];
    if (mysqlConfigs.length > 0) {
      // MySQL connection config is read from environment variables.
      // MYSQL_HOST, MYSQL_USER, MYSQL_PASSWORD, MYSQL_DATABASE must be set.
      const mysqlHost = process.env.MYSQL_HOST;
      const mysqlUser = process.env.MYSQL_USER;
      const mysqlPassword = process.env.MYSQL_PASSWORD;
      const mysqlDatabase = process.env.MYSQL_DATABASE;

      if (mysqlHost && mysqlUser && mysqlDatabase) {
        mysqlPool = createPool({
          host: mysqlHost,
          user: mysqlUser,
          password: mysqlPassword,
          database: mysqlDatabase,
          connectionLimit: 2,
          waitForConnections: true,
          enableKeepAlive: true,
        });
        log.info({ host: mysqlHost, database: mysqlDatabase }, "mysql2 pool created for trigger sources");
      } else {
        log.warn("MySQL trigger sources configured but MYSQL_HOST/MYSQL_USER/MYSQL_DATABASE env vars missing — skipping");
      }

    Update `src/heartbeat/checks/inbox.ts` to add reconciler/fallback behavior per CONTEXT.md locked decision: "InboxSource REPLACES the existing heartbeat inbox check as primary delivery path. The existing heartbeat becomes a reconciler/fallback."

    The heartbeat inbox check currently reads ALL unprocessed messages and delivers them. When InboxSource is active (watching via chokidar), this is redundant and risks racing.

    Modification approach — add a "reconciler mode" guard:

    1. Add a module-level `let inboxSourceActive = false;` flag.
    2. Export a setter: `export function setInboxSourceActive(active: boolean): void { inboxSourceActive = active; }` — called from daemon.ts after InboxSource registration.
    3. In the `execute()` function, if `inboxSourceActive` is true:
       - Still read messages (as a safety net / reconciler)
       - Log at debug level: "inbox check running in reconciler mode (InboxSource is primary)"
       - Only process messages that are OLDER than a threshold (e.g., 2 * heartbeat interval = 120s) — these are likely messages that InboxSource missed (e.g., files written while InboxSource was stopped between daemon restarts)
       - Messages newer than the threshold are left for InboxSource to handle
    4. If `inboxSourceActive` is false, behavior is unchanged (backward compatible).

    This prevents the race condition from Research Pitfall 7 where both InboxSource and heartbeat check try to process the same file simultaneously. InboxSource handles recent files (instant delivery), heartbeat handles stale files (reconciliation).
