<objective>
Change the default agent model from sonnet to haiku and implement fork-based escalation when haiku hits capability limits.

Purpose: Agents run cheaply on haiku by default. When haiku fails (3+ consecutive errors or keyword trigger), the system automatically forks a session with a more capable model, handles the task, and returns to haiku. No permanent model drift.

Output: Modified schema.ts with haiku default, new escalation.ts module with EscalationMonitor, daemon wiring, full test coverage.
</objective>
