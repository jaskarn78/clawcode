  <action>
    1. Create `src/cli/commands/schedules.ts` (per D-12):

       Follow the exact pattern from `health.ts` and `status.ts`:

       **Types:**
       ```typescript
       type ScheduleStatusEntry = {
         readonly name: string;
         readonly agentName: string;
         readonly cron: string;
         readonly enabled: boolean;
         readonly lastRun: number | null;
         readonly lastStatus: "success" | "error" | "pending";
         readonly lastError: string | null;
         readonly nextRun: number | null;
       };

       type SchedulesResponse = {
         readonly schedules: readonly ScheduleStatusEntry[];
       };
       ```

       **formatNextRun(timestamp: number | null, now?: number): string**
       - If null, return "-"
       - Otherwise compute relative time: "in Xs", "in Xm", "in Xh", "in Xd"
       - Accept optional `now` for testability (same pattern as health.ts formatTimeAgo)

       **formatSchedulesTable(data: SchedulesResponse, now?: number): string**
       - If no schedules, return "No scheduled tasks"
       - Columns: AGENT, TASK, CRON, NEXT RUN, LAST STATUS
       - Use ANSI colors: GREEN for "success", RED for "error", DIM for "pending" and "(disabled)"
       - For error status, append ` (lastError)` truncated to 40 chars
       - For disabled schedules, show "(disabled)" in DIM in the LAST STATUS column
       - Calculate column widths dynamically (same approach as status.ts)
       - Use padEnd for alignment

       **registerSchedulesCommand(program: Command): void**
       - Register `program.command("schedules").description("Show scheduled tasks and their status")`
       - Action: `sendIpcRequest(SOCKET_PATH, "schedules", {})` then `console.log(formatSchedulesTable(result))`
       - Error handling: same pattern as health.ts (ManagerNotRunningError -> helpful message, generic error -> stderr)

    2. Update `src/cli/index.ts`:
       - Add `import { registerSchedulesCommand } from "./commands/schedules.js";`
       - Add `registerSchedulesCommand(program);` after `registerHealthCommand(program);`

       - Test formatNextRun with various timestamps
       - Do NOT test the registerSchedulesCommand function (requires IPC mocking -- covered by integration)
