<objective>
Build the core OpenAI HTTP surface: request/response Zod types, bidirectional OpenAI↔Claude tool-use translator with streamed tool-call accumulator, SSE writer with keepalive + backpressure + client-disconnect, and the `node:http` server that routes `POST /v1/chat/completions` + `GET /v1/models`. This is the executable core of Phase 69. Session continuity, daemon integration, CLI, and E2E smoke come in Plan 03.

Purpose: After this plan, the OpenAI endpoint is a pure, testable module — you can stand it up in Vitest with a mocked SessionAdapter that returns pre-recorded SDK stream fixtures, and it round-trips byte-for-byte to what the Python OpenAI SDK expects. Plan 03 wires the real SessionManager + ConversationStore in.

- `src/openai/types.ts` — Zod v4 schemas for all request/response shapes (chat.completion, chat.completion.chunk, error, models list).

- `src/openai/stream.ts` — SSE writer wrapping `http.ServerResponse` with keepalive timer, backpressure (`drain` wait), [DONE] terminator, client-disconnect hook.
- `src/openai/server.ts` — `node:http` listener with routing, body parsing (size-capped), CORS, error shape, auth delegation.
- Three new test files with recorded-fixture driven unit + integration tests covering OPENAI-01, OPENAI-02, OPENAI-03, OPENAI-06.
