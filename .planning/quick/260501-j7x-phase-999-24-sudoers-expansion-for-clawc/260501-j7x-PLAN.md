<objective>
Expand `/etc/sudoers.d/clawcode` on the production clawdy server (jjagpal@100.98.211.108) with a new `CLAWCODE_SERVICE` command alias granting the `clawcode` user passwordless `systemctl reload clawcode.service` and `systemctl restart clawcode.service`. The agent fell back to direct `kill -HUP <pid>` on the daemon's own PID, and the daemon died silently from SIGHUP. Granting a legitimate, narrow reload/restart path eliminates the fallback that caused the outage.

Output: Updated sudoers file on clawdy, plus SUMMARY.md documenting the diff, pre/post `sudo -l -U clawcode` snapshots, file mode/owner verification, and rollback procedure.
