    **Step 3 — Category 4 (tuple narrowing on .mock.calls):**
    vitest 4 narrows `.mock.calls` to `[]` when no overload information is available. Fix by typing the mock at creation:

    ```ts
    // Before (fails):
    const send = vi.fn();
    send("hi");
    expect(send.mock.calls[0][0]).toBe("hi"); // calls[0] = []

    // After (passes):
    const send = vi.fn<(msg: string) => void>();
    send("hi");
    expect(send.mock.calls[0][0]).toBe("hi"); // calls[0] = [msg: string]
    ```

    Or, if the mock comes from `vi.mocked(realFn)`, the fn itself must have a non-overloaded type — look at the declaration of the real function. If the real fn has overloads, narrow at the call site:
    `const calls = vi.mocked(realFn).mock.calls as Array<[ExpectedArg1, ExpectedArg2]>;`
    The `as Array<[...]>` is acceptable here because we're recovering type info that overload-resolution discarded — this is NOT `as any`.

    Walk every file in error category 4 and apply the appropriate fix.
