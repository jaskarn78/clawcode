## The One Cross-Phase Gap

**Symptom:** `clawcode migrate openclaw verify <finmentum-agent>` exits 1 with "SOUL.md: missing".

**Root cause:**
- `src/migration/config-mapper.ts:85` — `soulFile = join(targetMemoryPath, "SOUL.md")` for all agents
- For finmentum family: `targetMemoryPath = <root>/finmentum/memory/<agentId>`
- `src/cli/commands/migrate-openclaw.ts:436` — workspace-copier copies to `targetBasePath = <root>/finmentum/`
- SOUL.md lands at `<root>/finmentum/SOUL.md`, not `<root>/finmentum/memory/<agent>/SOUL.md`
- `src/migration/verifier.ts:163` — `existsSync(resolved.soulFile)` → false → check fails

**Runtime is unaffected** because `session-config.ts` has a 3-branch lazy-read that falls through to `<workspace>/SOUL.md` when `soulFile` doesn't exist. Agents work. But `verify` doesn't know about that fallback.

**Two valid fixes (deferred to gap-closure if accepted):**
1. Change config-mapper for shared-basePath agents: `soulFile = join(targetBasePath, "SOUL.md")` so YAML matches on-disk
2. Extend workspace-copier to populate per-agent `memoryPath/SOUL.md` for each shared-basePath member

Both are small (<20 LOC + one regression test each).
