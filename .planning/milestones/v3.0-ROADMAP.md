## Success criteria for v3.0 milestone

1. **Subagent context isolation closed end-to-end** — both failure modes from 999.57 observation (2026-05-15 18:04 + parent run) regression-tested + sentinel'd via static-grep. Plans 02 + 03 merged.
2. **Plugin SDK manifest pattern lands** — every loaded skill / MCP server has a validated manifest at daemon boot. Manifest-less surfaces produce a structured warn log, not a silent load.
3. **Autonomous skill creation gated by operator review** — staged drafts live in `~/.clawcode/agents/<agent>/skills/drafts/`; operator-approve flow surfaces in Discord.
4. **`clawcode` MCP server addressable from external Claude Code session** — `delegate_task` / `ask_agent` / `task_status` callable; round-trip latency < 2s for `ask_agent`.
5. **No-useful-tokens timeout fires correctly** — synthetic stall test (keepalive-only stream for N+1 seconds) triggers turn failure; fin-acq 16-min stall pattern is structurally impossible.
6. **999.4 honest CLI/dashboard usage surface** — `clawcode usage` shows correct units and consistent utilization derive. (999.5 status fallbacks deferred 2026-05-15 — no operator-observable scope.)
7. **999.50 tmux skill installable** — `/clawcode-skills-install tmux` works; agent can use `/tmux` slash command to list/attach/inspect sessions.
8. **999.56 voice channel `/vc join` works in admin-clawdy** — agent joins channel, transcribes, generates response, speaks back. Operator-observable in a single Discord exchange.
9. **999.53 deferral disposition decided** — either Plan 94-08 lands in v3.0 window and 999.53 promotes, or 999.53 remains deferred and re-files to v3.x.
