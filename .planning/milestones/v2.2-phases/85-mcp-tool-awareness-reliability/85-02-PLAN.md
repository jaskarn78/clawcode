<objective>
Fix TOOL-02 / TOOL-05 / TOOL-07 by reshaping the MCP section of the system prompt: add the explicit "MCP tools are pre-authenticated" framing, render a live per-server tool-status table, and pin the verbatim-error instruction — all inside the v1.7 **stable cached prefix** so eviction through compaction can't strip it.

Purpose: Today `session-config.ts:289-298` renders a bare bullet list of MCP server names with raw shell commands. It has no readiness signal, no authentication framing, no error-handling rule, and (because descriptions are absent) it leaves the agent's tool-use decision-making to guesswork. The phantom-error class lives in that gap: an agent seeing `- **1password**: \`op-mcp\`` in its prompt with zero context assumes the tool might not be set up. This plan makes the prompt authoritative: tools exist, they're authenticated, here's live status, report errors verbatim.

Output: A pure `mcp-prompt-block.ts` module with the preamble + table + rule; `session-config.ts` switched to consume it; two test files pinning that the full block lands in the stable prefix (not the mutable suffix).
</objective>

    ```typescript
    import type { McpServerSchemaConfig } from "../config/schema.js";
    import type { McpServerState } from "../mcp/readiness.js";

    /**
     * Phase 85 TOOL-02 — canonical preauth framing. Exported so tests can pin
     * against a single source of truth and so other system-prompt consumers
     * (future assemblers) import the string rather than re-typing it.
     */
    export const MCP_PREAUTH_STATEMENT =
      "**MCP tools are pre-authenticated.** Every server listed below has already " +
      "completed its authentication handshake at agent startup. You do not need " +
      "to log in, unlock a vault, or prompt the user to 'set up' an MCP server. " +
      "If you need a tool from one of these servers, call it directly.";

    /**
     * Phase 85 TOOL-05 — verbatim-error rule. Pinned by a snapshot + grep
     * regression test (Task 2 integration test). Wording is deliberately
     * conservative: it forbids confabulated "misconfigured" claims UNLESS the
     * MCP server itself returned that word in its error.
     */
    export const MCP_VERBATIM_ERROR_RULE =
      "If an MCP tool reports an error, include the actual error message " +
      "verbatim; do not assume the tool is misconfigured unless the error " +
      "explicitly states misconfiguration.";

    export type RenderMcpPromptInput = {
      readonly servers: readonly McpServerSchemaConfig[];
      /**
       * Per-server readiness state. Keyed by server.name. When a server has no
       * entry, its row renders with status `unknown` (pre-heartbeat state on a
       * fresh boot before the first mcp-reconnect tick has run).
       */
      readonly stateByName: ReadonlyMap<string, McpServerState>;
    };

    /**
     * Render the MCP section of the system prompt. Output is pure markdown
     * with:
     *   1. A heading `## MCP Tools (pre-authenticated)`
     *   2. The MCP_PREAUTH_STATEMENT (TOOL-02)
     *   3. A live status table (TOOL-02 live surface)
     *   4. The MCP_VERBATIM_ERROR_RULE (TOOL-05)
     *
     * IMPORTANT: This block lands in the STABLE PREFIX via session-config's
     * `toolDefinitions` source (TOOL-07 — survives compaction).
     *
     * SECURITY: intentionally renders ONLY `{name, status, tools?, lastError.message}`.
     * NEVER includes `command`, `args`, or `env` (Pitfall 12 closure).
     */
    export function renderMcpPromptBlock(input: RenderMcpPromptInput): string {
      if (input.servers.length === 0) return "";

      const header = "## MCP Tools (pre-authenticated)";
      const preamble = MCP_PREAUTH_STATEMENT;

      const rows = input.servers.map((s) => {
        const state = input.stateByName.get(s.name);
        const baseStatus = state?.status ?? "unknown";
        const status = state?.optional && baseStatus !== "ready"
          ? `${baseStatus} (optional)`
          : baseStatus;
        // Tools column placeholder until q.mcpServerStatus() pipeline is wired
        // (out of scope for v2.2). We still pin the column so a later plan can
        // populate it without a prompt-shape change that invalidates the cache.
        const tools = "—";
        const lastErrRaw = state?.lastError?.message ?? "";
        // Escape pipes + newlines so a multi-line RPC error doesn't break the
        // markdown table. We escape, we do not truncate — TOOL-04 requires the
        // actual error message to survive verbatim.
        const lastErr = lastErrRaw
          .replace(/\|/g, "\\|")
          .replace(/\r?\n/g, " ")
          .trim();
        return `| ${s.name} | ${status} | ${tools} | ${lastErr} |`;
      });

      const table = [
        "| Server | Status | Tools | Last Error |",
        "|--------|--------|-------|------------|",
        ...rows,
      ].join("\n");

      const rule = MCP_VERBATIM_ERROR_RULE;

      return [header, preamble, table, rule].join("\n\n");
    }
    ```
