    **Constructor config:**
    ```typescript
    type SlashCommandHandlerConfig = {
      readonly client: Client;           // discord.js Client (reuse from bridge or create new, per D-13)
      readonly routingTable: RoutingTable;
      readonly sessionManager: SessionManager;
      readonly resolvedAgents: readonly ResolvedAgentConfig[];
      readonly botToken: string;
      readonly guildIds: readonly string[];  // extracted from agent channel configs
      readonly log?: Logger;
    };
    ```

    **Public methods:**

    1. `async register(): Promise<void>` — Per D-03, D-04: Use discord.js REST module to register guild-scoped commands.
       - For each guild, collect all agent commands (merge defaults per D-11 with agent custom per D-12)
       - Use `REST.put(Routes.applicationGuildCommands(clientId, guildId), { body: commands })` for bulk overwrite
       - Log the number of commands registered per guild

    2. `async start(): Promise<void>` — Register `interactionCreate` listener on the client.
       - When interaction is a ChatInputCommandInteraction:
         - Per D-07: Look up agent via `getAgentForChannel(routingTable, interaction.channelId)`
         - If no agent found, reply ephemeral: "This channel is not bound to an agent."
         - Per D-10: Call `interaction.deferReply()` immediately (allows up to 15 min for response)
         - Per D-08: Format command + args into a message using `formatCommandMessage()`
         - Per D-09, D-15: Call `sessionManager.sendToAgent(agentName, formattedMessage)`
         - Per D-10: Call `interaction.editReply()` with the response, truncated to 2000 chars
         - On error: `interaction.editReply()` with error message

    3. `async stop(): Promise<void>` — Remove interaction listener. Per D-05, leave commands registered (Discord handles stale gracefully).

    **Pure helper functions (exported for testing):**

    4. `formatCommandMessage(def: SlashCommandDef, options: ReadonlyMap<string, string | number | boolean>): string`
       - Replace `{optionName}` placeholders in claudeCommand with option values
       - Append any unmatched options as "key: value" on new lines

    5. `resolveAgentCommands(agentSlashCommands: readonly SlashCommandDef[]): readonly SlashCommandDef[]`
       - Start with DEFAULT_SLASH_COMMANDS
       - Override any matching names with agent custom commands
       - Return merged array

    **Guild ID extraction:** Extract unique guild IDs from the discord.js client cache after login (the client already has guild info from the bridge connection). Alternatively accept guildIds in config.

    **Error handling:** Wrap interaction handling in try/catch. If deferReply fails (interaction expired), log and skip. If sendToAgent fails, editReply with "Command failed: {error}". Use ephemeral responses for errors.

    Create test file testing `formatCommandMessage` and `resolveAgentCommands` pure functions.
