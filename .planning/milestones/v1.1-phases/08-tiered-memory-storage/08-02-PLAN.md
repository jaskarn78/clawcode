**Cold archival methods:**

1. `archiveToCold(entry: MemoryEntry): string` -- per research Pattern 3:
   - Get embedding via `store.getEmbedding(entry.id)` -- if null, log warning and skip
   - Build YAML frontmatter with all metadata fields including `archived_at` and `embedding_base64` (D-15)
   - Use `yaml` package's `stringify()` for frontmatter (research "Don't Hand-Roll")
   - Filename: `{id}-{slug}.md` where slug is first 40 chars, lowercased, `[^a-z0-9]+` replaced with `-`, trimmed
   - `mkdirSync(coldDir, { recursive: true })` before write (Pitfall 5)
   - Write file with `writeFileSync`
   - Delete from SQLite via `store.delete(entry.id)` (D-14 -- removes from both memories and vec_memories)
   - Return the file path

2. `rewarmFromCold(filePath: string): Promise<MemoryEntry>` -- per D-08:
   - Read the cold archive file
   - Parse YAML frontmatter to extract metadata (split on `---` delimiters, parse with `yaml` package's `parse()`)
   - Extract content from markdown body (after second `---` delimiter, skip the `# Memory:` header line)
   - Re-embed content via `embedder.embed(content)` for fresh embedding (D-08, Pitfall 6 -- always re-embed)
   - Insert into SQLite with tier='warm' using a dedicated insert that preserves `access_count` from archive metadata (Pitfall 1: start at archived access_count + 1 since the search hit counts)
   - Delete the cold archive file after successful insertion
   - Return the new MemoryEntry

**Hot tier management:**

3. `refreshHotTier(): void` -- per D-10, D-12:
   - Get all current hot memories via `store.listByTier('hot', 1000)`
   - Demote any that fail `shouldDemoteToWarm()` back to warm via `store.updateTier(id, 'warm')` (D-06)
   - Get warm candidates via `store.listByTier('warm', 100)`
   - Filter warm candidates by `shouldPromoteToHot()` (D-05)
   - If qualifying candidates exist, score them using `scoreAndRank()` (D-12) -- note: this requires building SearchResult objects with a distance of 0 (since we are not doing a vector search, just scoring by relevance)
   - Select top `hotBudget` from combined existing-hot (not demoted) + newly qualifying, update tiers accordingly
   - Log tier transitions for debugging (CONTEXT.md specific idea)

4. `getHotMemories(): readonly MemoryEntry[]` -- returns current hot tier sorted by importance desc

5. `runMaintenance(): void` -- full maintenance cycle:
   - Step 1: Demote stale hot -> warm (via refreshHotTier logic)
   - Step 2: Archive low-relevance warm -> cold (check all warm with `shouldArchiveToCold()`, archive those that qualify)
   - Step 3: Promote qualifying warm -> hot (via refreshHotTier logic)
   - Log summary: `{ demoted: N, archived: N, promoted: N }`

**Utility functions (module-level, exported for testing):**

6. `embeddingToBase64(embedding: Float32Array): string` -- per research code example
7. `base64ToEmbedding(base64: string): Float32Array` -- per research code example
8. `generateColdSlug(content: string): string` -- sanitize to `[a-z0-9-]` from first 40 chars
