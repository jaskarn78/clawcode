    - Test: downloadAttachment handles fetch failure gracefully (returns error result, does not throw)
    - Test: downloadAttachment times out after DOWNLOAD_TIMEOUT_MS (30000)
    - Test: isImageAttachment returns true for image/png, image/jpeg, image/gif, image/webp
    - Test: isImageAttachment returns false for application/pdf, text/plain, null
    - Test: cleanupAttachments removes files older than maxAgeMs from a directory
    - Test: cleanupAttachments skips files younger than maxAgeMs
  </behavior>
  <action>
    **attachment-types.ts** (per D-01, D-04, D-12):
    Create types file with:
    - `AttachmentInfo`: `{ readonly name: string; readonly url: string; readonly contentType: string | null; readonly size: number; readonly proxyUrl: string }`
    - `DownloadResult`: `{ readonly success: boolean; readonly path: string | null; readonly error: string | null; readonly attachmentInfo: AttachmentInfo }`
    - `MAX_ATTACHMENT_SIZE = 25 * 1024 * 1024` (25MB, per D-10)
    - `DOWNLOAD_TIMEOUT_MS = 30_000` (30s, per D-12)
    - `DEFAULT_CLEANUP_MAX_AGE_MS = 24 * 60 * 60 * 1000` (24h)

    **attachments.ts** (per D-02, D-03, D-04, D-09, D-11):
    - `extractAttachments(messageAttachments: Collection<string, Attachment>): readonly AttachmentInfo[]` -- converts discord.js attachments to our type
    - `isImageAttachment(contentType: string | null): boolean` -- checks if content type is image/*
    - `downloadAttachment(info: AttachmentInfo, targetDir: string): Promise<DownloadResult>` -- fetches URL with AbortController timeout, writes to `{targetDir}/{Date.now()}-{sanitized-name}`, sanitize filename (replace non-alphanumeric except dots/dashes with underscore), rejects >25MB, uses atomic write pattern (write to .tmp, rename). Per D-09, download immediately since CDN URLs expire.
    - `downloadAllAttachments(attachments: readonly AttachmentInfo[], targetDir: string, log?: Logger): Promise<readonly DownloadResult[]>` -- downloads all in parallel, logs warnings for failures and oversized files (per D-10)
    - `formatAttachmentMetadata(results: readonly DownloadResult[]): string` -- returns formatted block like:
      ```
      <attachments>
        <attachment name="photo.png" type="image/png" size="12345" local_path="/path/to/file" />
      </attachments>
      ```
      For failed downloads, include error attribute instead of local_path.
    - `cleanupAttachments(dir: string, maxAgeMs?: number): Promise<number>` -- removes files older than maxAgeMs (default 24h per D-13), returns count removed. Uses readdir + stat + unlink.

    Use `node:fs/promises` for all I/O. Use native `fetch` (Node 22) for downloads. Follow inbox.ts atomic write pattern. Import pino Logger type for optional logging.
