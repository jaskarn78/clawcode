    5. Create `src/memory/store.ts` — MemoryStore class per D-01, D-02, D-03, D-04, D-05, D-06:
       - Constructor takes dbPath (string). Opens better-sqlite3 Database.
       - Enables WAL mode: `db.pragma("journal_mode = WAL")`
       - Sets busy_timeout: `db.pragma("busy_timeout = 5000")`
       - Sets synchronous NORMAL: `db.pragma("synchronous = NORMAL")`
       - Loads sqlite-vec: `import * as sqliteVec from "sqlite-vec"; sqliteVec.load(db);`
       - Creates `memories` table with columns per D-02 (using IF NOT EXISTS)
       - Creates `session_logs` table with columns per D-03 (using IF NOT EXISTS)
       - Creates `vec_memories` virtual table: `CREATE VIRTUAL TABLE IF NOT EXISTS vec_memories USING vec0(memory_id TEXT PRIMARY KEY, embedding float[384] distance_metric=cosine)`
       - Use prepared statements for all operations (better-sqlite3 best practice)
       - Methods:
         - `insert(input: CreateMemoryInput, embedding: Float32Array): MemoryEntry` — generates nanoid ID, inserts into BOTH memories AND vec_memories in a transaction, returns the created entry
         - `getById(id: string): MemoryEntry | null` — retrieves by ID, increments access_count and updates accessed_at per D-20, D-21
         - `delete(id: string): boolean` — deletes from BOTH memories AND vec_memories in a transaction
         - `listRecent(limit: number): readonly MemoryEntry[]` — ordered by created_at DESC
         - `recordSessionLog(entry: Omit<SessionLogEntry, 'id' | 'createdAt'>): SessionLogEntry` — inserts into session_logs table
         - `close(): void` — closes the database connection
       - All methods return NEW readonly objects (never mutate)
       - Wrap all operations in try/catch, throw MemoryError with dbPath context

    1. Create `src/memory/embedder.ts` — EmbeddingService per D-07, D-08, D-09:
       - Class with private `pipeline: FeatureExtractionPipeline | null`
       - `async warmup(): Promise<void>` — creates pipeline with `pipeline("feature-extraction", "Xenova/all-MiniLM-L6-v2")` from `@huggingface/transformers`. Idempotent (only creates once). Per D-09.
       - `async embed(text: string): Promise<Float32Array>` — calls warmup if not ready, runs text through pipeline with `{ pooling: "mean", normalize: true }`, converts output.tolist()[0] to Float32Array (384 dimensions). Per D-08.
       - `isReady(): boolean` — returns whether pipeline is warmed up
       - Throw EmbeddingError on failures with descriptive messages
       - Truncate input text to ~200 words before embedding (research note: model handles max 256 tokens)

    2. Create `src/memory/search.ts` — SemanticSearch per D-10:
       - Constructor takes a better-sqlite3 Database instance
       - Prepares the KNN search statement using vec0 MATCH syntax:
         ```sql
         SELECT m.id, m.content, m.source, m.importance, m.access_count,
                m.tags, m.created_at, m.updated_at, m.accessed_at, v.distance
         FROM vec_memories v
         INNER JOIN memories m ON m.id = v.memory_id
         WHERE v.embedding MATCH ?
           AND k = ?
         ORDER BY v.distance
         ```
       - `search(queryEmbedding: Float32Array, topK: number): readonly SearchResult[]` — runs KNN query, updates access_count and accessed_at for each result per D-20/D-21, returns SearchResult array
       - Access update uses prepared statement: `UPDATE memories SET access_count = access_count + 1, accessed_at = ? WHERE id = ?`
       - All returned objects are new readonly objects

    3. Create `src/memory/session-log.ts` — SessionLogger per D-11, D-12, D-13, D-14:
       - Constructor takes memoryDir (string) — the `{workspace}/memory/` directory path
       - `async appendEntry(timestamp: string, role: 'user' | 'assistant', content: string): Promise<void>` — appends to `{memoryDir}/YYYY-MM-DD.md` where date is extracted from timestamp. Creates file with header `# Session Log: YYYY-MM-DD\n` if it doesn't exist. Entry format per D-12:
         ```
         ## HH:MM:SS [role]
         content
         ```
       - `async flushConversation(entries: ReadonlyArray<{timestamp: string, role: 'user' | 'assistant', content: string}>): Promise<string>` — appends all entries to today's log file, returns the file path. Per D-13.
       - Uses `node:fs/promises` appendFile/writeFile. No streams needed (daily logs are small).
       - Returns the file path for session_logs table tracking.
