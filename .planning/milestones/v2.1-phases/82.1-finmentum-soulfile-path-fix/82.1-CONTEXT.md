# Phase 82.1: Finmentum soulFile/identityFile Path Fix - Context

**Gathered:** 2026-04-21
**Status:** Ready for planning (gap closure for v2.1 milestone audit finding)
**Mode:** Gap closure

<domain>
## Phase Boundary

Close the single cross-phase wiring gap identified by the v2.1 milestone audit: `config-mapper.ts` writes `soulFile: <root>/finmentum/memory/<agent>/SOUL.md` to `clawcode.yaml` for the 5 finmentum-family agents, but `workspace-copier` places the actual SOUL.md (and IDENTITY.md) at `<root>/finmentum/SOUL.md` (shared root). The YAML pointer doesn't match on-disk; `verify` subcommand fails for these agents even though runtime works via session-config's lazy fallback.

Fully closes **CONF-01**, **WORK-02**, and **MIGR-04** which are currently `partial` in REQUIREMENTS.md.

</domain>

<decisions>
## Implementation Decisions

### Fix Strategy: Align YAML pointer with on-disk location
- **Chosen:** Modify `src/migration/config-mapper.ts` — for shared-basePath agents (finmentum family), emit `soulFile: join(targetBasePath, "SOUL.md")` and `identityFile: join(targetBasePath, "IDENTITY.md")` instead of `join(targetMemoryPath, ...)`. Dedicated-workspace agents stay unchanged (`targetWorkspace/SOUL.md`).
- **Rejected alternative:** Extend workspace-copier to duplicate SOUL.md/IDENTITY.md into each per-agent memoryPath subdir. This would diverge copies (5x writes for the same content) and create a maintenance trap if any agent edits SOUL.md in one location.
- **Detection of "shared-basePath":** Use existing `isFinmentumFamily(agentId)` helper from Phase 76 — no new logic needed.

### Regression Test
- Unit test in `src/migration/__tests__/config-mapper.test.ts` — for each finmentum-family agent, assert `mapped.soulFile === join(targetBasePath, "SOUL.md")` and same for identityFile.
- Integration test extension in `src/cli/commands/__tests__/migrate-openclaw.test.ts` — run full apply for a finmentum member, then run `verify`, assert exit code 0 and `workspace-files-present` check passes for SOUL.md + IDENTITY.md.
- Must NOT regress the 10 dedicated-workspace agent path (soulFile still resolves to `<workspace>/SOUL.md`).

### Acceptance
- `clawcode migrate openclaw verify finmentum-content-creator` exits 0 with all 4 checks passing (was: exits 1 on SOUL.md missing)
- Same for fin-acquisition/fin-research/fin-playground/fin-tax
- Dedicated-workspace agents unaffected
- No new npm deps

### Claude's Discretion
- Exact test assertions
- Whether to also update an existing Phase 78 test fixture or add new ones
