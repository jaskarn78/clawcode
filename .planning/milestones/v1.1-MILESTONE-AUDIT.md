## Cross-Phase Integration

### Critical Bugs (Fix Required)

#### 1. IPC Protocol Missing Methods — `threads` and `usage`

**Severity:** CRITICAL (runtime failure)
**File:** `src/ipc/protocol.ts` — `IPC_METHODS` Zod enum
**Impact:** `clawcode threads` and `clawcode usage` CLI commands return `-32600 Invalid Request`

The IPC server validates method names against `IPC_METHODS` before routing. `"threads"` and `"usage"` are implemented in `daemon.ts` `routeMethod()` but absent from the enum, making them unreachable.

**Fix:** Add `"threads"` and `"usage"` to `IPC_METHODS` array in `protocol.ts`.

#### 2. WebhookManager Not Wired to Message Delivery

**Severity:** HIGH (feature non-functional)
**Files:** `src/discord/bridge.ts`, `src/discord/webhook-manager.ts`
**Impact:** WHID-01 through WHID-04 — agents cannot post with custom display names/avatars

`WebhookManager` is initialized in daemon, queryable via IPC/CLI, but `DiscordBridge.sendResponse()` only uses `channel.send()` (bot identity). `WebhookManager.send()` is never called.

**Fix:** Wire `WebhookManager` into `DiscordBridge` constructor; use `webhookManager.send()` when webhook is configured for the agent.

#### 3. TierManager.runMaintenance() Never Called

**Severity:** MEDIUM (feature partially non-functional)
**File:** `src/memory/tier-manager.ts`
**Impact:** AMEM-08, AMEM-09 — warm-to-cold archival never triggers; cold tier stays empty; `rewarmFromCold()` is dead code

`refreshHotTier()` runs at agent start only. No heartbeat check calls `runMaintenance()`.

**Fix:** Add a heartbeat check module (similar to consolidation check) that calls `tierManager.runMaintenance()` periodically.
