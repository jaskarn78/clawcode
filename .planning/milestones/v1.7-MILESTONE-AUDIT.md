### TECH DEBT: TOOL-01 Orphaned Helper

`runWithConcurrencyLimit` exported from `src/mcp/tool-dispatch.ts` but never invoked in production code. `perf.tools.maxConcurrent` config field is parsed + Zod-validated but has no runtime effect. The TOOL-01 parallelism property holds because the Claude Agent SDK dispatches tool_use blocks in parallel by default, but the advertised control surface is non-functional.

**Impact:** User setting `maxConcurrent: 1` to serialize for debugging would see no behavior change.

**Recommended fix:** Either (a) wire `runWithConcurrencyLimit` into the MCP server tool-dispatch path, or (b) remove `perf.tools.maxConcurrent` from schema + docs.

**Severity:** tech_debt — not blocking milestone, but actively misleads operators.
