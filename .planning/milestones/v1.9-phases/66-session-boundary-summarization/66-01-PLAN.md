<objective>
Complete the CONV-03 write path so CreateMemoryInput can carry sourceTurnIds and MemoryStore.insert() persists them atomically with the memory row. Phase 64 only added the column and read path; the insert path still hard-codes source_turn_ids=null. This plan closes that gap so Plan 02's SessionSummarizer can record conversation-turn lineage on every summary memory.

Purpose: SESS-04 requires session summaries be stored as standard MemoryEntry objects. Lineage back to source turns (CONV-03) must be written in the same atomic transaction as the memory row — otherwise a race between insert and a UPDATE leaves a briefly-null lineage that breaks downstream consumers. This plan is a surgical 3-file change (types, store, test) that makes insert() honor the sourceTurnIds input.

Output: src/memory/types.ts (CreateMemoryInput extended), src/memory/store.ts (insertMemory statement + insert body updated), src/memory/__tests__/store.test.ts (roundtrip regression test).
</objective>
