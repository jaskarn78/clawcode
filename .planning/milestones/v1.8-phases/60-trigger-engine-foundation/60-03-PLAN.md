    NOTE: SchedulerSource does NOT wrap TaskScheduler as a black box. Instead, it REPLACES the TaskScheduler's role for prompt-based schedules by creating its own cron jobs that route through `ingest()`. Handler-based schedules (memory consolidation, etc.) still use TaskScheduler directly because they don't produce agent turns and don't need dedup/causation.

    The approach:
    - SchedulerSource creates Cron jobs for prompt-based schedules only.
    - When a cron fires, it calls `this.options.ingest(event)` with the schedule as a TriggerEvent.
    - Handler-based schedules are added to a separate TaskScheduler instance for backward-compatible execution.
    - The existing TaskScheduler class is NOT modified — SchedulerSource delegates handler-based schedules to it.

    Class `SchedulerSource` implementing `TriggerSource`:

    ```typescript
    export class SchedulerSource implements TriggerSource {
      readonly sourceId = "scheduler" as const;
      private readonly cronJobs: Cron[] = [];
      private readonly scheduleEntries: Array<{ agentName: string; entry: ScheduleEntry }> = [];
      private readonly ingestFn: (event: TriggerEvent) => Promise<void>;
      private readonly log: Logger;
      private readonly locks = new Map<string, boolean>();
    ```

    Constructor stores options. Iterates `resolvedAgents`, for each agent's enabled schedules that have a `prompt` (not handler-only), stores `{ agentName, entry }` in `scheduleEntries`.

    `start()`: Creates Cron jobs for each scheduleEntry. On each cron fire:
    1. Check per-agent lock (sequential execution per agent — mirrors TaskScheduler pattern).
    2. If locked, log info + skip.
    3. Set lock.
    4. Build TriggerEvent:
       ```typescript
       const event: TriggerEvent = {
         sourceId: "scheduler",
         idempotencyKey: `${entry.name}:${Date.now()}`,
         targetAgent: agentName,
         payload: entry.prompt,
         timestamp: Date.now(),
       };
       ```
    5. Call `await this.ingestFn(event)`.
    6. Release lock in finally block.
    Store each Cron in `this.cronJobs`.

    `stop()`: Stop all Cron jobs (`job.stop()` for each). Clear cronJobs array.

    `async poll(since: string | null): Promise<readonly TriggerEvent[]>`:
    Compute missed cron ticks between `since` (epoch string or null) and now for each scheduleEntry:
    1. Parse `since` as integer (or use 0 if null).
    2. For each schedule with a prompt:
       a. Create a temporary `new Cron(entry.cron)`.
       b. Iterate: compute next run from `sinceDate`, collect all ticks until past `now`.
       c. For each tick, create TriggerEvent with `idempotencyKey: ${entry.name}:${tickEpoch}`.
       d. Stop the temporary Cron after iteration.
    3. Return collected events sorted by timestamp.

    IMPORTANT: Use `Cron.nextRun(startDate)` or `Cron.nextRuns(count, startDate)` to compute missed ticks. The croner API uses `nextRun()` which returns the next Date after the internal reference point. Create a new Cron for each schedule with `{ paused: true }` and use its `nextRuns()` method with a reasonable count limit (e.g., 1000 ticks max per schedule to prevent runaway iteration).

    **3. Create `src/heartbeat/checks/task-retention.ts`:**

    Follow the trace-retention.ts pattern exactly:

    ```typescript
    import { subDays } from "date-fns";
    import type { CheckContext, CheckModule, CheckResult } from "../types.js";

    const DEFAULT_RETENTION_DAYS = 7;
    const DEFAULT_REPLAY_MAX_AGE_MS = 86_400_000; // 24h

    const taskRetentionCheck: CheckModule = {
      name: "task-retention",
      interval: 3600, // Run every hour, not every heartbeat tick

      async execute(context: CheckContext): Promise<CheckResult> {
        const { taskStore, sessionManager, agentName } = context;

        // Guard: taskStore not injected (pre-Phase 60 daemon or test stub)
        if (!taskStore || typeof taskStore.purgeCompleted !== "function") {
          return { status: "healthy", message: "No task store" };
        }

        // Only run on the FIRST agent tick per heartbeat cycle to avoid
        // redundant purges (tasks.db is daemon-scoped, not per-agent).
        // The check is idempotent so duplicates are harmless, just wasteful.
        const runningAgents = sessionManager.getRunningAgents();
        if (runningAgents.length > 0 && runningAgents[0] !== agentName) {
          return { status: "healthy", message: "Skipped (not first agent)" };
        }

        const agentConfig = sessionManager.getAgentConfig(agentName);
        const retentionDays = agentConfig?.perf?.taskRetentionDays ?? DEFAULT_RETENTION_DAYS;
        const cutoffDate = subDays(new Date(), retentionDays);
        const cutoffMs = cutoffDate.getTime();

        const deletedTasks = taskStore.purgeCompleted(cutoffMs);

        // Also purge trigger_events older than 2x replayMaxAgeMs (default 48h)
        const replayMaxAgeMs = DEFAULT_REPLAY_MAX_AGE_MS; // TODO: read from config when available
        const triggerCutoffMs = Date.now() - (2 * replayMaxAgeMs);
        const deletedTriggerEvents = taskStore.purgeTriggerEvents(triggerCutoffMs);

        const msg = deletedTasks > 0 || deletedTriggerEvents > 0
          ? `Purged ${deletedTasks} task(s), ${deletedTriggerEvents} trigger event(s)`
          : "No expired rows";

        return {
          status: "healthy",
          message: msg,
          metadata: { deletedTasks, deletedTriggerEvents, retentionDays, cutoffMs },
        };
      },
    };

    export default taskRetentionCheck;
    ```
