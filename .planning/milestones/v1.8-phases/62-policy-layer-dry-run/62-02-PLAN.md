    Create `src/triggers/policy-watcher.ts` following the `ConfigWatcher` pattern from `src/config/watcher.ts`.

    PolicyWatcherOptions type:
    ```typescript
    export type PolicyWatcherOptions = {
      readonly policyPath: string;           // .clawcode/policies.yaml
      readonly auditPath: string;            // .clawcode/policy-audit.jsonl
      readonly onReload: (evaluator: PolicyEvaluator, diff: PolicyDiff) => void;
      readonly onError?: (error: Error) => void;
      readonly log: pino.Logger;
      readonly debounceMs?: number;          // default 500
      readonly configuredAgents?: ReadonlySet<string>;
    };
    ```

    PolicyWatcher class:
    - `start(): Promise<PolicyEvaluator>` — reads policies.yaml, calls loadPolicies(), creates PolicyEvaluator, starts chokidar watch. Returns the initial evaluator. THROWS on invalid policy (boot rejection per POL-01, Pitfall 5 from RESEARCH.md). If file doesn't exist, start with empty rules (no policy file = no rules = deny all non-default events).
    - `stop(): Promise<void>` — close watcher, clear timer
    - `getCurrentEvaluator(): PolicyEvaluator` — returns current evaluator

    On chokidar "change" event:
    1. Debounce (setTimeout, clear on rapid changes, .unref() the timer)
    2. Read file content (fs.readFile)
    3. Call loadPolicies(content) in try/catch
    4. On success: diffPolicies(oldRules, newRules), write success audit entry, call onReload callback
    5. On failure: log error ("WARN: policies.yaml reload failed -- keeping previous policy" per Pitfall 5), write error audit entry, call onError callback if provided

    JSONL audit trail — write directly (don't reuse AuditTrail class since the shape differs):
    ```typescript
    type PolicyAuditEntry = {
      timestamp: string;       // ISO 8601
      action: "reload" | "boot";
      diff: PolicyDiff | null;
      status: "success" | "error";
      error?: string;
    };
    ```
    Append one JSON line per event to auditPath. Create parent directory on first write (mkdir recursive).

    **Key error message distinction (Pitfall 5):**
    - Boot: throw `PolicyValidationError` with message "FATAL: policies.yaml invalid -- daemon cannot start: ${zodIssues}"
    - Hot-reload: log.warn "policies.yaml reload failed -- keeping previous policy: ${error.message}"

    Test file: Use `vi.mock("chokidar")` or create temp files in `os.tmpdir()` for integration-style tests. Test the reload logic by calling internal methods directly where chokidar mocking is cumbersome.
