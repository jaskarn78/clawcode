    Define exported pure functions:
    - `formatTokenCount(n: number): string` — if n < 1000 return `${n}`, if n < 1_000_000 return `${(n/1000).toFixed(1)}K`, else `${(n/1_000_000).toFixed(1)}M`. Export this for reuse by tasks-list.
    - `formatDuration(startedAt: number, endedAt: number | null): string` — if endedAt is null return "running", else compute diff in ms. If < 1000 return `${ms}ms`, if < 60000 return `${(ms/1000).toFixed(1)}s`, else `${(ms/60000).toFixed(1)}m`.
    - `queryTriggerFires(opts: { dbPath: string; sinceMs: number; source?: string; agent?: string }): readonly TriggerFireRow[]` — opens tasks.db read-only, uses a temporal proximity LEFT JOIN to correlate trigger_events with their resulting tasks.

    **Join strategy (temporal proximity):** The trigger_events table has no causation_id column. The link between a trigger_event and the task it spawned is indirect (the causation_id is generated at TriggerEngine.ingest, stored on the task row via TurnOrigin, but NOT on trigger_events). Use temporal proximity:

    ```sql
    SELECT
      te.created_at,
      te.source_id,
      te.source_kind,
      te.idempotency_key,
      t.target_agent,
      t.status AS task_status,
      t.started_at AS task_started_at,
      t.ended_at AS task_ended_at
    FROM trigger_events te
    LEFT JOIN tasks t ON t.started_at BETWEEN te.created_at - 1000 AND te.created_at + 10000
    WHERE te.created_at > ?
    ORDER BY te.created_at DESC
    ```

    If --source filter: add `AND te.source_id = ?`
    If --agent filter: add `AND t.target_agent = ?`

    For duplicate matches (multiple tasks in the window), use GROUP BY te.rowid and take MIN(ABS(t.started_at - te.created_at)). This is imprecise for high-frequency overlapping triggers but acceptable for a v1 CLI display. Document this limitation in a code comment.

    7. `formatTriggersTable(rows)` — same pattern as formatDryRunTable: calculate column widths, pad, color-code the Result column (green for complete, red for failed/timed_out/cancelled/orphaned, yellow for running/pending/awaiting_input).
