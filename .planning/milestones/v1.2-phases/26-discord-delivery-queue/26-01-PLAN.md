  <behavior>
    - enqueue(agentName, channelId, content) persists a pending entry and returns its id
    - processNext() picks the oldest pending/retryable entry, calls deliverFn, marks delivered on success
    - processNext() on delivery failure increments attempts, sets lastError, computes nextRetryAt with exponential backoff (baseDelayMs * 2^attempts, capped at maxDelayMs)
    - After maxAttempts (3) failures, entry status becomes 'failed' permanently
    - getStats() returns counts of pending, inFlight, failed, delivered, totalEnqueued
    - getFailedEntries(limit?) returns the most recent permanently failed entries
    - start() begins a setInterval loop calling processNext at processingIntervalMs
    - stop() clears the interval
    - Exponential backoff: delay = min(baseDelayMs * 2^attempts, maxDelayMs)
  </behavior>
  <action>
    Create src/discord/delivery-queue.ts implementing the DeliveryQueue class:

    Constructor: `(config: { db: Database; deliverFn: DeliverFn; config?: Partial<DeliveryQueueConfig>; log?: Logger })`

    SQLite table `delivery_queue`:
    - id TEXT PRIMARY KEY
    - agent_name TEXT NOT NULL
    - channel_id TEXT NOT NULL
    - content TEXT NOT NULL
    - status TEXT NOT NULL DEFAULT 'pending' (CHECK IN ('pending','in_flight','failed','delivered'))
    - attempts INTEGER NOT NULL DEFAULT 0
    - max_attempts INTEGER NOT NULL DEFAULT 3
    - created_at TEXT NOT NULL
    - last_attempt_at TEXT
    - next_retry_at TEXT
    - last_error TEXT
    - delivered_at TEXT
    - INDEX on (status, next_retry_at) for efficient queue polling

    Methods:
    - `enqueue(agentName, channelId, content): string` — insert pending row, return nanoid
    - `processNext(): boolean` — SELECT oldest where status='pending' OR (status='in_flight' AND next_retry_at <= now), UPDATE to in_flight, call deliverFn. On success: status='delivered', deliveredAt=now. On error: attempts++, lastError=err.message, if attempts >= maxAttempts then status='failed' else status='pending' with nextRetryAt = now + min(baseDelayMs * 2^attempts, maxDelayMs). Return true if processed an entry.
    - `getStats(): DeliveryStats` — COUNT queries grouped by status
    - `getFailedEntries(limit = 50): readonly DeliveryEntry[]` — SELECT where status='failed' ORDER BY created_at DESC LIMIT
    - `start(): void` — setInterval calling processNext
    - `stop(): void` — clearInterval

    Use better-sqlite3 directly (same pattern as memory store). Use pino logger. Use nanoid for IDs.
