    - Skills changes: re-link skills for affected agents via linkAgentSkills
    - Webhook changes: rebuild webhook identities and update WebhookManager
    - Only subsystems with actual changes get updated (no unnecessary rebuilds)
    - Session manager gets updated agent configs (setAllAgentConfigs)
    - Returns a summary of what was reloaded for logging

       ```
       export class ConfigReloader {
         constructor(opts: {
           sessionManager: SessionManager;
           taskScheduler: TaskScheduler;
           heartbeatRunner: HeartbeatRunner;
           webhookManager: WebhookManager;
           skillsCatalog: SkillsCatalog;
           routingTableRef: { current: RoutingTable };  // mutable ref so daemon can read updated table
           log: pino.Logger;
         })

         async applyChanges(diff: ConfigDiff, newResolvedAgents: readonly ResolvedAgentConfig[]): Promise<ReloadSummary>
       }
       ```

         - Any change with fieldPath containing "webhook" -> rebuild webhook identities
       - Always call sessionManager.setAllAgentConfigs(newResolvedAgents) when there are reloadable changes
       - ReloadSummary type: { subsystemsReloaded: string[]; agentsAffected: string[] }

       ```typescript
       // 11c. Initialize config hot-reload
       const auditTrailPath = join(MANAGER_DIR, "config-audit.jsonl");
       const routingTableRef = { current: routingTable };

       const configReloader = new ConfigReloader({
         sessionManager: manager,
         taskScheduler,
         heartbeatRunner,
         webhookManager,
         skillsCatalog,
         routingTableRef,
         log,
       });

       const configWatcher = new ConfigWatcher({
         configPath,
         auditTrailPath,
         onChange: async (diff, newResolvedAgents) => {
           const summary = await configReloader.applyChanges(diff, newResolvedAgents);
           log.info({ subsystems: summary.subsystemsReloaded, agents: summary.agentsAffected }, "config hot-reloaded");
         },
         log,
       });
       await configWatcher.start();
       log.info({ configPath, auditTrail: auditTrailPath }, "config watcher started");
       ```

    2. Update the shutdown function to stop the config watcher:
