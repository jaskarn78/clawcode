  <name>Task 2: Config file watcher with debounce</name>
  <files>src/config/watcher.ts, src/config/__tests__/watcher.test.ts, package.json</files>
  <read_first>src/config/types.ts, src/config/differ.ts, src/config/audit-trail.ts, src/config/loader.ts, src/shared/logger.ts</read_first>
  <behavior>
    - ConfigWatcher watches a config file path using chokidar
    - On file change, debounces 500ms then loads+validates new config via loadConfig
    - If validation fails, logs error and keeps old config (does not crash)
    - If validation succeeds, calls diffConfigs(oldConfig, newConfig)
    - If changes exist, records to audit trail and calls onChange callback with ConfigDiff and new resolved agents
    - Non-reloadable changes log a warning per changed field
    - start() begins watching, stop() closes the watcher
    - Multiple rapid changes within debounce window only trigger one reload
  </behavior>
  <action>
    1. Install chokidar: run `npm install chokidar` (already in CLAUDE.md recommended stack)

    2. Create src/config/watcher.ts:
       ```
       export class ConfigWatcher {
         constructor(opts: {
           configPath: string;
           auditTrailPath: string;
           onChange: (diff: ConfigDiff, resolvedAgents: ResolvedAgentConfig[]) => Promise<void>;
           log: pino.Logger;
         })
         async start(): Promise<void>  // loads initial config, starts chokidar watch
         async stop(): Promise<void>   // closes chokidar watcher
         getCurrentConfig(): Config    // returns last valid config
       }
       ```
       - Uses chokidar.watch(configPath, { persistent: true, ignoreInitial: true })
       - On 'change' event, sets a debounce timer (500ms). If timer already set, clears and resets.
       - When debounce fires: loadConfig(configPath), on success -> diffConfigs -> if changes -> audit.record -> call onChange callback
       - On loadConfig failure: log.error with validation details, do NOT update stored config
       - For non-reloadable changes: log.warn("Config field '{fieldPath}' changed but requires daemon restart to take effect")
       - Stores currentConfig internally, updates only on successful reload

    3. Write watcher.test.ts:
       - Test debounce: write file twice quickly, onChange called once
       - Test validation failure: write invalid YAML, no crash, old config preserved
       - Test non-reloadable warning: change model field, warning logged
       - Test happy path: change channels, onChange called with correct diff
       - Use tmp directory with real file writes for integration-style tests
       - Mock the onChange callback to verify it receives correct ConfigDiff
  </action>
