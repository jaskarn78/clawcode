  <behavior>
    ACL parser:
    - parseSecurityMd with "## Channel ACLs\n- channel: 123\n  users: [alice, bob]" returns [{ channelId: "123", allowedUserIds: ["alice", "bob"], allowedRoles: [] }]
    - parseSecurityMd with empty/missing file returns empty array
    - parseSecurityMd with "## Channel ACLs\n- channel: 456\n  roles: [admin, mod]" returns roles populated
    - checkChannelAccess("123", "alice", [], acls) returns true
    - checkChannelAccess("123", "eve", [], acls) returns false
    - checkChannelAccess for channel not in ACLs returns true (open by default)

    Approval log:
    - ApprovalLog.record(entry) appends JSONL line to file
    - ApprovalLog.loadAllowAlways(agentName) returns previously persisted allow-always patterns
    - ApprovalLog.recordAllowAlways(agentName, pattern) persists and is retrievable
  </behavior>
  <action>
    1. Create src/security/acl-parser.ts:
       - parseSecurityMd(filePath: string): Promise<readonly ChannelAcl[]>
         Reads a SECURITY.md file from the agent's workspace. Parses a simple YAML-in-markdown format:
         ```
         ## Channel ACLs
         - channel: <channelId>
           users: [userId1, userId2]
           roles: [roleName1]
         ```
         Uses regex parsing (not a full YAML parser) to extract channel blocks.
         Returns empty array if file missing or no ACL section found.

       - checkChannelAccess(channelId: string, userId: string, userRoles: readonly string[], acls: readonly ChannelAcl[]): boolean
         Returns true if: (a) channelId has no ACL entry (open by default), or (b) userId is in allowedUserIds, or (c) any of userRoles is in allowedRoles.

    2. Create src/security/approval-log.ts:
       - class ApprovalLog (follows AuditTrail pattern from config/audit-trail.ts):
         - constructor(opts: { filePath: string, log: pino.Logger })
         - record(entry: ApprovalAuditEntry): Promise<void> — appends JSONL line
         - loadAllowAlways(agentName: string): string[] — reads file, filters for allow-always decisions for this agent, returns patterns
         - recordAllowAlways(agentName: string, pattern: string, approvedBy: string): Promise<void> — records an allow-always entry

       Uses the same ensureDirectory + appendFile pattern as AuditTrail.
       The allow-always entries are stored as regular audit entries with decision="allow-always" and the pattern in the command field.

    3. Create test files for both modules using vitest.
       For acl-parser tests: use tmp files with test SECURITY.md content.
       For approval-log tests: use tmp directory, write entries, verify JSONL content and loadAllowAlways filtering.
