## Milestone Goal

Production-ready agent operations (auto-start, systemd, control commands, webhook auto-provisioning) and document RAG capabilities for agent knowledge augmentation.

## Key Accomplishments

1. **Production deployment hardening** — systemd integration (Phase 43), auto-start on daemon boot (Phase 42), webhook auto-provisioning on startup (Phase 48)
2. **Agent-to-agent Discord communication** — MCP tool + webhook embeds + bridge routing for visible, auditable inter-agent messaging (Phase 44)
3. **Memory intelligence upgrades** — eager auto-linking on memory save (Phase 45), cron-scheduled consolidation via TaskScheduler (Phase 46)
4. **Operator control surface** — Discord slash commands for fleet management: /clawcode-start, /clawcode-stop, /clawcode-restart, /clawcode-fleet (Phase 47)
5. **Document RAG** — text/markdown/PDF ingestion, word-based chunking with overlap, per-agent DocumentStore with sqlite-vec KNN search, 4 MCP tools (Phase 49)

### Phase 42: Auto-start agents on daemon boot
**Goal:** Agents boot automatically when the daemon starts; no separate IPC start-all call needed.
**Requirements:** (none formal — infrastructure)
**Plans:** 1/1 complete

### Phase 43: Systemd production integration
**Goal:** Fix systemd unit template for reliable production starts with correct ExecStart, PATH, and env var loading.
**Requirements:** SYSINT-01, SYSINT-02, SYSINT-03
**Plans:** 1/1 complete

### Phase 44: Agent-to-agent Discord communication
**Goal:** Agents send visible, auditable messages to each other through Discord via MCP tool posting webhook embeds to target agent channels.
**Requirements:** A2A-01, A2A-02, A2A-03, A2A-04, A2A-05, A2A-06
**Plans:** 2/2 complete

### Phase 45: Memory auto-linking on save
**Goal:** New/updated memories get graph edges to similar memories immediately on save instead of waiting for the 6-hour heartbeat cycle.
**Requirements:** AUTOLINK-01
**Plans:** 1/1 complete

### Phase 46: Scheduled memory consolidation
**Goal:** Memory consolidation runs on a configurable cron schedule per agent via TaskScheduler instead of the fixed 24h heartbeat check.
**Requirements:** CONSOL-01
**Plans:** 1/1 complete

### Phase 47: Discord slash commands for control
**Goal:** Operators can manage the agent fleet via Discord slash commands (start, stop, restart, fleet status) that bypass agent sessions and go directly to the daemon via IPC.
**Requirements:** CTRL-01, CTRL-02, CTRL-03, CTRL-04
**Plans:** 1/1 complete

### Phase 48: Webhook identity per agent
**Goal:** Auto-provision Discord webhooks for each agent's bound channel on daemon startup, eliminating manual webhookUrl configuration.
**Requirements:** WEBHOOK-AUTO-01
**Plans:** 1/1 complete

### Phase 49: RAG over documents
**Goal:** Agents can ingest documents (text, markdown, PDF), chunk and embed them, then search over chunks using semantic similarity via MCP tools.
**Requirements:** RAG-CHUNK, RAG-STORE, RAG-PDF, RAG-INGEST, RAG-SEARCH, RAG-DELETE
**Plans:** 2/2 complete
