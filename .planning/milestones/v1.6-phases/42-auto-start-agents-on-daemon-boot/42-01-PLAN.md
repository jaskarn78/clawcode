# Plan 42-01: Auto-start agents on daemon boot

## Goal

Agents boot automatically when the daemon starts. The `start-all` CLI spawns the daemon process and waits for it to become responsive, then prints the status table. No separate IPC `start-all` request is needed from the CLI.

## Current State

- `startDaemon()` already contains the auto-start block (lines 605-613 in daemon.ts): a `void` async IIFE that calls `manager.startAll(resolvedAgents)` after the IPC server is created. This is correct.
- `start-all.ts` (lines 144-165) still sends a redundant IPC `start-all` request after `waitForDaemon()` succeeds — this races against the daemon's own auto-start, can trigger double-starts, and is the source of the unreliability described in the phase goal.
- The fix is entirely in `start-all.ts`: remove the IPC `start-all` block and replace it with a simple status display.

```typescript
}
```

The daemon now handles agent boot internally. The CLI only needs to confirm the daemon is alive and show its initial status.
