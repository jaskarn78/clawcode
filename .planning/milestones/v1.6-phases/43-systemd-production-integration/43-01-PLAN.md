<objective>
Fix the systemd unit template in scripts/install.sh so the clawcode service starts reliably in production.

Purpose: The current unit has four concrete bugs that prevent a clean systemd start: (1) ExecStart calls the JS file directly without a node interpreter, (2) it passes the wrong sub-command (`daemon` instead of `start-all --foreground`), (3) PATH is absent so `op` and other binaries in /usr/bin are not found, (4) WorkingDirectory is set but ExecStart doesn't honour it for an absolute path anyway — but the path must be correct.

Output: Updated scripts/install.sh whose install_service() emits a working unit file. No new files. No behaviour changes outside the unit template.
</objective>

The CLI entry point is `dist/cli/index.js`.
The foreground sub-command signature (from start-all.ts) is:
  node dist/cli/index.js start-all --foreground --config <path>

daemon.ts already handles OP_SERVICE_ACCOUNT_TOKEN by reading /etc/clawcode/env itself
(line ~377), so systemd does NOT need to resolve secrets — it just needs the env file
present so the token reaches the daemon's environment. The EnvironmentFile line is
correct as-is; what is missing is the PATH variable.

Node.js is installed at /usr/bin/node (verified: `which node` → /usr/bin/node).
The `op` CLI (1Password) lives at /usr/bin/op on standard Ubuntu installations.

Current broken unit line:
  ExecStart=${CLAWCODE_DIR}/dist/cli/index.js daemon --config ${CLAWCODE_CONFIG}

Required fixed unit line:
  ExecStart=/usr/bin/node ${CLAWCODE_DIR}/dist/cli/index.js start-all --foreground --config ${CLAWCODE_CONFIG}

Missing environment line to add (after NODE_ENV line):
  Environment=PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin

<task type="auto">
  <name>Task 1: Fix systemd unit template in install_service()</name>
  <files>scripts/install.sh</files>
  <action>
Edit the heredoc inside install_service() (lines ~207-252). Make exactly these four changes:

1. Replace ExecStart line:
   BEFORE: ExecStart=${CLAWCODE_DIR}/dist/cli/index.js daemon --config ${CLAWCODE_CONFIG}
   AFTER:  ExecStart=/usr/bin/node ${CLAWCODE_DIR}/dist/cli/index.js start-all --foreground --config ${CLAWCODE_CONFIG}

   Rationale: The JS file has no shebang, so systemd cannot exec it directly. The
   sub-command was wrong ('daemon' does not exist; 'start-all --foreground' is the
   foreground entrypoint used for systemd/container deployments).

2. Add PATH to Environment block (insert after "Environment=HOME=/home/${CLAWCODE_USER}"):
   Environment=PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin

   Rationale: systemd services inherit a stripped PATH. Without this, the daemon
   cannot locate 'op' (/usr/bin/op) when resolving op:// secret references, nor
   other binaries claude agents may invoke.

3. Confirm WorkingDirectory line is already correct (${CLAWCODE_DIR} = /opt/clawcode).
   Do NOT change it — it is correct.

4. Confirm EnvironmentFile line reads: EnvironmentFile=-/etc/clawcode/env
   The leading '-' means "silently skip if missing" — this is correct behaviour.
   Do NOT change it.

After editing, verify the heredoc is syntactically intact: the closing EOF must be
at column 0 with no trailing spaces, and all ${VAR} expansions inside the heredoc
remain unquoted (the heredoc delimiter is unquoted 'EOF', so expansion is intentional).
