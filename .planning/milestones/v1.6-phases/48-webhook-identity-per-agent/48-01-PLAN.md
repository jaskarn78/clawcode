  <action>
    Create `src/discord/webhook-provisioner.ts` exporting:

    ```typescript
    import type { Client } from "discord.js";
    import type { WebhookIdentity } from "./webhook-types.js";
    import type { Logger } from "pino";

    export type ProvisionConfig = {
      readonly client: Client;
      readonly agents: readonly {
        readonly name: string;
        readonly channels: readonly string[];
        readonly webhook?: {
          readonly displayName: string;
          readonly avatarUrl?: string;
          readonly webhookUrl?: string;
        };
      }[];
      readonly manualIdentities: ReadonlyMap<string, WebhookIdentity>;
      readonly log: Logger;
    };

    export async function provisionWebhooks(config: ProvisionConfig): Promise<Map<string, WebhookIdentity>>;
    ```

    Implementation logic:
    1. Start with a new Map copied from `config.manualIdentities` (manual URLs preserved as-is).
    2. For each agent in `config.agents`:
       - If agent already in the map (has manual webhookUrl), skip.
       - If agent has no `webhook` config (no displayName), skip.
       - If agent has no channels, skip.
       - Take the first channel ID from `agent.channels`.
       - Fetch the channel via `config.client.channels.fetch(channelId)`.
       - If channel is not a text-based guild channel (check for `fetchWebhooks` method), log warning and skip.
       - Call `channel.fetchWebhooks()` to get existing webhooks.
       - Find a webhook owned by the bot: `webhooks.find(w => w.owner?.id === config.client.user?.id)`.
       - If found, use its `w.url` as the webhookUrl.
       - If not found, create one: `channel.createWebhook({ name: agent.webhook.displayName, avatar: agent.webhook.avatarUrl ?? null })` and use its `url`.
       - Set the identity in the map: `{ displayName: agent.webhook.displayName, avatarUrl: agent.webhook.avatarUrl, webhookUrl }`.
       - Wrap in try/catch per agent. On error, log and continue.
    3. Return the merged map.

    - Test: no existing webhook triggers createWebhook and uses returned URL.
    - Test: channel fetch error is caught and agent skipped (no throw).
