## Critical Gaps

### TIER-04: Escalation budget not wired at runtime

`EscalationBudget` class exists (`src/usage/budget.ts`) with 20 passing tests. However:
- `escalationBudget` field not added to `ResolvedAgentConfig` in `src/shared/types.ts`
- `resolveAgentConfig` in `src/config/loader.ts` does not pass through `escalationBudget`
- `EscalationBudget` never imported or instantiated in `src/manager/daemon.ts`
- `budgetOptions` never passed to `EscalationMonitor` constructor
- `alertCallback` never connected to `bridge.sendBudgetAlert`

**Fix:** 4-5 line changes across types.ts, loader.ts, and daemon.ts to wire the existing code.

### LOAD-03: graphContext slot empty

The context assembler has a `graphContext` source with a 2000-token budget, but `buildSessionConfig` passes `graphContext: ""`. No code pre-fetches graph-enriched context for the system prompt.

**Mitigation:** This is by design — graph context is available on-demand via `memory_lookup`. The slot exists for future pre-fetching. Acceptable as tech debt.

### TIER-05: Slash command indirect routing

The `/model` command uses `claudeCommand` pattern, routing through agent reasoning instead of direct IPC. This is the established pattern for all ClawCode slash commands, so it's consistent but relies on agent interpretation.

**Mitigation:** This matches the existing slash command architecture. All other slash commands work this way. Acceptable.
